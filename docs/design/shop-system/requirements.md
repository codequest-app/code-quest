## 核心概念

### 為什麼需要商店系統？

**問題陳述**:
- Claude Code 的功能管理（Skills、MCP、Subagent）過於技術化
- 用戶需要查閱文檔才能了解可用功能
- 缺乏探索和發現的樂趣
- 沒有成長和獎勵的感覺

**商店系統的優勢**:
- ✅ RPG 化功能管理（直觀、有趣）
- ✅ 探索式發現（像逛商店一樣）
- ✅ 成長獎勵系統（解鎖新功能）
- ✅ 引導式學習（新手友好）

### 設計原則

1. **沉浸式體驗**: 所有功能管理都融入 RPG 世界
2. **探索樂趣**: 鼓勵用戶主動探索新功能
3. **成長感**: 隨著等級提升解鎖更強大的功能
4. **簡化複雜**: 用 RPG 概念簡化技術概念
5. **保持實用**: 不只是裝飾，要真正幫助用戶

## 商店系統架構

### 城鎮廣場（Town Square）

城鎮作為所有商店的入口，提供統一的導航界面。

**城鎮佈局**:
```
🏰 RPG-CLI 城鎮 (探索模式專屬)

     🌳        🏪 廣場         🌳

   🔮           ⚒️           📚
 技能商店      工匠鋪      魔法圖書館

   🧙           🏆           🎯
 傭兵公會      寶物庫      訓練場

                💰
               錢莊
```

**進入方式**:
- 探索模式主菜單選擇「🏪 城鎮」
- 戰鬥中無法進入
- 部分功能可用快捷鍵（如 Skill 商店）

## 七大商店功能定位

### 1. 🔮 技能商店（Skills Shop）

**作用**: 查看、解鎖、升級所有 Skills（官方 + 自定義）

**映射關係**:
- Claude Code Skills → RPG 技能
- Skill 文檔 → 技能說明
- Skill 參數 → 技能屬性

**核心功能**:
- 瀏覽所有可用技能（官方和自定義）
- 查看技能詳情（MP 消耗、冷卻時間、效果）
- 解鎖新技能（需滿足等級和金幣要求）
- 升級已有技能（降低 MP、減少冷卻、增加傷害）
- 查看使用統計（使用次數、成功率）

**技能升級系統**:

**升級效果**:
1. 降低 MP 消耗
2. 減少冷卻時間
3. 增加傷害/效果
4. 解鎖新特性

**升級成本公式**:
```javascript
upgradeCost = basePrice * Math.pow(1.5, currentLevel - 1)

範例:
Lv.1 → Lv.2: 50 金幣
Lv.2 → Lv.3: 75 金幣
Lv.3 → Lv.4: 112 金幣
Lv.4 → Lv.5: 168 金幣
```

---

### 2. ⚒️ 工匠鋪（Skill Forge）

**作用**: 創建、編輯自定義 Skills

**映射關係**:
- 創建 Skill → 鍛造技能
- 編輯 Skill → 改良技能
- 測試 Skill → 試煉

**核心功能**:
- 六步驟技能創建引導流程
- 技能模板庫（快速創建常用技能）
- 技能測試和預覽
- 編輯現有自定義技能

**六步驟創建流程**:
1. **技能命名**: 設定名稱、圖標、描述
2. **命令設定**: 定義調用命令和實際執行命令
3. **資源設定**: 設置 MP 消耗和冷卻時間
4. **戰鬥效果**: 配置傷害、特殊效果、動畫
5. **解鎖條件**: 設置等級、金幣、前置技能要求（可選）
6. **預覽和測試**: 測試執行並保存

**預設模板分類**:
- 🧪 測試類：快速測試、覆蓋率測試、E2E 測試
- 📦 Git 類：快速提交、推送到遠端、創建分支
- 🔧 建構類：開發建構、生產建構、清理快取

---

### 3. 📚 魔法圖書館（MCP Tools Library）

**作用**: 瀏覽、安裝、配置 MCP 工具

**映射關係**:
- MCP 工具 → 魔法書
- 安裝工具 → 學習魔法
- 配置參數 → 調整魔法

**核心功能**:
- 瀏覽已安裝的 MCP 工具
- 發現和安裝新的 MCP 工具
- 查看工具詳情（功能、版本、評分、使用統計）
- 配置工具參數
- 卸載不需要的工具

**工具分類**:
- 已安裝
- 熱門
- 最新
- 分類（數據庫、網頁、AI、開發工具等）

**工具資訊**:
- 功能描述
- 版本號
- 作者
- 評分和評論
- 使用次數統計
- 系統需求
- 提供的魔法列表

---

### 4. 🧙 傭兵公會（Subagent Guild）

**作用**: 查看、管理、自定義 Subagent

**映射關係**:
- Subagent → 傭兵/夥伴
- Subagent 類型 → 傭兵職業
- Subagent 配置 → 傭兵訓練

**核心功能**:
- 瀏覽所有可用傭兵（Bash、Explore、Plan 等）
- 查看傭兵詳情（等級、屬性、技能）
- 管理當前隊伍
- 設置默認召喚配置
- 查看傭兵使用統計

**傭兵屬性**:
- 等級（Lv.1-5）
- 職業（系統工程師、代碼考古學家等）
- 專長（Git 操作、代碼搜索、架構理解等）
- 基礎屬性（HP、MP、攻擊、防禦）
- 技能列表（每個技能有 MP 消耗）
- 召喚成本

**隊伍管理**:
- 查看當前隊伍成員
- 設置默認召喚傭兵
- 配置自動召喚規則
- 優先級設置

---

### 5. 🏆 寶物庫（Treasury）

**作用**: 成就、獎勵、隱藏內容解鎖

**映射關係**:
- 成就 → 寶藏
- 獎勵 → 寶物
- 解鎖內容 → 隱藏寶箱

**核心功能**:
- 成就系統（查看、追蹤、領取獎勵）
- 收藏品圖鑑（召喚獸、技能、裝備等）
- 隱藏技能解鎖
- 成就進度追蹤

**成就類型**:
- 戰鬥成就（首次戰鬥、連勝、百戰精英等）
- 技能成就（技能使用次數、技能組合等）
- 探索成就（訪問所有商店、完成教學等）
- 特殊成就（達到高等級、完成隱藏任務等）

**獎勵內容**:
- 解鎖新技能
- 獲得召喚獸
- 金幣獎勵
- 特殊稱號

**收藏品圖鑑分類**:
- 火系召喚獸
- 水系召喚獸
- 特殊召喚獸
- 隱藏召喚獸（需達成特殊條件）

---

### 6. 🎯 訓練場（Training Ground）

**作用**: 安全測試環境、教學、練習

**映射關係**:
- 測試環境 → 訓練場
- 教學 → 訓練課程
- 模擬戰鬥 → 練習戰

**核心功能**:
- 新手教學課程
- 模擬戰鬥（不消耗資源）
- 技能測試（調試技能）
- Combo 練習（發現技能組合）

**新手教學課程**:
1. 第 1 課：基礎操作
   - 探索模式導覽
   - 對話與提問
   - 查看角色狀態

2. 第 2 課：技能使用
   - 如何使用技能
   - MP 和冷卻管理
   - 技能組合技巧

3. 第 3 課：戰鬥系統
   - 戰鬥流程
   - 召喚夥伴
   - 戰鬥策略

**模擬戰鬥特點**:
- 不消耗 HP/MP
- 不消耗技能冷卻
- 不獲得經驗值和金幣
- 可以無限重試

**技能測試模式**:
- 不消耗 MP
- 不進入冷卻
- 顯示詳細執行日誌
- 可以調整參數測試

**Combo 練習系統**:
- 查看已發現的 Combo
- 練習已知 Combo
- 自由組合發現新 Combo
- 提示未發現的神秘 Combo

---

### 7. 💰 錢莊（Bank）

**作用**: 資源統計、成本分析、預算管理

**映射關係**:
- 金幣統計 → 財務報表
- 成本分析 → 支出明細
- 預算管理 → 理財規劃

**核心功能**:
- 資源總覽（當前金幣、收支統計）
- 模型成本統計（Claude、Gemini 等）
- 收入明細（戰鬥獎勵、成就獎勵等）
- 支出明細（技能解鎖、升級、道具購買）
- 預算管理（設置每日預算、超支處理）
- 統計圖表（歷史趨勢分析）

**資源統計**:
- 當前金幣數量
- 今日收入/支出/淨收入
- AI 模型成本總計
- 預算使用率

**模型成本統計**:
- 各模型使用次數和成本
- 總成本和預算比較
- 成本節省分析
- 使用率圖表

**預算管理**:
- 每日 AI 模型成本預算
- 每日金幣預算
- 超出預算時的處理策略：
  - 切換到便宜模型
  - 停止使用
  - 僅警告，繼續使用
- 金幣透支設置

**統計圖表**:
- 金幣趨勢（過去 7 天）
- 成本趨勢（過去 7 天）
- 收支對比
- 導出報表

## 與現有系統整合

### 場景系統整合（Scene System）

**探索模式主菜單擴展**:
```
探索模式主菜單:
├─ 📊 狀態
├─ 🔮 技能
├─ 🧙 夥伴
├─ 🏆 成就
├─ 🏪 城鎮 ⭐ 新增
│  ├─ 🔮 技能商店
│  ├─ ⚒️ 工匠鋪
│  ├─ 📚 魔法圖書館
│  ├─ 🧙 傭兵公會
│  ├─ 🏆 寶物庫
│  ├─ 🎯 訓練場
│  └─ 💰 錢莊
├─ 🌲 Worktree
└─ ⚙️ 設定
```

**快捷訪問**:
- 按 `K` 鍵快速打開技能商店
- 按 `G` 鍵快速打開傭兵公會
- 按 `M` 鍵快速打開 MCP 圖書館

**場景切換規則**:
- 城鎮只能在探索模式進入
- 戰鬥中無法訪問城鎮
- 從城鎮可以返回探索模式
- 訓練場中可以進行模擬戰鬥

---

### 戰鬥系統整合（Battle System）

**戰鬥中快捷訪問**:
- 技能選擇時顯示技能商店中的資訊（等級、MP、冷卻）
- 召喚夥伴時顯示傭兵公會資訊（屬性、技能）
- 成就達成時自動彈出寶物庫通知

**技能資訊同步**:
- 戰鬥中使用的技能數據來自技能商店
- 技能升級立即影響戰鬥表現
- 技能使用次數統計回傳給商店

**獎勵發放**:
- 戰鬥勝利獲得金幣
- 達成成就彈出通知
- 經驗值影響等級和解鎖

---

### 異步戰鬥系統整合（Async Battle System）

**非阻塞設計**:
- 商店操作不阻塞戰鬥進行
- 可以在戰鬥背景運行時訪問訓練場
- 模擬戰鬥與真實戰鬥隔離

**後台操作**:
- 技能升級在後台完成
- MCP 工具安裝異步處理
- 成就檢測實時進行

---

### 多模型整合（Multi-Model Integration）

**錢莊整合成本統計**:
- 追蹤 Claude 使用次數和成本
- 追蹤 Gemini 使用次數和成本
- 計算節省比例
- 預算超支時自動切換模型

**模型選擇策略**:
- 根據預算自動選擇模型
- 簡單任務優先使用便宜模型
- 複雜任務使用強大模型
- 用戶可手動設置策略

---

### 地圖系統整合（Map System）

**城鎮位置**:
- 城鎮位於地圖上的商業區
- 從地圖可以直接進入城鎮
- 城鎮是探索模式的重要地標

**商店圖標**:
- 地圖上顯示城鎮圖標 🏪
- 點擊城鎮圖標打開商店廣場
- 新功能解鎖時圖標有通知標記

## 技術需求

### 數據持久化

**需要保存的數據**:
- 用戶擁有的技能和等級
- 技能升級歷史
- 已安裝的 MCP 工具和配置
- Subagent 隊伍配置
- 成就達成狀態
- 收藏品解鎖狀態
- 金幣和資源統計
- 使用統計（技能使用次數、成功率等）
- 模型成本統計

**數據格式**:
- 使用 JSON 文件存儲用戶數據
- 使用 SQLite 存儲統計數據（可選）
- 配置文件使用 YAML 或 JSON

---

### API 設計

**商店管理 API**:
```typescript
// 獲取技能列表
getSkills(filter?: SkillFilter): Skill[]

// 解鎖技能
unlockSkill(skillId: string): Promise<boolean>

// 升級技能
upgradeSkill(skillId: string): Promise<boolean>

// 創建自定義技能
createCustomSkill(config: SkillConfig): Promise<Skill>
```

**成就系統 API**:
```typescript
// 檢查成就條件
checkAchievements(): void

// 領取成就獎勵
claimAchievement(achievementId: string): Promise<Reward>

// 獲取成就進度
getAchievementProgress(achievementId: string): Progress
```

**資源管理 API**:
```typescript
// 獲取金幣餘額
getGoldBalance(): number

// 扣除金幣
deductGold(amount: number): boolean

// 添加金幣
addGold(amount: number, source: string): void

// 獲取成本統計
getCostStats(period: TimePeriod): CostStats
```

### 性能考慮

**優化策略**:
- 技能列表懶加載（分頁顯示）
- 成就檢測防抖（避免頻繁檢測）
- 統計數據緩存（減少計算）
- 圖鑑圖片懶加載

**資源限制**:
- 單個用戶最多 100 個自定義技能
- 成就系統最多追蹤 200 個成就
- 統計數據保留最近 30 天
