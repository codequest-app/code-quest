# 商店系統（Shop System）

**依賴**: 地圖系統
**被依賴**: 無

---

## 📖 系統概要

商店系統將 Claude Code 的功能管理（Skills、MCP、Subagent）RPG 化為城鎮中的 7 個商店。用戶以探索式方式發現和解鎖功能，而非查閱技術文檔。每個商店提供特定類型的服務，營造沉浸式的 RPG 體驗，同時簡化複雜的技術概念。

---

## 🎯 核心功能

### 7 個商店

**🔮 技能商店（Skill Shop）**
- 功能：購買和解鎖新 Skills
- 位置：商業街左側
- 解鎖條件：等級需求
- 貨幣：金幣

**⚒️ 工匠鋪（Skill Forge）**
- 功能：升級現有技能、自定義技能
- 位置：商業街中央
- 特色：Skill Forge 六步驟流程
- 貨幣：金幣 + 技能點

**📚 魔法圖書館（Knowledge Library）**
- 功能：查閱文檔、教學指南
- 位置：商業街右側
- 免費：知識問答不消耗資源
- 特色：互動式教學

**🧙 傭兵公會（Mercenary Guild）**
- 功能：召喚和管理 Subagent 夥伴
- 位置：公會大廳區域
- 特色：查看夥伴屬性、設定默認夥伴
- 貨幣：金幣（召喚費用）

**🏆 寶物庫（Treasure Vault）**
- 功能：購買道具、Buff、消耗品
- 位置：商業街深處
- 稀有度：普通/稀有/史詩/傳說
- 貨幣：金幣

**💰 錢莊（Cost Exchange）**
- 功能：成本追蹤、模型切換、預算管理
- 位置：商業街入口
- 特色：實時成本顯示、模型比較
- 整合：多模型整合系統

**🎯 訓練場（Training Ground）**
- 功能：測試技能、模擬戰鬥、教學任務
- 位置：城鎮外圍
- 免費：測試不消耗資源
- 特色：安全環境學習

### 依賴關係

- **地圖系統** → 所有商店位於城鎮商業街/公會大廳

- **道具系統** → 寶物庫購買和管理道具

## 💡 設計要點

### 為什麼需要商店系統？

**問題**：
- Claude Code 功能管理過於技術化
- 用戶需要查閱文檔才能了解可用功能
- 缺乏探索和發現的樂趣
- 沒有成長和獎勵的感覺

**解決方案**：
- RPG 化功能管理（直觀、有趣）
- 探索式發現（像逛商店一樣）
- 成長獎勵系統（解鎖新功能）
- 引導式學習（新手友好）

### Skill Forge 六步驟

工匠鋪的核心功能，自定義和升級技能：

```
1. 選擇基礎技能
   ↓
2. 調整 Prompt 模板
   ↓
3. 配置工具權限
   ↓
4. 設定 Metadata（MP、冷卻、圖標）
   ↓
5. 測試技能（在訓練場）
   ↓
6. 保存和命名
```

### 典型流程

**購買新技能**：
```
1. 進入城鎮 → 商業街
2. 進入技能商店
3. 瀏覽可用技能（按分類）
4. 查看技能詳情（MP、冷卻、效果）
5. 檢查解鎖條件（等級、前置技能）
6. 購買（消耗金幣）
7. 技能加入技能欄
```

**升級技能（Skill Forge）**：
```
1. 進入工匠鋪（需 Lv.5+）
2. 選擇要升級的技能
3. 六步驟流程：
   - 調整 Prompt
   - 配置工具
   - 設定 Metadata
   - 測試
   - 保存
4. 消耗金幣 + 技能點
5. 技能升級完成
```
