# Worktree 手動管理系統

**依賴**: 地圖系統（公會大廳）、場景系統
**被依賴**: 非同步戰鬥系統

---

## 📖 系統概要

Worktree 手動管理系統將 Git Worktree 功能 RPG 化為「平行世界」系統。用戶可在公會大廳的時空管理器中手動創建、切換、合併平行世界，每個世界對應一個 Git worktree，實現多分支並行開發的沉浸式體驗。

---

## 🎯 核心功能

### 1. 平行世界概念

```
Git Worktree          →    平行世界 RPG
────────────────────────────────────────
main branch           →    主世界
feature branch        →    特性世界（冒險）
fix branch            →    修復世界（緊急任務）
experiment branch     →    實驗世界（探索）
hotfix branch         →    熱修復世界（危機處理）
```

### 2. 五種世界類型

**主世界（Main World）** 🏰
- 對應：main/master 分支
- 特性：最高保護級別，不可刪除
- 圖標：城堡
- 作用：穩定版本基地

**特性世界（Feature World）** ⚔️
- 對應：feature/* 分支
- 特性：開發新功能
- 圖標：劍
- 作用：功能開發的冒險世界

**修復世界（Fix World）** 🛡️
- 對應：fix/* 分支
- 特性：修復 Bug
- 圖標：盾牌
- 作用：緊急任務世界

**實驗世界（Experiment World）** 🔮
- 對應：experiment/* 分支
- 特性：臨時測試，可隨時刪除
- 圖標：水晶球
- 作用：安全探索新想法

**熱修復世界（Hotfix World）** 🚨
- 對應：hotfix/* 分支
- 特性：緊急修復，最高優先級
- 圖標：警報
- 作用：危機處理世界

### 3. 時空管理器介面

提供三種視圖管理平行世界：

**列表視圖**：
- 所有世界的清單
- 顯示世界名稱、類型、創建時間、狀態
- 快速切換和操作

**時間線圖視圖**：
- 分支歷史的視覺化
- 分支點和合併點標記
- 提交歷史時間線

**看板視圖**：
- 按類型分組（進行中/完成/已合併）
- 拖拽操作
- 適合管理多個並行任務

### 4. 世界操作

**創建新世界**：
- MP 消耗：10
- 選擇基礎世界（從哪個分支創建）
- 命名和選擇類型
- 自動創建對應 worktree

**切換世界**：
- MP 消耗：5
- 保存當前世界狀態（可選 stash）
- 切換到目標世界
- 同步 UI 狀態

**合併世界**：
- MP 消耗：20
- 選擇源世界和目標世界
- 執行 Git 合併
- 獲得經驗值獎勵
- 可選刪除源世界

**刪除世界**：
- 確認對話框（防止誤刪）
- 主世界不可刪除
- 自動清理 worktree 目錄

### 5. Stash 管理

**臨時存檔系統**：
- 保存未提交的變更
- 跨世界移動變更
- Stash 列表管理
- 應用/刪除 stash

## 🔗 與其他系統的關係

### 依賴關係

- **地圖系統** → 公會大廳是 Worktree 管理入口
- **場景系統** → 只能在探索模式操作

### 被依賴關係

- **非同步戰鬥系統** → 自動創建 Worktree 用於並發戰鬥

### 兩種 Worktree 模式

| 特性 | 手動管理 | 自動管理（非同步戰鬥） |
|-----|---------|---------------------|
| 觸發方式 | 用戶主動在公會大廳操作 | 複雜戰鬥自動創建 |
| 生命週期 | 用戶手動刪除 | 戰鬥結束自動清理 |
| 用途 | 功能開發、Bug 修復 | 並發任務隔離 |
| 可見性 | 在時空管理器可見 | 在後台戰鬥面板可見 |

## 💡 設計要點

### 為什麼需要平行世界系統？

**問題**：
- Git Worktree 功能強大但難以理解
- 多分支開發缺乏視覺化管理
- 用戶不清楚何時使用 worktree
- 切換分支容易遺失未提交變更

**解決方案**：
- 平行世界比喻易於理解
- 視覺化管理介面
- RPG 化操作降低學習曲線
- Stash 管理保護變更

### 關鍵設計決策

1. **探索模式專屬**
   - 戰鬥中無法操作，避免混亂
   - 在公會大廳安全管理

2. **MP 消耗機制**
   - 創建/切換/合併需要 MP
   - 鼓勵謹慎使用
   - 防止濫用

3. **五種世界類型**
   - 簡化 Git 分支策略
   - 提供清晰的用途分類

## 🚀 快速參考

### 世界類型速查

| 類型 | 圖標 | 創建 MP | 切換 MP | 適用場景 |
|-----|------|--------|--------|---------|
| 主世界 | 🏰 | - | 5 | 穩定版本 |
| 特性世界 | ⚔️ | 10 | 5 | 新功能開發 |
| 修復世界 | 🛡️ | 10 | 5 | Bug 修復 |
| 實驗世界 | 🔮 | 10 | 5 | 實驗測試 |
| 熱修復世界 | 🚨 | 10 | 5 | 緊急修復 |

### 操作流程速查

**創建新世界**：
```
1. 進入公會大廳
2. 打開時空管理器
3. 點擊「創建新世界」
4. 選擇基礎世界（從哪個分支）
5. 輸入世界名稱
6. 選擇世界類型
7. 確認（消耗 10 MP）
8. 自動創建 worktree 和切換
```

**切換世界**：
```
1. 打開時空管理器
2. 選擇目標世界
3. （可選）保存當前變更到 Stash
4. 點擊「進入世界」
5. 確認（消耗 5 MP）
6. 切換完成
```

**合併世界**：
```
1. 打開時空管理器
2. 選擇要合併的世界
3. 選擇目標世界（通常是主世界）
4. 預覽合併操作
5. 確認（消耗 20 MP）
6. 執行 Git 合併
7. 獲得經驗值獎勵
8. （可選）刪除源世界
```
