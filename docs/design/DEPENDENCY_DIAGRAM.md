## 🔗 關鍵依賴關係說明

### 場景系統的核心地位
場景系統是 **L0 基礎層** 但被多個系統依賴：
- **戰鬥系統 (3)** → 判斷是否在戰鬥模式
- **商店系統 (6)** → 只能在探索模式訪問
- **Worktree 系統 (8)** → 只能在探索模式操作
- **互動事件系統 (7)** → Plan Mode 需要進入「靜止之間」

### 戰鬥系統的樞紐地位
戰鬥系統被 **5 個系統** 依賴，是核心樞紐：
- 夥伴系統、召喚獸系統、互動事件系統（L2 擴展）
- 非同步戰鬥系統、多模型整合系統（L3 高級）

### 地圖系統的基礎地位
地圖系統是 **唯一無依賴** 的系統，被 3 個系統依賴：
- 場景系統 (2)
- 商店系統 (6)
- Worktree 系統 (8)

### 特殊關係

**夥伴 vs 召喚獸**（並列關係）：
- 都依賴戰鬥系統
- 互相獨立，無依賴關係
- 功能互補：夥伴=持續支援，召喚獸=爆發支援

**非同步戰鬥的雙重依賴**：
- 依賴戰鬥系統（執行戰鬥邏輯）
- 依賴 Worktree 系統（自動創建隔離環境）

**多模型整合的跨層依賴**：
- 依賴戰鬥系統（執行 AI 任務）
- 依賴商店系統（錢莊顯示成本和切換模型）

## 🎯 實作建議順序

基於依賴關係，建議的實作順序：

### 階段 1：基礎層 (L0)
```
1. 地圖系統 (1) ← 無依賴，最先實作
2. 場景系統 (2) ← 依賴地圖
3. 商店系統 (6) ← 依賴地圖 + 場景
4. Worktree系統 (8) ← 依賴地圖 + 場景
```

### 階段 2：核心戰鬥 (L1)
```
5. 戰鬥系統 (3) ← 依賴場景
```

### 階段 3：戰鬥擴展 (L2)
```
6. 夥伴系統 (4) ← 依賴戰鬥
7. 召喚獸系統 (5) ← 依賴戰鬥
8. 互動事件系統 (7) ← 依賴戰鬥 + 場景
```

### 階段 4：高級功能 (L3)
```
9. 多模型整合系統 (10) ← 依賴戰鬥 + 商店
10. 非同步戰鬥系統 (9) ← 依賴戰鬥 + Worktree
```

### 階段 5：呈現層 (L4)
```
11. UI 互動系統 (11) ← 依賴所有系統
```
