# 地圖系統（Map System）

**依賴**: 無（基礎系統）
**被依賴**: 場景系統、商店系統、Worktree 系統

---

## 📖 系統概要

地圖系統提供 Code Quest 世界的空間結構，定義城鎮、野外、副本三大區域。解決了抽象「模式」的問題，讓用戶在具體的「場所」中活動，提升沉浸感和空間感。每個場所有明確的功能和限制，遭遇戰只在特定區域發生，符合傳統 RPG 玩家的直覺。

---

## 🎯 核心功能

### 三大區域

1. **🏰 城鎮區域（Town Zone）**
   - 安全區，不會遭遇戰鬥
   - 提供商業街、酒館、公會大廳、靜止之間等服務
   - 休息、補給、交易、管理的中心

2. **🌲 野外區域（Wilderness）**
   - 危險區域，會隨機遭遇戰鬥
   - 任務型 Prompt 在此觸發戰鬥
   - 執行代碼任務的主要場所

3. **🏔️ 副本區域（Dungeons）**
   - 特定任務專屬區域
   - 劇情戰鬥和 Boss 戰
   - 完成前無法退出

### 場所管理

- **場所定義**：每個場所有名稱、類型、可用功能、訪問條件
- **地圖導航**：用戶可在場所間移動，查看當前位置
- **遭遇戰機制**：野外區域隨機遭遇，副本區域固定遭遇
- **限制規則**：戰鬥中無法切換場所

## 🔗 與其他系統的關係

### 作為基礎系統

地圖系統是其他系統的空間載體：

- **場景系統** → 依賴地圖系統定義的區域劃分
- **商店系統** → 所有商店位於城鎮商業街
- **Worktree 系統** → 公會大廳是 Worktree 管理入口
- **戰鬥系統** → 野外和副本區域觸發戰鬥

### 關鍵整合點

1. **城鎮 → 商業街**
   - 7 個商店的物理位置
   - 參見 [商店系統](../shop-system/)

2. **城鎮 → 公會大廳**
   - Worktree「平行世界」管理
   - 參見 [Worktree 系統](../worktree-manual-system/)

3. **城鎮 → 靜止之間**
   - Plan Mode 的物理位置
   - 參見 [場景系統](../scene-system/)

4. **野外 + 副本 → 戰鬥觸發**
   - 遭遇戰生成規則
   - 參見 [戰鬥系統](../battle-system/)

## 💡 設計要點

### 為什麼需要地圖系統？

**問題**：
- 當前設計只有抽象的「探索模式」和「戰鬥模式」
- 商店設計了 7 個，但沒說明它們在哪裡
- 用戶可以在任何地方觸發任何操作，缺乏空間感

**解決方案**：
- 地圖提供具體的場所（城鎮、野外、副本）
- 每個場所有明確的功能和限制
- 遭遇戰只在特定區域發生（傳統 RPG 規則）

### 關鍵設計決策

1. **三區域劃分** - 簡單清晰，符合傳統 RPG
2. **城鎮安全區** - 讓用戶有安心管理的空間
3. **野外危險區** - 增加緊張感和探險感
4. **副本特殊區** - 提供劇情和挑戰

## 🚀 快速參考

### 場所速查

| 區域 | 場所 | 功能 | 限制 |
|-----|------|------|------|
| 城鎮 | 商業街 | 7個商店 | 探索模式專屬 |
| 城鎮 | 酒館 | AI對話、任務公告 | 隨時可進 |
| 城鎮 | 公會大廳 | Worktree管理 | 探索模式專屬 |
| 城鎮 | 靜止之間 | Plan Mode | 戰鬥觸發時自動進入 |
| 野外 | 森林/山脈/平原 | 代碼任務、遭遇戰 | 隨機戰鬥 |
| 副本 | 特定副本 | 劇情戰鬥、Boss戰 | 無法退出，完成才離開 |

### 遭遇戰觸發規則

- **城鎮**: 永不觸發戰鬥
- **野外**:
  - 隨機遭遇（5% 機率/分鐘）
  - 任務型 Prompt 100% 觸發
- **副本**: 固定位置觸發劇情戰鬥
