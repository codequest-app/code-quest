# 地圖系統 - 需求定義

**日期**: 2026-02-05
**版本**: v1.0
**狀態**: 設計階段

---

## 核心概念

### 設計理念

```
RPG 需要具體的「空間」而不是抽象的「模式」

問題:
- 當前設計只有抽象的「探索模式」和「戰鬥模式」
- 商店系統設計了 7 個商店，但沒說明它們在哪裡
- 用戶可以在任何地方觸發任何操作，缺乏空間感和沉浸感

解決方案:
- 地圖 = 具體的場所（城鎮、野外、副本）
- 每個場所有明確的功能和限制
- 遭遇戰只在特定區域發生（傳統 RPG 規則）
- 用戶通過地圖導航在不同場所間移動
```

### 世界觀設定

```
CodeLand 世界

你是一名代碼魔法師，居住在 CodeLand 的世界中。
這個世界由三大區域組成：

🏰 城鎮區域（Town）
  - 安全區，不會遭遇戰鬥
  - 提供各種服務和設施
  - 休息、補給、交易、管理的中心

🌲 野外區域（Wilderness）
  - 危險區域，會隨機遭遇戰鬥
  - 任務型 Prompt 在此觸發戰鬥
  - 執行代碼任務的主要場所

🏔️ 副本區域（Dungeons）
  - 特定任務專屬區域
  - 劇情戰鬥和 Boss 戰
  - 完成前無法退出
```

### 概念映射

```
傳統 RPG              →    CodeLand RPG
─────────────────────────────────────────
城鎮（休息、購物）     →    城鎮區域
野外（遭遇戰）         →    野外區域（代碼任務戰鬥）
副本（Boss 戰）        →    副本區域（複雜任務）
酒館（接任務、對話）   →    酒館（AI 對話）
商業街（購物）         →    商業街（7 個商店）
冒險者公會             →    公會大廳（Worktree 管理）
修行室                 →    靜止之間（Plan Mode）
```

## 區域定義

### 1. 🏰 城鎮區域（Town Zone）

#### 核心概念
- **安全區**：不會觸發戰鬥
- **服務中心**：提供各種開發輔助功能
- **社交空間**：與 AI 對話、管理資源

#### 場所列表

##### 1.1 🏪 商業街（Shopping District）

**位置**: 城鎮中心偏東

**功能**: 包含 7 個商店，提供技能、工具、夥伴管理

**子場所**:
```
商業街
├─ 🔮 技能商店（Skills Shop）
│  - 查看、解鎖、升級技能
│  - 技能詳情和試用
│
├─ ⚒️ 工匠鋪（Skill Forge）
│  - 創建自定義技能
│  - 6 步驟引導式創建流程
│
├─ 📚 魔法圖書館（MCP Tools Library）
│  - 瀏覽和安裝 MCP 工具
│  - 工具配置和管理
│
├─ 🧙 傭兵公會（Subagent Guild）
│  - 管理戰鬥夥伴（Subagents）
│  - 查看夥伴能力和專長
│
├─ 🏆 寶物庫（Treasury）
│  - 查看成就和獎勵
│  - 召喚獸圖鑑
│  - 收集品展示
│
├─ 🎯 訓練場（Training Ground）
│  - 教學和練習
│  - 模擬戰鬥（不消耗資源）
│  - 技能測試
│
└─ 💰 錢莊（Bank）
   - 資源統計（金幣、經驗值）
   - AI 模型成本追蹤
   - 預算管理
```

**限制**:
- ❌ 戰鬥中無法進入商業街
- ✅ 探索模式隨時可進入
- ✅ 可快捷鍵直接進入特定商店（1-7）

**互動方式**:
- 走到商業街區域 → 點擊特定商店建築 → 進入商店
- 或使用快捷鍵（1: 技能商店, 2: 工匠鋪, 3: 圖書館...）

##### 1.2 🍺 酒館（Tavern）

**位置**: 城鎮中心偏西

**功能**: AI 對話、日常討論、技術諮詢

**使用場景**:
```
場景 1: 日常對話
用戶: "解釋一下 React 的 useEffect"
→ 在酒館與 AI 對話
→ 不觸發戰鬥
→ 即時回答

場景 2: 技術諮詢
用戶: "我應該用 Redux 還是 Zustand？"
→ 在酒館討論
→ AI 提供建議
→ 不消耗 MP

場景 3: 規劃討論
用戶: "我想要重構整個架構，該怎麼做？"
→ AI 建議："這是複雜任務，要進入靜止之間制定計劃嗎？"
→ 可選擇進入靜止之間（Plan Mode）
```

**NPC**:
- 🧙‍♂️ **酒館老闆（AI 助手）**: 友善的 AI，回答各種問題
- 👥 **其他冒險者（未來擴展）**: 查看其他開發者的任務進度

**特性**:
- 💬 對話型 Prompt 主要處理場所
- 🆓 不消耗 MP
- ⚡ 即時回答（主 Claude CLI）
- 🔗 可引導到其他場所（如：靜止之間、野外）

**限制**:
- ✅ 任何時候都可進入（除了副本中）
- ❌ 不能在酒館觸發戰鬥

---

##### 1.3 ⏸️ 靜止之間（Stasis Chamber）

**位置**: 城鎮最北端（獨立建築）

**功能**: Plan Mode 專屬空間，深度思考和戰術規劃

**概念**:
- 時間凍結的規劃室
- 進入後外界時間暫停
- 可以無限時間制定計劃
- 類似「精神時光屋」的獨立空間

**視覺設計**:
```
外觀:
- 神秘的圓形建築
- 灰白色調
- 周圍有靜止的粒子效果
- 時鐘指針靜止不動

內部:
- 抽象的空間（無重力感）
- 灰色調世界
- 靜止的代碼片段漂浮
- 戰術規劃看板
- AI 助手化身為引導者
```

**使用場景**:

**場景 1: 戰鬥前規劃（主動進入）**
```
用戶在野外遭遇強敵
→ 複雜度分析: 15（非常困難）
→ AI 建議: "敵人過於強大，是否進入靜止之間制定戰術？"
→ 用戶選擇進入
→ 傳送到靜止之間
→ 外界時間暫停
→ 在靜止之間中 Plan Mode 展開
→ 查看敵人弱點、制定策略、選擇技能
→ 規劃完成後回到野外
→ 時間恢復流動
→ 開始戰鬥
```

**場景 2: 複雜任務規劃（自動觸發）**
```
用戶: "重構整個系統架構"
→ AI 偵測到需要 Plan Mode
→ 自動引導: "這需要深思熟慮，帶你進入靜止之間"
→ 傳送到靜止之間
→ Plan Mode 在此展開
→ AI 協助制定計劃
→ 完成後決定是否執行
```

**場景 3: 主動進入深度思考**
```
用戶在城鎮
→ 走到靜止之間建築
→ 點擊進入
→ AI: "歡迎來到靜止之間，在這裡你可以冷靜思考"
→ 可以討論任何複雜問題
→ 啟動 Plan Mode 工具
```

**特性**:
- ⏸️ 時間暫停（遊戲內時間不流動）
- 🧠 深度思考模式
- 📋 Plan Mode 專屬場所
- 🎯 戰術規劃看板
- 💡 AI 深度分析輔助

**台詞**:
```
進入時:
"時間在此靜止，唯有你的思緒在流動。"
"靜止之間為你開啟，專注思考，找到答案。"

規劃時:
"冷靜下來，一步一步思考..."
"時間不會催促你，慢慢來。"

離開時:
"帶著你的計劃回到世界吧。"
"時間將再次流動。"
```

**限制**:
- ✅ 探索模式隨時可進入
- ❌ 戰鬥中無法進入（但戰鬥中可暫停進入）
- ✅ Plan Mode 只能在此處觸發

##### 1.4 🏛️ 公會大廳（Guild Hall）

**位置**: 城鎮中心

**功能**: Worktree 管理中心（平行時間線管理）

**概念**:
- 管理你的所有「平行世界」（Worktrees）
- 查看正在進行的戰鬥（綁定的 Worktrees）
- 手動創建/切換/合併 Worktrees

**使用場景**:

**場景 1: 查看所有 Worktrees**
```
用戶進入公會大廳
→ 看到「時空管理器」介面
→ 列表顯示:
   - 🏰 主世界（main）- 穩定
   - ⚔️ 戰鬥分支 1（auth-refactor）- 戰鬥中
   - ⚔️ 戰鬥分支 2（payment-fix）- 戰鬥中
   - 🔮 實驗分支（ui-redesign）- 閒置
→ 可點擊查看詳情或切換
```

**場景 2: 手動創建 Worktree**
```
用戶: "我想開一個新分支試驗新想法"
→ 在公會大廳點擊「創建新時間線」
→ 選擇類型（feature/fix/experiment/hotfix）
→ 填寫資訊（名稱、描述、預估時間）
```

**場景 3: 管理已完成的戰鬥**

```
戰鬥結束後，用戶選擇「保留分支」

   - 繼續在該分支工作
```

**NPC**:
- 👨‍💼 **公會管理員**: 協助 Worktree 管理
- 📋 **任務公告板**: 顯示正在進行的戰鬥列表

**介面**:
- 時空管理器（Worktree 列表）
- 創建新時間線按鈕
- 清理建議
- 統計面板

**特性**:
- 📊 可視化所有 Worktrees
- ⚡ 快速切換
- 🔀 合併管理
- 🧹 智能清理建議

**限制**:
- ✅ 探索模式隨時可進入
- ❌ 戰鬥中無法操作 Worktree（但可查看）
- ✅ 戰鬥自動創建的 Worktree 也會顯示在此

### 2. 🌲 野外區域（Wilderness Zone）

#### 核心概念
- **危險區域**：會隨機遭遇戰鬥
- **任務執行地**：代碼任務在此觸發戰鬥
- **資源豐富**：完成戰鬥獲得經驗值和金幣

#### 子區域

```
野外
├─ 🌳 森林（Forest）
│  - 入門級任務（複雜度 5-8）
│  - 遭遇率：中等
│
├─ ⛰️ 山脈（Mountains）
│  - 中級任務（複雜度 8-12）
│  - 遭遇率：高
│
├─ 🏜️ 荒野（Wasteland）
│  - 高級任務（複雜度 12-15）
│  - 遭遇率：很高
│
└─ 🌋 火山（Volcano）
   - 專家級任務（複雜度 15+）
   - 遭遇率：極高
```

#### 遭遇戰機制

**觸發條件**:
```javascript
// 在野外時，任務型 Prompt 觸發戰鬥
if (currentZone === 'wilderness' && promptType === 'task') {
  const complexity = analyzeComplexity(prompt);

  if (complexity >= 8) {
    // 創建戰鬥
    startBattle({
      prompt,
      complexity,
      zone: currentSubZone  // forest/mountains/wasteland/volcano
    });
  } else {
    // 簡單任務，在主 CLI 完成
    handleSimpleTask(prompt);
  }
}
```

**使用場景**:

**場景 1: 簡單任務（複雜度 < 8）**
```
用戶在野外
→ "幫我修改這個按鈕顏色"
→ 複雜度: 3
→ 不觸發戰鬥
→ 主 CLI 直接完成
→ 提示: "任務太簡單，不需要戰鬥"
```

**場景 2: 中等任務（複雜度 8-12）**
```
用戶在森林區域
→ "實作用戶登入功能"
→ 複雜度: 10
→ 觸發遭遇戰！
→ 生成敵人: "功能實現魔物 Lv.10"
→ 提示: "遭遇戰鬥！"
→ 自動創建 worktree: feature/user-login
→ 啟動戰鬥實例
→ 戰鬥開始
```

**場景 3: 困難任務（複雜度 12+）**
```
用戶在山脈區域
→ "重構整個架構"
→ 複雜度: 15
→ 觸發強敵！
→ 生成敵人: "架構重構惡魔 Lv.15"
→ AI 建議: "敵人很強，是否先進入靜止之間制定戰術？"
→ 用戶可選擇:
   - 直接戰鬥
   - 先進入靜止之間（Plan Mode）
```

**遭遇率設定**:

```javascript
  mountains:  0.5,  // 50%
  wasteland:  0.7,  // 70%

}

// 隨機遭遇（未來擴展）
// 在野外移動時，每 N 步有機率觸發隨機戰鬥
```

#### 特性
- ⚔️ 戰鬥主要發生地
- 🌍 開放世界探索
- 📈 敵人等級隨區域提升
- 💎 戰鬥獎勵豐富

#### 限制
- ❌ 戰鬥中無法進入城鎮（需先完成或逃跑）
- ✅ 可隨時返回城鎮（非戰鬥時）
- ✅ 可在野外使用道具和技能

### 3. 🏔️ 副本區域（Dungeon Zone）

#### 核心概念
- **特定任務專屬**：特殊的複雜任務或劇情任務
- **完成前無法退出**：需要完成戰鬥才能離開
- **Boss 戰**：高難度敵人和特殊機制

#### 副本類型

```
副本列表
├─ 🐛 Bug 洞窟（Bug Cave）
│  - 類型：Debug 任務專屬
│  - Boss：Bug 之王
│  - 特殊機制：需要追蹤錯誤日誌
│
├─ 🏗️ 架構迷宮（Architecture Maze）
│  - 類型：架構重構任務專屬
│  - Boss：遺留代碼守護者
│  - 特殊機制：需要制定完整計劃
│
├─ 🧪 測試試煉場（Testing Arena）
│  - 類型：測試覆蓋任務專屬
│  - Boss：測試覆蓋監督者
│  - 特殊機制：需要達到 100% 覆蓋率
│
└─ 🔒 安全地牢（Security Dungeon）
   - 類型：安全漏洞修復任務
   - Boss：漏洞注入魔王
   - 特殊機制：需要通過安全檢查
```

#### 觸發方式

**方式 1: 特定 Prompt 觸發**
```
用戶: "修復這個嚴重的安全漏洞"
→ 偵測到「安全」關鍵字 + 高複雜度
→ AI: "這是安全地牢任務，一旦進入無法退出，確定要開始嗎？"
→ 用戶確認
→ 傳送到安全地牢
→ 戰鬥開始
```

**方式 2: 主動進入（未來擴展）**
```
在城鎮的副本入口
→ 點擊「Bug 洞窟」
→ 顯示副本資訊（難度、獎勵、建議等級）
→ 點擊「挑戰」
→ 進入副本
```

#### 特性
- 🎯 目標明確（修復 Bug、重構、測試）
- 🔒 進入後無法退出（完成前）
- 👑 Boss 戰鬥
- 🏆 豐厚獎勵

#### 限制
- ❌ 進入後無法返回城鎮
- ❌ 無法在副本中創建新 Worktree（已綁定）
- ✅ 可以使用道具和召喚夥伴
- ✅ 可以在副本中進入靜止之間（Boss 戰前）

## 遭遇戰機制詳細設計

### 觸發邏輯

```javascript
class EncounterSystem {
  /**
   * 檢查是否觸發遭遇戰
   */
  checkEncounter(prompt, currentZone) {
    // 1. 檢查當前區域
    if (currentZone === 'town') {
      return { encounter: false, reason: '城鎮是安全區' };
    }

    if (currentZone === 'dungeon') {
      // 副本中，所有任務型 Prompt 都是戰鬥
      return { encounter: true, forced: true };
    }

    // 2. 分析 Prompt 類型
    const promptType = this.analyzePromptType(prompt);

    if (promptType === 'dialog') {
      // 對話型 - 不觸發戰鬥
      return { encounter: false, reason: '對話型請求' };
    }

    // 3. 分析複雜度
    const complexity = this.analyzeComplexity(prompt);

    if (complexity < 8) {
      // 簡單任務 - 不觸發戰鬥
      return {
        encounter: false,
        reason: '任務過於簡單',
        handleIn: 'main_cli'
      };
    }

    // 4. 複雜度 >= 8，在野外觸發戰鬥
    if (currentZone === 'wilderness') {
      return {
        encounter: true,
        complexity,
        zone: this.getCurrentSubZone(),  // forest/mountains/etc
        createWorktree: true
      };
    }

    return { encounter: false };
  }

  /**
   * 生成敵人
   */
  generateEnemy(prompt, complexity, zone) {
    // 基於複雜度和區域生成敵人
    const enemyTypes = this.getEnemyTypesForZone(zone);
    const enemyType = this.selectEnemyType(promptKeywords, enemyTypes);

    return {
      name: this.generateEnemyName(enemyType, complexity),
      level: complexity,
      hp: this.calculateEnemyHP(complexity),
      attack: this.calculateEnemyAttack(complexity),
      defense: this.calculateEnemyDefense(complexity),
      type: enemyType,
      zone
    };
  }
}
```

## 與其他系統的關係

### 整合點

**與 Shop System 整合**:
- 商店位於城鎮的商業街
- 進入商店需要先導航到商業街區域
- 快捷鍵可直接進入（跳過導航動畫）

**與 Worktree-Battle Integration 整合**:
- 公會大廳 = Worktree 管理中心
- 野外遭遇戰自動創建 worktree
- 戰鬥結束後在公會大廳處理合併

**與 Battle System 整合**:
- 野外 = 戰鬥主要發生地
- 城鎮 = 安全區，不觸發戰鬥
- 副本 = 特殊戰鬥場所

**與 Scene System 整合**:
- Scene System 被地圖系統取代
- 探索模式 → 城鎮區域
- 戰鬥模式 → 野外/副本區域

**與 Interactive Events 整合**:
- Plan Mode → 靜止之間
- AskUserQuestion → 可在任何場所觸發（暫停時間）
- 錯誤/警告 → 戰鬥中的反噬傷害
