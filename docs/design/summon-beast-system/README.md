# 召喚獸系統（Summon Beast System）

**依賴**: 戰鬥系統、夥伴系統
**被依賴**: 無

---

## 📖 系統概要

召喚獸系統提供關鍵時刻的強力支援。與長期陪伴的夥伴不同，召喚獸是短暫、爆發型的戰術選擇。通過技能、組合技、MCP 工具或道具召喚，提供立即的強力效果或持續數回合的支援。高 MP 消耗確保玩家需要策略性判斷使用時機。

---

## 🎯 核心功能

### 1. 召喚獸 vs 夥伴

| 特性 | 夥伴 (Companion) | 召喚獸 (Summon Beast) |
|-----|-----------------|---------------------|
| 來源 | Subagent | 技能/道具/組合技/MCP工具 |
| 持續時間 | 戰鬥全程 | 限時（1-3回合）或單次行動 |
| 成長性 | 有經驗值、升級 | 無成長，固定能力 |
| 槽位 | 夥伴槽（最多2個） | 召喚獸槽（最多1個，獨立） |
| MP消耗 | 召喚時消耗 | 每次召喚都消耗 |
| 行動方式 | 參與回合順序 | 立即行動或特定觸發 |
| 用途 | 持續戰鬥支援 | 爆發傷害、緊急支援 |

### 2. 四種召喚類型

**Skill Summons（技能召喚獸）**
- 通過專門的召喚 Skill 觸發
- 高 MP 消耗（60-100 MP）
- 強力即時效果
- 範例：Code Dragon（代碼之龍）- 一次性巨大傷害

**Combo Summons（組合技召喚獸）**
- 連續使用特定技能組合觸發
- 中等 MP 消耗（40-60 MP）
- 獎勵性質，鼓勵技能組合
- 範例：Perfect Trinity（完美三位一體）- 代碼生成+審查+測試

**MCP Tool Summons（MCP 工具召喚獸）**
- 特定 MCP 工具映射為召喚獸
- 工具功能 = 召喚獸能力
- MP 消耗由工具複雜度決定
- 範例：GitHub Guardian（使用 gh CLI）

**Item Summons（道具召喚獸）**
- 使用特殊道具召喚
- 不消耗 MP，消耗道具
- 稀有度決定效果強度
- 範例：Ancient Scroll（遠古卷軸）

### 3. 四種行為類型

**immediate（立即型）**
- 召喚後立即執行一次行動
- 執行完畢後消失
- 適合爆發傷害或緊急治療
- 範例：Code Dragon - 造成 500 傷害後消失

**automatic（自動型）**
- 召喚後每回合自動行動
- 持續固定回合數（1-3 回合）
- 不需玩家干預
- 範例：Code Sentinel - 每回合自動攻擊，持續 3 回合

**passive（被動型）**
- 提供持續性 Buff/Debuff
- 不主動攻擊
- 持續整場戰鬥或固定回合
- 範例：Wisdom Aura - 提升全隊智慧 +30%

**interactive（互動型）**
- 召喚後待命，等待玩家指令
- 可選擇性觸發技能
- 持續回合數內可用
- 範例：Strategy Advisor - 可選擇諮詢策略或執行攻擊

## 🔗 與其他系統的關係

### 依賴關係

- **戰鬥系統** → 召喚獸在戰鬥中提供支援
- **夥伴系統** → 與夥伴互補，不衝突

### 設計互補性

召喚獸與夥伴的設計是互補而非競爭：
- 夥伴 = 穩定持續的支援
- 召喚獸 = 關鍵時刻的爆發

玩家可同時擁有：
- 2 個夥伴（長期）
- 1 個召喚獸（短期）

## 💡 設計要點

### 設計理念

```
召喚獸 = 特殊支援
├─ 關鍵時刻使用
├─ 強力但短暫
└─ 戰術性選擇
```

### 關鍵設計決策

1. **高 MP 消耗**
   - 確保玩家慎重使用
   - 增加資源管理深度
   - 不是每場戰鬥都能用

2. **短暫持續時間**
   - 避免與夥伴系統重疊
   - 強調「關鍵時刻」的概念
   - 鼓勵把握時機

3. **多樣化來源**
   - 技能、組合技、MCP、道具
   - 提供多種獲取方式
   - 增加系統豐富度

## 🚀 快速參考

### 召喚獸範例

**Code Dragon（代碼之龍）** - Skill Summons
- 類型：immediate（立即型）
- 召喚方式：使用 `summon-code-dragon` 技能
- MP 消耗：80
- 效果：造成 500 點傷害，立即消失
- 適用場景：敵人 HP 較低時一擊必殺

**Perfect Trinity（完美三位一體）** - Combo Summons
- 類型：automatic（自動型）
- 觸發條件：連續使用「代碼生成」→「代碼審查」→「測試生成」
- MP 消耗：50
- 效果：3 回合內每回合造成 150 傷害
- 適用場景：鼓勵使用完整開發流程

**GitHub Guardian（GitHub 守護者）** - MCP Tool Summons
- 類型：passive（被動型）
- 觸發條件：使用 GitHub MCP 工具
- MP 消耗：30
- 效果：持續戰鬥，提供「代碼追蹤」Buff（+20% 代碼型傷害）
- 適用場景：使用 GitHub 相關功能時自動召喚

**Ancient Scroll（遠古卷軸）** - Item Summons
- 類型：interactive（互動型）
- 召喚方式：使用道具「遠古卷軸」
- MP 消耗：0（消耗道具）
- 效果：3 回合內可選擇「攻擊」或「治療」
- 適用場景：靈活應對戰況變化

### 召喚流程

```
1. 檢查召喚條件
   ├─ 技能：Skill 可用
   ├─ 組合技：技能序列完成
   ├─ MCP：工具被使用
   └─ 道具：道具可用

2. 檢查資源
   ├─ MP 是否足夠
   ├─ 召喚獸槽是否空閒
   └─ 冷卻時間是否完成

3. 執行召喚
   ├─ 扣除 MP/道具
   ├─ 播放召喚動畫（魔法陣）
   └─ 召喚獸出現

4. 執行行為
   ├─ immediate：立即行動並消失
   ├─ automatic：註冊到回合系統
   ├─ passive：應用 Buff/Debuff
   └─ interactive：等待玩家指令

5. 生命週期管理
   ├─ 回合倒數（automatic/interactive）
   ├─ 持續回合結束後消失
   └─ 清理效果和狀態
```
