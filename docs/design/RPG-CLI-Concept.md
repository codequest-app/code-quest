# RPG風格 AI CLI 網頁介面專案構想

## 專案概述

將 claude-cli 或 gemini-cli 包裝成具有RPG遊戲風格的網頁介面，讓使用者能以更有趣、更具互動性的方式與AI助手對話。

---

## 核心概念

### 遊戲化元素映射

| 遊戲元素 | 對應功能 |
|---------|---------|
| 用戶 | 冒險者/玩家 |
| Claude/Gemini | NPC導師/魔法師 |
| 對話 | 任務/Quest |
| Prompt模板 | 技能/魔法 |
| 對話歷史 | 冒險日誌 |
| API使用量 | HP/MP值 |

---

## 技術架構

### 技術堆疊建議

```
前端層:
├── React/Vue.js (UI框架)
├── PixiJS/Phaser (遊戲引擎) 或純CSS RPG風格
├── WebSocket (實時通訊)
└── LocalStorage/IndexedDB (本地數據存儲)

後端層:
├── Node.js + Express (API服務器)
├── child_process (調用CLI工具)
└── WebSocket Server (實時推送)

CLI整合:
├── claude-cli
└── gemini-cli
```

### 系統架構圖

```
[瀏覽器] <--WebSocket--> [Node.js Server] <--child_process--> [claude-cli/gemini-cli]
    |                           |
    ├── RPG UI                  ├── Session管理
    ├── 動畫/音效               ├── Prompt處理
    └── 本地存儲                └── Stream輸出處理
```

## 功能設計

### 1. 角色系統

#### 職業選擇
- **劍士 (Developer)**: 擅長代碼生成、debug
  - 起始技能：代碼生成、錯誤診斷
  - 特殊能力：代碼重構大師

- **法師 (Writer)**: 擅長文案創作、內容生成
  - 起始技能：文章撰寫、內容潤色
  - 特殊能力：創意湧現

- **遊俠 (Designer)**: 擅長UI/UX建議、設計諮詢
  - 起始技能：設計建議、使用者體驗分析
  - 特殊能力：美學洞察

#### 等級與成長
- **經驗值系統**：
  - 每次對話獲得經驗值
  - 使用token數量 = 經驗值
  - 升級解鎖新技能和功能

- **屬性面板**：
  ```
  Lv.5 冒險者
  ━━━━━━━━━━━━━━━━
  HP: ████████░░  80/100 (API配額)
  MP: ██████░░░░  60/100 (Token餘額)
  EXP: ███░░░░░░░ 300/500
  ━━━━━━━━━━━━━━━━
  力量: 5 (代碼能力)
  智慧: 8 (創意思維)
  幸運: 3 (回應品質)
  ```

### 2. 任務系統

#### 任務類型

**日常任務** (每日重置)
- 🔹 進行3次對話 (獎勵: 50 EXP)
- 🔹 生成代碼片段 (獎勵: 30 EXP + 技能點)
- 🔹 使用特定技能 (獎勵: 道具)

**主線任務** (長期目標)
- 📖 完成首個完整專案 (獎勵: 稱號 + 特殊技能)
- 📖 累計對話達100次 (獎勵: 新職業解鎖)
- 📖 收集所有基礎技能 (獎勵: 進階技能樹)

**成就系統**
- 🏆 首次對話
- 🏆 連續使用7天
- 🏆 單次對話超過1000 tokens
- 🏆 解決複雜問題 (需使用多輪對話)

#### 技能分類

**進階技能** (Lv.10解鎖)

- 🌟 架構設計 (消耗MP: 50)

**終極技能** (Lv.30解鎖)
- 💫 全棧開發指導
- 🎯 AI輔助結對編程

## 實現重點

### 1. CLI整合方案

```javascript
// 後端調用CLI的範例
const { spawn } = require('child_process');

function callClaudeCLI(prompt, stream = true) {
  const claude = spawn('claude', ['-p', prompt, '--stream']);

  claude.stdout.on('data', (data) => {
    // 通過WebSocket推送給前端
    wsConnection.send({
      type: 'ai_response_chunk',
      data: data.toString()
    });
  });

  return claude;
}
```

### 2. 實時通訊

```javascript
// WebSocket處理streaming輸出
io.on('connection', (socket) => {
  socket.on('cast_skill', async ({ skillName, prompt }) => {
    const stream = await callClaudeCLI(prompt);

    // 模擬RPG對話效果
    stream.on('chunk', (text) => {
      socket.emit('npc_speaking', {
        text,
        animation: 'talking'
      });
    });
  });
});
```

## 視覺風格選項與範例

### Option A: 像素風 (Pixel Art) 🎮

**視覺特色：**
- 8-bit/16-bit復古美術風格
- 像素化的UI元素、圖標、字體
- 色彩鮮明，邊界清晰
- 懷舊感強，適合呈現經典RPG氛圍

**代表遊戲參考：**
- **Final Fantasy (IV-VI)**: 經典JRPG像素風格
- **Stardew Valley**: 現代像素風格的成功案例
- **Undertale**: 簡潔但有趣的像素UI
- **Terraria**: 豐富的像素圖標系統

**UI特色元素：**
```
┌─────────────────┐
│ ❤❤❤❤❤ HP: 100  │  ← 像素化心型HP
│ ⚡⚡⚡⚡⚡ MP: 100  │  ← 閃電型MP
│ LV.5  EXP ▓▓▓░░ │  ← 經驗值進度條
└─────────────────┘
```

**實現難度：** ⭐⭐☆☆☆ (相對容易)
- CSS可實現基礎效果
- 或使用 PixiJS 渲染像素圖形

**設計資源：**
- [CraftPix - Free Pixel Art UI for RPG](https://craftpix.net/freebies/free-basic-pixel-art-ui-for-rpg/)
- [Itch.io - Pixel Art UI Assets](https://itch.io/game-assets/tag-pixel-art/tag-user-interface)
- [GameDev Market - Pixel Art RPG UI](https://www.gamedevmarket.net/asset/pixel-art-rpg-ui)

**適合場景：**
- 想要營造復古、輕鬆、有趣的氛圍
- 開發資源有限（像素素材相對容易製作）
- 目標用戶喜歡復古遊戲風格

## 技術挑戰與解決方案

### 挑戰1: CLI調用的權限和環境問題
**解決方案**:
- 使用electron包裝成桌面應用
- 或提供本地server模式，用戶在本機運行

### 挑戰2: Streaming輸出的同步
**解決方案**:
- WebSocket確保實時性
- 前端buffer處理，確保打字機效果流暢

### 挑戰3: API費用管理
**解決方案**:
- 設計HP/MP系統對應實際API配額
- 提供「休息」功能（等待配額恢復）
- 技能冷卻時間限制高頻使用
