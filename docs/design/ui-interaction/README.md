# UI 互動系統（UI Interaction）

**依賴**: 所有系統（呈現層）
**被依賴**: 無

---

## 📖 系統概要

UI 互動系統定義 Code Quest 的完整 UI 規範，確保一致的用戶體驗。涵蓋六大設計原則、完整介面規範、動畫系統、無障礙設計和效能優化。作為所有系統的呈現層，統一視覺風格（Pixel Art）和互動模式。

---

## 🎯 核心功能

### 1. 六大設計原則

**即時回饋（Immediate Feedback）**：
- 所有操作有即時視覺/音效回饋
- HP/MP/EXP 變化顯示動畫過渡
- 技能施放有視覺效果和音效
- 按鈕點擊有按下動畫

**狀態可見（State Visibility）**：
- HP/MP/EXP 始終可見於狀態欄
- 當前等級和資源顯著顯示
- AI 處理狀態清晰指示
- 連線狀態始終顯示

**減少認知負擔（Reduced Cognitive Load）**：
- 預設技能降低使用門檻
- 常用任務透過快捷鍵訪問
- 從技能模板自動生成 Prompt
- 清晰的視覺層級

**漸進式揭露（Progressive Disclosure）**：
- 進階功能初始隱藏
- 按需展開區塊
- 避免overwhelm新用戶
- 漸進式功能介紹

**錯誤容忍（Error Tolerance）**：
- 失敗重試機制
- 對話自動儲存
- 網路重連處理
- 復原/取消操作

**無障礙（Accessibility）**：
- 鍵盤導航支援
- 螢幕閱讀器相容
- 高對比模式
- 彈性字體大小

### 2. 完整介面規範

**狀態欄（頂部，固定）**：
- 玩家等級和名稱
- HP 條（10格進度條）：`❤️ HP: 80/100 ████████░░`
- MP 條（10格進度條）：`⚡ MP: 60/100 ██████░░░░`
- 金幣顯示：`💰 1,250`
- 通知、設定、歷史圖標

**對話區（可捲動，虛擬化）**：
- AI 頭像和名稱
- 狀態指示（思考中/施放中/完成）
- 訊息氣泡與時間戳記
- 代碼區塊語法高亮
- 複製和收藏按鈕
- 平滑捲動行為

**輸入區**：
- 多行文字輸入
- 佔位文字：「告訴魔法師你的需求...」
- 語音輸入按鈕
- 檔案附件按鈕
- 發送按鈕（Enter 快捷鍵）
- 清除對話按鈕

**技能欄（底部或側邊）**：
- 顯示 5-6 個常用技能
- 顯示技能圖標、名稱、MP 消耗
- 指示技能狀態（可用/冷卻/鎖定/MP不足）
- Hover 顯示技能描述
- 點擊觸發技能參數對話框
- 拖放重新排序支援

### 3. 動畫系統

**打字機效果**：
- AI 回應逐字顯示
- 速度：50ms/字元（可調整）
- 每 3 個字元播放打字音效
- Streaming 輸出即時渲染

**HP/MP/EXP 動畫**：
- 進度條平滑填充/清空（300-500ms）
- 浮動數字顯示變化量
- 顏色指示類型（綠色=增加，紅色=減少）
- 每種變化播放對應音效

**技能施放動畫**：
- 技能圖標放大發光（200ms）
- 魔法粒子效果（300ms）
- MP 條減少階梯動畫（300ms）
- 音效播放

**升級動畫**：
- EXP 條填滿至 100%
- 閃光效果（300ms）
- 等級數字縮放動畫（500ms）
- 全螢幕慶祝 Modal
- 顯示獎勵（HP/MP 增加、新技能、金幣）
- 音效：「level_up.wav」

### 4. 無障礙實作

**鍵盤導航**：
- Tab 順序：輸入框 → 發送按鈕 → 技能按鈕 → 頂欄按鈕
- 快捷鍵：
  - Enter：發送訊息
  - Shift+Enter：換行
  - Ctrl+K：清空輸入
  - Esc：關閉 Modal/取消操作
  - Ctrl+H：開啟歷史
  - Ctrl+1~5：使用技能 1-5

**螢幕閱讀器**：
- 所有互動元素有 aria-labels
- 狀態變化透過 aria-live 宣告
- 進度條有文字等價物
- 按鈕描述其行動
- 表單有適當標籤

**高對比模式**：
- 偵測系統偏好 `prefers-contrast: high`
- 邊框寬度增加至 3px
- 使用純黑和純白
- 維持 7:1 對比度
- 避免僅依賴顏色

### 5. 效能優化

**虛擬捲動**：
- 只渲染可見訊息（±5 緩衝）
- 卸載螢幕外內容
- 更新時維持捲動位置
- 使用 @tanstack/react-virtual

**Lazy Loading**：
- 圖片按需載入
- 延遲非關鍵腳本
- 漸進增強

**RAF 動畫**：
- 使用 requestAnimationFrame
- 確保 60 FPS 流暢動畫
- 避免強制同步佈局

## 💡 設計要點

### 視覺風格：Pixel Art

**選定理由**：
- 復古、輕鬆、有趣的氛圍
- 開發資源相對容易製作
- 適合 RPG 主題

**實作要點**：
```css
.pixel-art {
  image-rendering: pixelated;
  image-rendering: -moz-crisp-edges;
  image-rendering: crisp-edges;
}

.pixel-font {
  font-family: 'Press Start 2P', cursive;
}
```

### 效能目標

- 初始載入：< 3 秒
- 技能啟動：< 100ms
- 訊息發送：< 200ms
- 首個 AI 回應塊：< 2 秒
- UI 動畫幀率：60 FPS

## 🚀 快速參考

### 狀態顯示格式

```
❤️ HP: 80/100  ████████░░
⚡ MP: 60/100  ██████░░░░
⭐ EXP: 300/500 ███░░░░░░░
```

**規則**：
- 格式：`current/max` 數字右對齊
- 進度條：正常 10 格，緊湊 4 格
- 填滿：`█`，空白：`░`
- 顏色：HP=#E74C3C, MP=#3498DB, EXP=#F39C12

### 技能卡片狀態

**可用**：
```
┌────┐
│⚔️  │  ← 全彩，可點擊
│Code│
│MP10│
└────┘
```

**冷卻中**：
```
┌────┐
│⚔️  │  ← 灰階，倒數覆蓋層
│Code│
│ 3s │
└────┘
```

**MP 不足**：
```
┌────┐
│⚔️  │  ← 紅色色調，搖晃動畫
│Code│
│MP10│
└────┘
```

### 動畫時序

- 打字機：50ms/字元
- MP 變化：300ms 階梯動畫
- HP 變化：500ms 填充動畫
- 升級閃光：300ms
- 浮動文字：1000ms 淡出
