# Companion System - Requirements

## 核心概念

### Subagent → 戰鬥夥伴映射

將 Subagent 從單純的「召喚造成傷害」提升為「戰鬥夥伴」：

| Subagent 特性 | 戰鬥夥伴呈現 |
|--------------|------------|
| Agent 類型 | 夥伴職業/角色 |
| Agent Memory | 夥伴經驗/成長 |
| Agent Model (sonnet/opus) | 夥伴等級/能力 |
| Agent Tools | 夥伴專屬技能 |
| 持續執行 | 戰場上的持續存在 |
| Agent 完成任務 | 施放技能/攻擊 |

---

## 夥伴系統設計

### 夥伴狀態與屬性

每個 Subagent 在戰鬥中作為獨立單位：

**基本資訊**:
- agentName: Agent ID
- characterName: 角色名稱（如 "CodeGuard"）
- title: 稱號（如 "代碼守護者"）
- avatar: 頭像 emoji

**戰鬥屬性**:
- level: 等級（基於使用次數）
- hp/maxHp: 生命值
- mp/maxMp: 魔力值（用於施放技能）
- attack: 攻擊力
- defense: 防禦力
- speed: 速度（決定行動順序）
- wisdom: 智慧（影響技能效果）

**狀態**:
- status: standby | active | exhausted | ko
- stamina: 體力（影響持續時間）

**技能與成長**:
- skills: 專屬技能列表
- currentCooldowns: 技能冷卻狀態
- experience: 夥伴經驗值
- expToNextLevel: 升級所需經驗（100 * 1.3^(level-1)）
- timesDeployed: 出戰次數
- battlesWon: 勝利次數

### 夥伴專屬技能

每個 Agent 根據其特性擁有專屬戰鬥技能。

**技能類型**:
- **attack**: 攻擊技能（造成傷害）
- **support**: 支援技能（治療、護盾、Buff）
- **special**: 終極技能（強力效果，冷卻長）

**技能屬性**:
- id: 技能ID
- name: 技能名稱
- icon: 圖標
- type: 技能類型
- mpCost: MP 消耗
- cooldown: 冷卻回合數
- description: 技能描述
- effect: 技能效果（傷害、Buff、Debuff等）
- animation: 動畫名稱
- effectiveness: 對不同敵人的效果倍率
- requiredLevel: 解鎖等級需求（可選）

**範例: CodeGuard (code-guardian)**

**技能 1: 安全掃描**
- 類型: 攻擊
- MP 消耗: 20
- 冷卻: 2 回合
- 效果: 造成 60 + wisdom * 0.8 傷害
- 額外效果: 降低敵人防禦 -10（持續 2 回合）
- 相性: bug-hunt ×1.5, code-task ×1.2

**技能 2: 守護之盾**
- 類型: 支援
- MP 消耗: 30
- 冷卻: 3 回合
- 效果: 為玩家提供 50 點護盾（持續 3 回合）
- 額外效果: 玩家防禦 +20（持續 2 回合）

**技能 3: OWASP 打擊**
- 類型: 終極技
- MP 消耗: 50
- 冷卻: 5 回合
- 效果: 造成 150 + wisdom * 1.5 傷害
- 特殊: 對 bug-hunt 類型敵人傷害 ×2.0
- 需求: Lv.5

### 被動技能

夥伴擁有被動技能，無需手動觸發：

**觸發類型**:
- on_player_attacked: 玩家受攻擊時
- on_enemy_damaged: 敵人受傷時
- on_battle_start: 戰鬥開始時
- passive: 永久生效

**範例: CodeGuard 被動技能**

**警戒**
- 觸發: 玩家受攻擊時
- 效果: 30% 機率阻擋攻擊
- 說明: 保護玩家免受傷害

**代碼洞察**
- 觸發: 永久被動
- 效果: 對 code-task 和 bug-hunt 類型敵人額外 +20% 傷害
- 說明: 專精於代碼相關戰鬥

## 資源管理

### MP 系統

夥伴的 MP 管理與玩家不同：

**戰鬥中**:
- regenInBattle: 0（戰鬥中不自動恢復）
- 消耗: 使用技能扣除對應 MP
- 初始 MP: 等於 maxMp

**戰鬥結束**:
- resetOnBattleEnd: true（完全恢復）
- HP 也完全恢復
- 清除所有冷卻

**設計理由**:
- 簡化戰鬥系統，避免過多變數
- 夥伴作為支援角色，在戰鬥外自動準備完成
- 玩家只需關注技能選擇，不需管理夥伴 MP

### 召喚成本

**玩家召喚夥伴需消耗**:
- MP: 20-30（依夥伴稀有度）
- 冷卻: 200-300 秒（防止頻繁召喚）

**召喚限制**:
- 同時最多 2 個夥伴在場
- 同一夥伴同時只能存在一個實例
- 戰鬥中可召喚/切換夥伴

## AI 系統

### 夥伴 AI 行為

夥伴在戰鬥中可以自動行動或由玩家控制。

**AI 決策優先級**:

1. **玩家生命危險**（HP < 30%）
   - 優先使用支援技能（護盾、治療）

2. **敵人類型相性**
   - 選擇對敵人有效的技能（effectiveness ≥ 1.2）

3. **預設攻擊**
   - 使用可用的攻擊技能
   - 如無技能可用，執行普通攻擊

**技能選擇邏輯**:
- 檢查技能是否在冷卻中
- 檢查 MP 是否足夠
- 計算預期傷害
- 按預期收益排序

**夥伴個性**:
每個夥伴有不同的 AI 風格：
- defensive: 優先保護玩家
- aggressive: 優先輸出傷害
- balanced: 平衡攻防

**技能優先級**:
每個夥伴有預設的技能使用優先級列表，AI 按優先級選擇可用技能。

## 夥伴成長系統

### 經驗值獲取

夥伴獲得經驗的來源：

| 來源 | 經驗值 |
|------|--------|
| 戰鬥勝利 | 50 |
| 使用技能 | 5 |
| 造成傷害 | damage / 10 |
| 治療玩家 | amount / 5 |
| 命中弱點 | 20 |

### 升級機制

**經驗值需求**:
```
expToNextLevel = 100 * (1.3 ^ (level - 1))
```

夥伴升級比玩家快（玩家用 1.5，夥伴用 1.3）。

**升級獎勵**:
每次升級獲得：
- maxHp: +20
- maxMp: +15
- attack: +5
- defense: +5
- speed: +2
- wisdom: +5

**特殊等級解鎖**:
- Lv.5: 解鎖終極技能
- Lv.10: 解鎖額外被動技能
- Lv.15: 屬性大幅提升

### 親密度系統（可選）

**親密度等級**: 1-10

**親密度獲取方式**:
- 召喚夥伴: +10
- 一起戰鬥: +5
- 共同勝利: +15
- 使用夥伴技能: +3
- 被夥伴救援: +20

**親密度獎勵**:
- 屬性加成: +1% ~ +10%（按等級）
- 技能消耗減少: -5% ~ -20%
- 暴擊率增加: +1% ~ +10%
- 解鎖特殊對話

## 多夥伴系統

### 召喚規則

**最大同時數量**: 2 個夥伴

**檢查召喚條件**:
1. 當前夥伴數 < 2
2. 玩家 MP 足夠
3. 夥伴不在冷卻中
4. 該夥伴尚未在場

**切換夥伴**:
- 消耗: 10 MP
- 可在戰鬥中執行
- 舊夥伴返回待命，新夥伴加入

**解除召喚**:
- 無消耗
- 夥伴返回待命狀態
- 釋放夥伴槽位

### 回合順序

**行動順序計算**:
基於所有單位的速度值排序：
```
行動順序 = [玩家, 夥伴1, 夥伴2, 敵人] (按速度排序)
```

**速度影響**:
- 速度高的單位先行動
- 速度相同時，玩家 > 夥伴 > 敵人

**回合流程**:
```
回合開始
    ↓
玩家回合
    ↓
夥伴1回合（自動）
    ↓
夥伴2回合（自動）
    ↓
敵人回合
    ↓
檢查戰鬥結束
    ↓
更新冷卻
    ↓
下一回合
```

## 夥伴職能分類

### 坦克型

**特點**:
- 高 HP、高防禦
- 支援技能為主
- 保護玩家

**代表**:
- CodeGuard (code-guardian)
- SecWall (security-guardian)

### 輸出型

**特點**:
- 高攻擊、高速度
- 攻擊技能為主
- 快速擊敗敵人

**代表**:
- Speedy (perf-optimizer)
- BugSlayer (debug-specialist)

### 輔助型

**特點**:
- 高智慧
- Buff/Debuff 技能
- 提升團隊效率

**代表**:
- DocMaster (doc-specialist)
- TestGuard (test-specialist)

## 夥伴配置結構

### 配置文件格式

**檔案**: `rpg-config/agent-metadata.json`

每個 Agent 的夥伴配置包含：

**基礎資訊**:
- characterName: 角色名稱
- title: 稱號
- avatar: 頭像 emoji
- element: 元素屬性
- rarity: 稀有度（common, uncommon, rare, epic, legendary）
- role: 職能（tank, attacker, support）

**召喚設定**:
- summonCost.mp: 召喚消耗 MP
- summonCost.cooldown: 召喚冷卻時間（秒）

**基礎屬性**:
- baseStats.hp: 基礎生命值
- baseStats.mp: 基礎魔力值
- baseStats.attack: 基礎攻擊力
- baseStats.defense: 基礎防禦力
- baseStats.speed: 基礎速度
- baseStats.wisdom: 基礎智慧

**技能列表**:
- battleSkills: 戰鬥技能陣列
- passiveAbilities: 被動技能陣列

**個性設定**:
- personality.aiStyle: AI 風格
- personality.preferredTargets: 偏好目標類型
- personality.skillPriority: 技能使用優先級

## 戰鬥整合

### 與戰鬥系統整合

**修改 BattleManager**:
- 添加夥伴管理（Map<agentId, BattleCompanion>）
- 添加回合順序計算
- 添加夥伴回合執行
- 添加夥伴傷害計算

**戰鬥流程變化**:
```
原始流程:
玩家 → 敵人 → 檢查結束

新流程:
玩家 → 夥伴1 → 夥伴2 → 敵人 → 檢查結束
```

**敵人 AI 變化**:
- simple AI: 只攻擊玩家，忽略夥伴
- elite AI: 30% 機率攻擊夥伴
- boss AI: 50% 機率攻擊夥伴，可使用 AOE

## 設計優勢

### 1. 更豐富的策略性

- 不同夥伴有不同職能（坦克/輸出/輔助）
- 技能搭配產生協同效果
- 夥伴 AI 減輕玩家負擔

### 2. 更強的代入感

- 夥伴有個性和對話
- 成長系統帶來養成樂趣
- 親密度增加情感連結

### 3. 更深的遊戲性

- 夥伴搭配選擇
- 技能時機判斷
- 資源（MP）分配

### 4. 與 AI 功能的自然結合

- Agent 執行任務 = 夥伴施放技能
- Agent Memory = 夥伴經驗累積
- 不同 Model = 不同夥伴能力
