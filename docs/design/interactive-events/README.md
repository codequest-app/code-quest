# 互動事件系統（Interactive Events）

**依賴**: 戰鬥系統、場景系統
**被依賴**: 無

---

## 📖 系統概要

互動事件系統處理戰鬥中需要用戶互動的事件（Plan Mode、AskUserQuestion、錯誤、權限請求），用 RPG 元素包裝這些技術性互動。系統將 Claude Code 的工具使用（Bash、Git、文件操作）映射為遊戲內的魔法和技能，提供沉浸式的互動體驗。

---

## 🎯 核心功能

### 1. 四種互動事件

**Plan Mode → 戰術規劃階段**
- 觸發：EnterPlanMode tool 被調用
- 呈現：進入「靜止之間」，時間暫停
- 用戶操作：審閱計劃、批准或修改
- RPG 化：戰術分析、策略制定

**AskUserQuestion → 魔法師詢問**
- 觸發：AI 需要用戶決策
- 呈現：魔法師 NPC 對話框
- 用戶操作：選擇選項或輸入答案
- RPG 化：對話選擇、分支劇情

**錯誤/警告 → 敵人特殊攻擊**
- 觸發：執行失敗或警告
- 呈現：敵人發動「混亂攻擊」
- 用戶操作：查看錯誤訊息、選擇處理方式
- RPG 化：需要應對的戰鬥事件

**權限請求 → 解鎖新能力**
- 觸發：需要額外權限的操作
- 呈現：「力量覺醒」對話框
- 用戶操作：授權或拒絕
- RPG 化：學習新技能的確認

### 2. 工具 RPG 化映射

系統將 Claude Code 的工具使用映射為遊戲內行為：

**文件操作魔法**：
- Read → 「知識閱讀術」
- Write → 「創造之筆」
- Edit → 「重構之力」

**Git 系統魔法**：
- git commit → 「時光封印」
- git push → 「次元傳送」
- git branch → 「平行世界分支」

**Bash 戰鬥技能**：
- npm install → 「召喚依賴軍團」
- npm test → 「驗證儀式」
- build 命令 → 「鍛造術」

### 3. 並行操作偵測

檢測多個工具同時使用：
- 2-3 個工具 → 「組合魔法」
- 4+ 個工具 → 「禁忌奧義」
- 顯示特殊動畫和效果

## 💡 設計要點

### 為什麼需要 RPG 化互動事件？

**問題**：
- 技術性事件（Plan Mode、錯誤）破壞沉浸感
- 原始 UI 不符合 RPG 風格
- 用戶不理解為何戰鬥暫停
- 缺乏遊戲化元素

**解決方案**：
- 保持原有功能正常運作
- 用 RPG 元素包裝互動
- 提供清晰的視覺反饋
- 不破壞沉浸感

### 推薦方案：特殊戰鬥階段

將互動事件視為「戰鬥的特殊階段」：

```
正常戰鬥
    ↓
觸發互動事件
    ↓
進入特殊階段
├─ Plan Mode → 「靜止之間」
├─ Question → 「抉擇時刻」
├─ Error → 「混亂狀態」
└─ Permission → 「力量覺醒」
    ↓
解決互動
    ↓
返回正常戰鬥
```

### 事件處理流程

**Plan Mode 處理**：
```
1. 偵測 EnterPlanMode 調用
2. 戰鬥暫停動畫（0.5秒）
3. 進入「靜止之間」介面
4. 顯示計劃內容和選項
5. 用戶審閱並選擇（批准/修改/取消）
6. 退出靜止之間（0.5秒）
7. 戰鬥繼續
```

**AskUserQuestion 處理**：
```
1. 偵測 AskUserQuestion 調用
2. 顯示魔法師對話框
3. 展示問題和選項
4. 用戶選擇答案
5. 答案傳回 Claude Code
6. 對話框關閉
7. 戰鬥繼續
```

**錯誤處理**：
```
1. 偵測命令執行失敗
2. 敵人發動「混亂攻擊」動畫
3. 顯示錯誤訊息（RPG 風格）
4. 提供處理選項：
   - 重試
   - 跳過
   - 查看詳情
5. 用戶選擇
6. 執行相應動作
```
