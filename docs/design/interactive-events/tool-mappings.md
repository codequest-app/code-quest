## 核心原則

### RPG 化設計原則

1. **每個工具都有對應的魔法**
   - 無論是 Bash、Read、Write 還是其他工具
   - 每個工具都有獨特的圖標和名稱

2. **MP 消耗與複雜度成正比**
   - 簡單查詢: 2-3 MP
   - 修改操作: 5-8 MP
   - 重型操作: 10-15 MP
   - 特殊操作: 15-20+ MP

3. **戰鬥中的效果多樣化**
   - 造成傷害（進度推進）
   - 獲得 Buff（加速/增強）
   - 發現弱點（額外傷害）
   - 治療/恢復（修復錯誤）

4. **即時反饋**
   - 所有操作都有視覺反饋
   - 戰鬥日誌記錄
   - 動畫效果

5. **失敗有代價**
   - 錯誤造成反噬傷害
   - 鼓勵謹慎操作

### 3. 施法失敗與反噬

**失敗類型**:

| 失敗類型 | 反噬倍率 | 範例 |
|---------|---------|------|
| 語法錯誤 | 1.0x | Syntax Error |
| 邏輯錯誤 | 0.8x | Logic Error |
| 權限錯誤 | 1.2x | Permission Denied |
| 網絡錯誤 | 0.6x | Network Timeout |
| 系統錯誤 | 1.5x | Out of Memory |

**反噬計算**:
```javascript
反噬傷害 = 基礎傷害 × 嚴重度 × 連續失敗倍率

基礎傷害 = 工具 MP 消耗 × 5
嚴重度 = 1.0 - 1.5
連續失敗倍率 = 1.0, 1.5, 2.0, 2.5...
```

**施法失敗範例**:
```
✏️ 改寫術！

修改文件: src/auth/login.ts

⚠️ 施法失敗！

錯誤: Syntax Error at line 42
Unexpected token ';'

💔 技能反噬！
受到 45 點反噬傷害

HP: 80/100 (-45)
MP: 74/100 (-6, MP 仍然消耗)

[🔄 重試] [🛡️ 防禦] [❌ 取消]
```

## 總結

### 核心設計價值

1. ✅ **完整映射**: 所有 Claude Code 工具都有對應的 RPG 魔法
2. ✅ **平衡設計**: MP 消耗與操作複雜度成正比
3. ✅ **豐富效果**: 傷害、治療、Buff、Debuff 多樣化
4. ✅ **即時反饋**: 所有操作都有清晰的視覺反饋
5. ✅ **失敗代價**: 錯誤造成反噬，鼓勵謹慎

### 實作優先級

**Phase 2.5** (必須):
- Bash 基礎分類（git, npm）
- 文件操作（Read, Write, Edit）
- MP 消耗計算
- 基礎戰鬥日誌

**Phase 3** (進階):
- Bash 完整分類（10+ 種）
- 並行操作（多重施法）
- 流式輸出
- 特殊效果

**Phase 4** (優化):
- 網絡操作
- 背景任務
- 自定義映射
- 更多視覺效果
