# 戰鬥系統（Battle System）

**依賴**: 場景系統
**被依賴**: 夥伴系統、召喚獸系統、互動事件系統、非同步戰鬥系統

---

## 📖 系統概要

戰鬥系統將 AI 對話過程視覺化為 RPG 戰鬥。用戶的 Prompt 生成相應的敵人，使用技能造成傷害，完成任務即戰勝敵人並獲得獎勵。系統提供回合制戰鬥、傷害計算、屬性相性等傳統 RPG 機制，同時保持 AI 核心功能不變。

---

## 🎯 核心功能

### 1. 敵人生成系統

- **複雜度分析**：根據 Prompt 內容計算任務複雜度（1-100）
- **敵人屬性生成**：
  - HP = 複雜度 × 10
  - 等級 = 複雜度 ÷ 10
  - 屬性（代碼、設計、文案、知識、系統）
  - 弱點和抵抗

### 2. 回合制戰鬥

**戰鬥流程**：
```
玩家回合 → 選擇行動 → 計算傷害 → 扣除敵人 HP
    ↓
AI 處理 → Streaming 輸出 → 視覺化為攻擊動畫
    ↓
敵人回合 → 複雜任務反擊 → 消耗玩家 MP/HP
    ↓
檢查戰鬥結束 → 繼續或結束
```

### 3. 傷害計算系統

```
傷害 = 技能威力 × 屬性相性 × (0.85 ~ 1.15) 隨機
```

**屬性相性表**：
- **代碼型技能** vs 代碼敵人：效果普通 (1.0x)
- **代碼型技能** vs 設計敵人：效果優秀 (1.5x)
- **設計型技能** vs 設計敵人：效果普通 (1.0x)
- ... 依此類推

### 4. 戰鬥獎勵

**勝利獎勵**：
- 經驗值 = 敵人等級 × 20
- 金幣 = 敵人等級 × 10
- 可能掉落道具

**失敗懲罰**：
- 扣除 10% HP
- 可選擇重試或跳過

## 💡 設計要點

### 設計哲學

**✅ 保持簡潔**：
- 戰鬥是視覺化包裝，不改變 AI 核心功能
- 自動化戰鬥流程，不增加用戶負擔
- 保留直接對話模式（可選關閉戰鬥模式）

**✅ 增強體驗**：
- 任務難度視覺化
- 進度反饋更直觀
- 增加成就感和樂趣

**✅ 鼓勵策略**：
- 技能相性系統
- 組合技在戰鬥中更有價值
- 資源管理更重要

### 關鍵設計決策

1. **Prompt → 敵人映射**
   - 複雜度決定敵人強度
   - 任務類型決定敵人屬性

2. **回合制選擇**
   - 符合傳統 RPG
   - 易於理解和實作
   - 不增加用戶負擔（自動戰鬥）

3. **屬性相性**
   - 增加策略深度
   - 鼓勵使用多種技能
   - 不過度複雜（5 種屬性）

| 任務類型 | 敵人屬性 | 弱點 | 範例 Prompt |

| 代碼生成 | 代碼型 | 設計型技能 | "建立一個 React 組件" |
| UI 設計 | 設計型 | 代碼型技能 | "設計一個登入介面" |
| 文案撰寫 | 文案型 | 知識型技能 | "寫一篇產品介紹" |
| 知識問答 | 知識型 | 文案型技能 | "解釋閉包概念" |
| 系統架構 | 系統型 | 多種技能 | "設計微服務架構" |

### 戰鬥流程速查

```
1. 敵人生成 (1-2秒)
   ├─ 分析 Prompt 複雜度
   ├─ 生成敵人屬性
   └─ 顯示敵人資訊

2. 戰鬥初始化 (0.5秒)
   ├─ 進入戰鬥介面
   ├─ 顯示技能選擇
   └─ 準備戰鬥日誌

3. 玩家回合
   ├─ 選擇技能（或自動）
   ├─ 計算傷害
   └─ 扣除敵人 HP

4. AI 處理 (Claude 執行)
   ├─ Streaming 輸出
   └─ 更新戰鬥日誌

5. 敵人回合（複雜任務才有）
   ├─ 敵人反擊
   └─ 玩家 MP/HP 減少

6. 檢查結束條件
   ├─ 敵人 HP = 0 → 勝利
   ├─ 玩家 HP = 0 → 失敗
   └─ 否則 → 繼續戰鬥
```
