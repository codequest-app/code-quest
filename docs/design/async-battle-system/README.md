# 非同步戰鬥系統（Async Battle System）

**依賴**: 戰鬥系統、Worktree 系統
**被依賴**: 無

---

## 📖 系統概要

非同步戰鬥系統解決傳統 CLI 阻塞問題。複雜任務在後台執行，用戶可繼續對話或開啟新戰鬥，實現真正的「多線作戰」。每個並發戰鬥自動創建獨立 Worktree，避免代碼修改衝突，戰鬥結束後自動合併和清理。

---

## 🎯 核心功能

### 1. 雙軌道系統

```
用戶輸入
    ↓
SmartRouter（複雜度分析）
    ↓
  ┌─────┴─────┐
  ↓           ↓
對話軌道    戰鬥軌道
(即時)      (異步)
  ↓           ↓
立即回應    ┌─簡單任務（主線）
            └─複雜任務（後台 + 自動 Worktree）
```

### 2. 智能路由（SmartRouter）

**複雜度分析算法**：
```javascript
分析 Prompt：
├─ 文件數量（+10/文件）
├─ 代碼行數（+0.1/行）
├─ 關鍵詞：「重構」「優化」「架構」（+20）
├─ 跨目錄操作（+15）
└─ 複雜度 = Σ 加權分數

路由決策：
├─ 複雜度 < 30  → 對話軌道（即時）
├─ 30 ≤ 複雜度 < 60 → 主線戰鬥（同步）
└─ 複雜度 ≥ 60 → 後台戰鬥（異步 + Worktree）
```

### 3. 並發控制

**限制規則**：
- 最多 3 個並發戰鬥
- 超出限制則排隊等待
- 主線戰鬥優先級最高

**資源管理**：
- 每個後台戰鬥獨立 MP 池
- 主線戰鬥共享玩家 MP
- 防止資源耗盡

### 4. 自動 Worktree 管理

**生命週期**：
```
複雜任務觸發
    ↓
自動創建 Worktree
    ├─ 分支名：async-battle-{battleId}
    ├─ 路徑：.worktrees/battle-{battleId}
    └─ 元數據：記錄戰鬥信息
    ↓
戰鬥進行（代碼隔離）
    ↓
戰鬥結束
    ├─ 自動合併到主分支
    ├─ 清理 Worktree
    └─ 更新玩家狀態
```

### 5. 三面板 UI

**UI 分配**：
- 左側（20%）：主對話區 - 快速對話
- 中央（50%）：主戰鬥區 - 當前焦點戰鬥
- 右側（30%）：後台戰鬥列表 - 並發戰鬥狀態

**戰鬥狀態**：
- queued：排隊中
- in_progress：進行中
- completing：完成中（合併 worktree）
- completed：已完成
- failed：失敗

## 🔗 與其他系統的關係

### 依賴關係

- **戰鬥系統** → 後台戰鬥使用相同戰鬥邏輯
- **Worktree 系統** → 自動創建和管理 Worktree

### 設計原則

**用戶永不阻塞**：
- 任何輸入都立即響應
- 複雜任務在後台執行
- 用戶可繼續對話或新任務

**代碼隔離**：
- 每個並發戰鬥獨立 Worktree
- 避免代碼修改互相干擾
- 自動合併和清理

## 💡 設計要點

### 問題陳述

**傳統 CLI 阻塞問題**：
- 用戶輸入命令後必須等待執行完成
- 無法同時進行多個任務
- 長時間運行的任務鎖住整個界面
- 用戶體驗差，無法進行其他操作

### 解決方案亮點

1. **智能路由** - 自動判斷任務複雜度
2. **並發執行** - 最多 3 個並發戰鬥
3. **代碼隔離** - 自動 Worktree 管理
4. **狀態可見** - 實時進度更新

## 🚀 快速參考

### 複雜度計算

| 因素 | 權重 | 範例 |
|-----|------|------|
| 文件數量 | +10/文件 | 5 個文件 = +50 |
| 代碼行數 | +0.1/行 | 200 行 = +20 |
| 關鍵詞 | +20 | 「重構」「架構」 |
| 跨目錄操作 | +15 | 修改多個目錄 |
| 外部依賴 | +10 | npm install |

**路由閾值**：
- < 30：對話軌道（即時回應）
- 30-59：主線戰鬥（同步執行）
- ≥ 60：後台戰鬥（異步 + Worktree）

### 並發戰鬥流程

```
1. 用戶輸入複雜任務
   ↓
2. SmartRouter 分析（複雜度 ≥ 60）
   ↓
3. 檢查並發槽位
   ├─ 有空槽 → 立即創建後台戰鬥
   └─ 已滿 → 加入排隊
   ↓
4. 自動創建 Worktree
   ├─ 分支：async-battle-{uuid}
   ├─ 路徑：.worktrees/battle-{uuid}
   └─ 隔離代碼環境
   ↓
5. 戰鬥在後台執行
   ├─ 實時狀態更新
   ├─ 顯示在右側面板
   └─ 用戶可繼續其他操作
   ↓
6. 戰鬥完成
   ├─ 自動合併到主分支
   ├─ 清理 Worktree
   ├─ 顯示完成通知
   └─ 獎勵加入玩家狀態
```
