# Async Battle System - UI Design

## UI 設計原則

### 1. 雙軌可見
- 對話軌道（左側）：主要交互區
- 戰鬥軌道（右側）：並發戰鬥狀態

### 2. 進度透明
- 實時進度條
- 清晰的狀態標識
- 完成時的視覺反饋

### 3. 最小干擾
- 後台戰鬥不阻塞對話
- 通知溫和不打斷
- 狀態更新不搶焦點

### 4. 快速切換
- 一鍵查看戰鬥詳情
- 方便切換多個戰鬥
- 支持快速操作（暫停、取消）

### 面板劃分

**左側：主對話區（60% 寬度）**
- Dialog Track：即時對話
- Main Sync：同步任務的戰鬥過程
- 用戶輸入框
- 歷史對話記錄

**右上：戰鬥列表（40% 寬度，上半部分）**
- 顯示所有進行中的異步戰鬥
- 每個戰鬥的簡要信息
- 快速狀態指示
- 點擊切換到詳情

**右下：戰鬥詳情（40% 寬度，下半部分）**
- 當前選中戰鬥的完整信息
- 敵人狀態
- 進度詳情
- 最近動作
- 操作按鈕

## 路徑特定 UI

### Dialog Track UI

**即時對話模式**：
```
> 你: 現在是什麼模式？

🤖 Claude:
當前在探索模式。你可以：
• 使用技能進行開發
• 前往商店購買物品
• 查看地圖探索區域
• 開啟新的戰鬥任務

> _
```

**特點**：
- 無戰鬥UI
- 即時響應（< 1 秒）
- 簡潔的對話格式
- 不影響右側戰鬥面板

---

### Main Sync UI

**同步戰鬥模式**：
```
> 你: 修復 login.ts 的類型錯誤

🤖 Claude:
⚔️ 戰鬥開始！

┌──────────────────────────────────┐

[編輯 login.ts...]
造成 20 點傷害！
Bug怪物 HP: 15/50
```

**特點**：
- 在主對話區顯示完整戰鬥過程

- 戰鬥結束後立即響應

## WebSocket 事件 UI 響應

### 事件類型與 UI 更新

| WebSocket 事件 | UI 響應 | 更新內容 |
|---------------|---------|---------|
| `battle:created` | 新增戰鬥卡片 | 戰鬥列表 +1 |
| `battle:started` | 狀態更新為運行中 | 紅色圖標，進度條 0% |
| `battle:progress` | 進度條動畫 | 更新百分比和 HP |
| `battle:tool_used` | 新增動作記錄 | 詳情面板滾動 |
| `battle:paused` | 狀態更新為暫停 | 黃色圖標 |
| `battle:resumed` | 狀態恢復運行 | 紅色圖標 |
| `battle:completed` | 完成通知 + 移除卡片 | 彈出通知，綠色勾選 |
| `battle:failed` | 失敗通知 | 彈出錯誤，灰色 X |
| `queue:added` | 新增排隊卡片 | 藍色排隊圖標 |
| `queue:started` | 從排隊轉運行 | 藍色 → 紅色 |

### 實時更新示例

**進度更新**：
```javascript
// WebSocket 收到 battle:progress 事件
{
  battleId: "battle_1",
  progress: 0.65,
  enemyHP: 175,
  enemyMaxHP: 500,
  timeElapsed: 383000 // 6m 23s
}

// UI 更新
- 進度條: 65% (帶動畫)
- HP: 175/500
- 時間: 6m 23s
```

**工具使用更新**：
```javascript
// WebSocket 收到 battle:tool_used 事件
{
  battleId: "battle_1",
  tool: "Edit",
  file: "auth.ts",
  damage: 25,
  timestamp: "15:32:15"
}

// UI 新增動作記錄
[15:32:15] ✏️ 編輯 auth.ts (傷害: 25)
```

## 使用場景流程

### 場景 1: 簡單對話

```
用戶: "當前模式是什麼？"
  ↓
[Dialog Track - 主對話區]
  ↓
系統立即響應: "探索模式"
  ↓
右側面板無變化
```

### 場景 2: 快速任務

```
用戶: "修復 bug.ts 的語法錯誤"
  ↓
[Main Sync - 主對話區]
  ↓
顯示戰鬥過程（同步）
讀取 → 編輯 → 測試
  ↓
戰鬥結束，顯示結果
  ↓
右側面板無變化
```

### 場景 3: 複雜任務

```
用戶: "重構整個認證系統"
  ↓
[Battle Async - 後台]
  ↓
主對話區: "戰鬥 #1 已開始"
  ↓
右側面板: 新增戰鬥卡片
  ↓
用戶可以繼續輸入
  ↓
[並行操作]
  - 左側: 繼續對話
  - 右側: 戰鬥進度更新
  ↓
戰鬥完成: 彈出通知
```

### 場景 4: 多任務並發

```
用戶啟動 3 個複雜任務
  ↓
戰鬥 #1, #2, #3 同時運行
  ↓
右側顯示 3 個戰鬥卡片
  ↓
用戶點擊卡片查看詳情
  ↓
右下面板顯示選中戰鬥
  ↓
用戶啟動第 4 個任務
  ↓
戰鬥 #4 進入排隊
  ↓
戰鬥 #1 完成
  ↓
戰鬥 #4 自動開始
```
