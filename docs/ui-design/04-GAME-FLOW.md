## 核心遊戲迴圈

Code Quest 的遊戲迴圈模擬傳統 RPG 的「探索→遭遇→戰鬥→獎勵→回城」循環：

```
    ┌──────────────────────────────────────────────────┐
    │                                                  │
    ▼                                                  │
┌────────┐    任務型 Prompt    ┌────────┐             │
│  探索   │ ───────────────→ │  遭遇   │             │
│  模式   │                   │  判定   │             │
└────┬───┘                   └────┬───┘             │
     │                             │                   │
     │ 對話型 Prompt               │ 複雜度 ≥ 8       │
     │ （留在探索）                │                   │
     │                             ▼                   │
     │                       ┌────────┐              │
     │                       │  戰鬥   │              │
     │                       │  模式   │              │
     │                       └────┬───┘              │
     │                             │                   │
     │              ┌──────────────┼───────────┐      │
     │              │              │            │      │
     │              ▼              ▼            ▼      │
     │         ┌────────┐   ┌────────┐   ┌────────┐  │
     │         │  勝利   │   │  失敗   │   │  逃跑   │  │
     │         └────┬───┘   └────┬───┘   └────┬───┘  │
     │              │            │             │      │
     │              ▼            │             │      │
```

## 場景切換系統

### 雙場景模型

遊戲在任何時刻只會處於兩種場景之一：

| | 探索模式 | 戰鬥模式 |
|---|---------|---------|
| **目的** | 管理、休息、對話 | 完成任務、獲得獎勵 |
| **觸發** | 應用啟動、戰鬥結束 | 任務型 Prompt + 複雜度 ≥ 8 |
| **HP 恢復** | +2/秒 | 0/秒 |
| **MP 恢復** | +1/秒 | +0.1/秒 |
| **可用操作** | 菜單、對話、商店、Worktree | 技能、夥伴、召喚獸、逃跑 |

### Prompt 分析邏輯

每當用戶輸入 Prompt，系統按以下順序分析：

```
用戶輸入 Prompt
    │
    ▼
┌──────────────────┐
│ 1. 檢查當前區域   │
│    城鎮？→ 不戰鬥 │
│    副本？→ 強制戰鬥│
└────────┬─────────┘
         │ 野外
         ▼
┌──────────────────┐
│ 2. 分析 Prompt    │
│    類型            │
└────────┬─────────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
 對話型     任務型
（不戰鬥）     │
              ▼
     ┌──────────────┐
     │ 3. 分析複雜度  │
     └────────┬─────┘
              │
         ┌────┴────┐
         │         │
         ▼         ▼
      < 8        ≥ 8
  （主CLI處理）  （觸發戰鬥）
```

**任務型 Prompt 關鍵字**（觸發戰鬥）：
- 動作詞：修復、創建、實作、重構、優化、測試、部署、配置…
- 代碼詞：bug、功能、api、組件、函數、類別、模組…
- 英文：fix, create, implement, refactor, optimize, test, deploy…

**對話型 Prompt 關鍵字**（不觸發戰鬥）：
- 問題詞：什麼、為什麼、怎麼、如何、是否…
- 解釋詞：解釋、說明、介紹、比較、區別…
- 英文：what, why, how, explain, describe, compare…

### 複雜度評分因素

| 因素 | 影響 |
|------|------|
| Prompt 長度 | 字數越多 → 複雜度越高 |
| 動作關鍵字數量 | 多個動作詞 → 複雜度越高 |
| 多步驟指令 | 包含「並且」「然後」→ 複雜度越高 |
| 技術深度 | 涉及高級技術詞彙 → 複雜度越高 |

**範例**：
- 「修復按鈕 Bug」→ 複雜度 3，不觸發戰鬥
- 「實作用戶登入功能」→ 複雜度 10，觸發戰鬥
- 「重構整個認證系統並確保向後兼容」→ 複雜度 15，觸發強敵

## 完整戰鬥流程

### 1. 遭遇觸發

```
用戶在野外輸入任務型 Prompt
    ↓
系統分析：區域 = 野外 ✓、類型 = 任務型 ✓、複雜度 ≥ 8 ✓
    ↓
觸發遭遇戰！
    ↓
根據複雜度 + 區域生成敵人
  • 名稱（基於 Prompt 主題）
  • 等級 = 複雜度值
  • HP = 50 × 複雜度
  • 屬性和弱點
    ↓
自動創建 Worktree（戰鬥分支）
    ↓
切換到戰鬥模式（畫面震動 + 敵人出現動畫）
```

### 2. 回合制戰鬥

每個回合的行動順序：

```
┌─────────────┐
│ 玩家回合     │  Claude 自動行動（使用工具 = 施放魔法）
│             │  用戶可手動觸發技能
│             │  用戶可召喚/切換夥伴
└──────┬──────┘
       ▼
┌─────────────┐
│ 夥伴回合     │  夥伴 AI 自動選擇技能
│ （如有夥伴） │  獨立行動，支援玩家
└──────┬──────┘
       ▼
┌─────────────┐
│ 敵人回合     │  敵人自動反擊
│             │  可能發動特殊技能
│             │  可能觸發互動事件（AskUserQuestion）
└──────┬──────┘
       ▼
    下一回合...
```

### 3. 戰鬥中的特殊事件

戰鬥進行中可能觸發以下事件（時間暫停處理）：

| 事件 | RPG 呈現 | 處理方式 |
|------|---------|---------|
| Plan Mode 觸發 | 戰術規劃階段 | 暫停戰鬥，傳送到靜止之間 |
| AskUserQuestion | 魔法師詢問 | 彈窗顯示，用戶回答後繼續 |
| 錯誤/警告 | 敵人特殊攻擊 | 顯示反噬傷害，提示處理 |
| 權限請求 | 解鎖新能力確認 | 彈窗確認，授權後繼續 |

→ 詳見 [systems/07-interactive-events.md](systems/07-interactive-events.md)

### 4. 戰鬥結束

**勝利**（敵人 HP = 0）：
```
敵人消失動畫 → 勝利文字 → 獎勵結算（EXP + Gold）
    → 檢查升級（如果 EXP 足夠則升級動畫）
    → 提示合併 Worktree（合併/保留/放棄）
    → 返回探索模式
```

**逃跑**：
```
消耗 30% 當前 MP → 逃跑動畫 → 返回探索模式
    → Worktree 保留（可稍後在公會大廳管理）
    → 不獲得獎勵
```

## 玩家旅程地圖

### 新手教學流程

```
1. 出生在城鎮中心
   NPC：「歡迎來到 CodeLand！讓我帶你熟悉這個世界。」
       ↓
2. 引導參觀城鎮
   酒館 → 商業街 → 公會大廳
       ↓
3. 第一次對話（酒館）
   用戶：「什麼是 React？」→ AI 回答，不觸發戰鬥
       ↓
4. 第一次簡單任務
   用戶：「創建 Hello World」→ 複雜度 2，主 CLI 完成，獲得少量 EXP
       ↓
5. 引導前往野外
   NPC：「想要更多經驗，去野外挑戰吧！」
       ↓
6. 第一次遭遇戰（森林，教學模式降低難度）
   用戶：「實作一個計數器組件」→ 觸發戰鬥 → 引導完成
       ↓
7. 戰鬥勝利，解鎖成就：「第一次勝利」
   引導合併 Worktree
       ↓
8. 教學完成
   NPC：「你已掌握基本操作，去探索更大的世界吧！」
```

### 日常開發流程

```
早上：
  城鎮休息恢復 → 酒館討論今日任務 → 前往野外

中午：
  輸入任務 Prompt → 觸發遭遇戰 → 自動創建 Worktree
  → 戰鬥中使用技能和夥伴 → 戰鬥勝利 → 保留分支

下午：
  返回城鎮 → 商業街升級技能 → 公會大廳查看分支
  → 繼續新任務或合併已完成的分支

晚上：
  公會大廳合併分支 → 獲得合併獎勵 → 住所休息存檔
```

### 多戰鬥並行流程

```
1. 野外開始戰鬥 1：「實作用戶認證」
   → Worktree: feature/auth → 戰鬥實例 #1
       ↓
2. 同時開始戰鬥 2：「修復支付 Bug」
   → Worktree: fix/payment → 戰鬥實例 #2
       ↓
3. 同時開始戰鬥 3：「重新設計 UI」
   → Worktree: feature/ui → 戰鬥實例 #3
       ↓
4. 公會大廳查看所有戰鬥進度
   → 選擇進入、暫停、或完成特定戰鬥
```

最多 3 個並發戰鬥（由非同步戰鬥系統管理），超出則排隊等待。

→ 詳見 [systems/09-async-battle-system.md](systems/09-async-battle-system.md)

### 緊急 Hotfix 流程

```
1. 正在進行功能開發戰鬥（feature/new-dashboard）
       ↓
2. 收到緊急通知：「生產環境登入失敗！」
       ↓
3. 返回城鎮（戰鬥暫停，Worktree 保留）
       ↓
4. 公會大廳創建 Hotfix Worktree（基於 main）
       ↓
5. 野外開始 Hotfix 戰鬥 → 快速修復 → 勝利
       ↓
6. 立即合併到 main → 推送 → 刪除 hotfix Worktree
       ↓
7. 回到公會大廳 → 切換回 feature/new-dashboard → 繼續開發
```

## 場景切換觸發條件總表

| 從 | 到 | 觸發條件 |
|----|----|---------|
| 探索 → 戰鬥 | 任務型 Prompt + 複雜度 ≥ 8 + 在野外/副本 |
| 戰鬥 → 探索 | 敵人 HP = 0（勝利） |
| 戰鬥 → 探索 | 玩家選擇逃跑（消耗 30% MP） |
| 任意 → 靜止之間 | Plan Mode 觸發 / 玩家主動進入 |
| 靜止之間 → 返回 | 計劃完成 / 玩家退出 |
| 城鎮 → 野外 | 玩家選擇前往（非戰鬥時） |
| 野外 → 城鎮 | 玩家選擇返回（非戰鬥時） |
| 任意 → 副本 | 特定 Prompt 觸發 + 確認 |
| 副本 → 城鎮 | 戰鬥完成（自動） |
