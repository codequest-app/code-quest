## 文檔結構

每個畫面文檔遵循統一的結構：

1. **畫面概述** - 功能定位、進入/退出條件
2. **完整布局** - ASCII 藝術展示完整畫面
3. **區塊劃分** - 詳細說明各個區塊
4. **組件清單** - 共用和特有組件列表
5. **互動設計** - 鍵盤、滑鼠、觸控操作
6. **轉場設計** - 進入/退出動畫和音效
7. **狀態變化** - 不同狀態下的顯示變化
8. **響應式設計** - 桌面/平板/手機適配
9. **無障礙設計** - ARIA、鍵盤導航、螢幕閱讀器
10. **技術規格** - 效能目標、資料結構、實作細節
11. **相關文檔** - 引用和連結

## 設計特色

### 視覺風格
- **RPG 像素風格** - 使用 Emoji 圖標和 ASCII 藝術
- **沉浸式設計** - 每個場所有獨特的視覺風格和氛圍
- **統一色彩** - 探索模式使用溫暖的綠色和藍色基調

### 互動設計
- **多平台支援** - 桌面、平板、手機完整適配
- **多種操作方式** - 鍵盤、滑鼠、觸控、快捷鍵
- **流暢轉場** - 精心設計的動畫和音效

### 技術規範
- **效能優化** - 目標 60 FPS，快速載入
- **無障礙支援** - 完整的 ARIA 標籤和鍵盤導航
- **響應式佈局** - 適應各種螢幕尺寸
