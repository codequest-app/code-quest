# 副本入口 (Dungeon Entrance)

## 畫面概述

- **功能定位**: 副本（Boss 戰）的入口和資訊展示
- **進入條件**: 從野外場景靠近副本入口，按 `E` 互動
- **退出條件**: 選擇挑戰副本（進入戰鬥），或返回野外
- **場景模式**: 探索模式 → 戰鬥模式（進入後）

## 區塊劃分

### 區塊 1: 頂部狀態欄
- 當前位置: CodeLand - [副本名稱] 入口
- 玩家資源顯示

### 區塊 2: 副本視覺區
- 副本名稱和圖標
- 入口門戶（帶動畫效果）
- 環境氛圍（陰暗、危險感）

### 區塊 3: 副本資訊欄（主要內容區）
- **難度**: 星級評價（1-5 星）
- **建議等級**: 等級範圍
- **預計時間**: 完成所需時間
- **Boss 資訊**: Boss 名稱和等級

### 區塊 4: 特殊機制說明
- 副本獨有的戰鬥機制
- 提示玩家需要注意的點

### 區塊 5: 獎勵預覽
- EXP 範圍
- 金幣範圍
- 特殊獎勵（技能、道具）

### 區塊 6: 警告和操作按鈕
- 紅色警告文字
- [挑戰副本] 和 [返回] 按鈕

## 互動設計

### 鍵盤操作
- **E**: 查看副本資訊（從野外靠近時）
- **Enter**: 確認挑戰副本
- **Esc**: 返回野外
- **↑/↓**: 在按鈕間切換焦點

### 滑鼠操作
- **點擊入口門戶**: 顯示副本資訊
- **點擊 [挑戰副本]**: 打開確認對話框
- **點擊 [返回]**: 返回野外
- **懸停獎勵**: 顯示詳細說明

### 觸控操作（iPad/iPhone）
- **點擊入口門戶**: 顯示資訊
- **點擊 [挑戰副本]**: 確認對話框
- **長按獎勵**: 顯示詳細說明

## 轉場設計

### 進入副本
- **來源**: 野外場景
- **動畫**: 門戶吸入效果 + 旋轉 (1.5s)
- **音效**: portal_enter.mp3
- **視覺**: 螺旋吸入，場景扭曲

### 完成副本
- **目標**: 野外場景
- **動畫**: 光芒閃爍 + 淡出 (1.0s)
- **音效**: victory.mp3
- **視覺**: 勝利慶祝，然後返回

## 狀態變化

### 等級不足警告
- 如果玩家等級低於建議等級
- [挑戰副本] 按鈕變黃色
- 顯示警告圖標 ⚠️

### MP 不足警告
- 如果 MP < 50
- 顯示警告: "MP 不足，戰鬥可能困難"
- 建議返回城鎮恢復

### 首次挑戰提示
- 首次進入副本時
- 顯示教學提示
- 說明副本機制和注意事項

### 副本已完成
- 如果已完成該副本
- 顯示 ✅ 已完成標記
- 獎勵改為「重複獎勵」（減少）

```javascript
const dungeon = {
  id: 'bug_cave',
  name: 'Bug 洞窟',
  icon: '🐛',
  difficulty: 3, // 1-5 stars
  recommendedLevel: [8, 12],
  estimatedTime: '30-45 分鐘',
  boss: {
    id: 'bug_king',
    name: 'Bug 之王',
    level: 12
  },
  specialMechanics: [
    '需要追蹤錯誤日誌',
    '3 個階段戰鬥'
  ],
  rewards: {
    exp: [800, 1200],
    gold: [400, 600],
    skills: ['debug_master'],
    items: []
  },
  warningMessage: '進入後無法退出，直到完成戰鬥',
  location: { x: 500, y: 300 }, // 在野外地圖的位置
  unlockRequirements: {
    level: 8,
    prerequisiteDungeons: []
  },
  completed: false,
  timesCompleted: 0,
  bestTime: null
}
```

### 進入條件檢查

```javascript
function canEnterDungeon(player, dungeon) {
  const checks = {
    levelSufficient: player.level >= dungeon.recommendedLevel[0],
    mpSufficient: player.mp >= 50,
    prerequisitesMet: dungeon.unlockRequirements.prerequisiteDungeons
      .every(id => player.completedDungeons.includes(id))
  }

  return {
    canEnter: Object.values(checks).every(v => v),
    warnings: Object.entries(checks)
      .filter(([k, v]) => !v)
      .map(([k, v]) => `${k} 不滿足`)
  }
}
```
