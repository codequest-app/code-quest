# 戰鬥模式畫面文檔總覽

## 階段 2 批次 2：戰鬥模式畫面設計

本目錄包含 RPG-CLI 戰鬥模式的 7 個核心畫面設計文檔，涵蓋單線戰鬥、並發戰鬥、技能選擇、夥伴管理、召喚獸和敵人顯示等所有戰鬥相關介面。

## 文檔列表

### 主戰鬥畫面

1. **[battle-main.md](./battle-main.md)** - 主戰鬥畫面
   - 單線戰鬥的核心介面
   - 敵人資訊、戰鬥日誌、行動選單
   - 傷害數字動畫和勝利畫面
   - 完整的鍵盤、滑鼠、觸控操作

2. **[battle-async.md](./battle-async.md)** - 三面板並發戰鬥
   - 支援最多 3 個戰鬥同時進行
   - 左側對話區（20%）+ 中央主視窗（50%）+ 右側戰鬥列表（30%）
   - 實時進度更新和 WebSocket 事件處理
   - 戰鬥狀態管理（queued/in_progress/paused/completed/failed）

### 戰鬥子畫面

3. **[skill-selection.md](./skill-selection.md)** - 技能選擇介面
   - 覆蓋層式技能選擇面板
   - 技能狀態：可用、冷卻、MP不足、未解鎖
   - 弱點技能自動推薦和高亮
   - 快捷鍵 Ctrl+1~9 快速施放

4. **[companion-panel.md](./companion-panel.md)** - 夥伴面板（戰鬥中）
   - 已召喚夥伴管理（最多 3 個）
   - 夥伴技能施放
   - 召喚和解除機制
   - 夥伴狀態：待命、活躍、疲憊、倒下

5. **[summon-display.md](./summon-display.md)** - 召喚獸顯示
   - 4 種行為類型：即時、自動、被動、互動
   - 魔法陣召喚動畫
   - 技能施放特效（龍息、浴火重生等）
   - 回合倒數和稀有度顯示

6. **[enemy-display.md](./enemy-display.md)** - 敵人資訊顯示
   - 簡化顯示（戰鬥畫面內嵌）
   - 詳細顯示（彈窗）
   - HP 條動畫和狀態變化
   - 弱點、抵抗、免疫系統
   - 敵人能力和獎勵預覽

### 總覽文檔

7. **[README.md](./README.md)** - 本文檔
   - 戰鬥模式文檔總覽
   - 文檔關係圖
   - 快捷鍵速查表

## 文檔結構

每個畫面文檔遵循統一的結構：

1. **畫面概述** - 功能定位、進入/退出條件
2. **完整布局** - ASCII 藝術展示完整畫面
3. **區塊劃分** - 詳細說明各個區塊
4. **組件清單** - 共用和特有組件列表
5. **互動設計** - 鍵盤、滑鼠、觸控操作
6. **轉場設計** - 進入/退出動畫和音效
7. **狀態變化** - 不同狀態下的顯示變化
8. **響應式設計** - 桌面/平板/手機適配
9. **無障礙設計** - ARIA、鍵盤導航、螢幕閱讀器
10. **技術規格** - 效能目標、資料結構、實作細節
11. **相關文檔** - 引用和連結

## 設計特色

### 視覺風格
- **Pixel RPG 風格** - 8-bit 字體、像素化邊框和背景
- **戰鬥氛圍** - 深紅色基調，緊張感設計
- **動態反饋** - 豐富的動畫和粒子效果
- **清晰資訊** - 色彩編碼的戰鬥日誌和狀態顯示

### 互動設計
- **多平台支援** - 桌面、平板、手機完整適配
- **快捷鍵系統** - Ctrl+數字鍵快速施放技能
- **即時反饋** - 傷害數字、HP 條動畫、音效配合
- **流暢轉場** - 精心設計的戰鬥開始/結束動畫

### 技術規範
- **效能優化** - 目標 60 FPS，< 100ms 響應時間
- **並發處理** - WebSocket 實時更新，最多 3 個並發戰鬥
- **狀態管理** - Zustand 或 Redux 管理複雜戰鬥狀態
- **無障礙支援** - 完整的 ARIA 標籤和鍵盤導航

## 場景關係圖

```
戰鬥主畫面 (battle-main.md)
├─ 技能選擇 (skill-selection.md)
│  └─ 施放技能 → 返回戰鬥
├─ 夥伴面板 (companion-panel.md)
│  ├─ 召喚夥伴 → 返回戰鬥
│  └─ 施放夥伴技能 → 返回戰鬥
├─ 召喚獸顯示 (summon-display.md)
│  └─ 嵌入戰鬥畫面
├─ 敵人顯示 (enemy-display.md)
│  ├─ 簡化顯示（嵌入）
│  └─ 詳細資訊（彈窗）
└─ 戰鬥結束
   ├─ 勝利 → 獎勵畫面 → 探索模式
   └─ 失敗 → 失敗畫面 → 探索模式

並發戰鬥 (battle-async.md)
├─ 對話軌道（左側 20%）
│  └─ 即時對話，不被戰鬥阻塞
├─ 主視窗（中央 50%）
│  └─ 顯示當前焦點戰鬥
└─ 戰鬥列表（右側 30%）
   ├─ 進行中戰鬥卡片
   ├─ 排隊中戰鬥
   └─ 戰鬥詳情面板
```

## 戰鬥流程圖

### 單線戰鬥流程
```
開始戰鬥
  ↓
[玩家回合]
  ├─ 選擇技能 → 施放 → 造成傷害
  ├─ 召喚夥伴 → 夥伴加入戰鬥
  ├─ 使用道具 → 恢復 HP/MP
  ├─ 防禦姿態 → 減少傷害
  └─ 逃跑 → 逃跑檢定
  ↓
[敵人回合]
  ├─ 選擇技能（AI）
  ├─ 施放技能 → 造成傷害
  └─ 特殊能力觸發
  ↓
[回合結束]
  ├─ 更新冷卻
  ├─ 處理持續效果
  ├─ 檢查勝負
  │   ├─ 敵人 HP = 0 → 勝利
  │   ├─ 玩家 HP = 0 → 失敗
  │   └─ 繼續 → 下一回合
  └─ 返回玩家回合
```

### 並發戰鬥流程
```
啟動複雜任務
  ↓
創建異步戰鬥
  ↓
檢查並發數
  ├─ < 3 個 → 立即開始
  └─ = 3 個 → 加入排隊
  ↓
[戰鬥進行中]
  ├─ WebSocket 實時更新
  ├─ 進度條動畫
  ├─ 最近動作記錄
  └─ 用戶可暫停/取消
  ↓
戰鬥完成
  ├─ 顯示完成通知
  ├─ 從戰鬥列表移除
  ├─ 檢查排隊
  │   └─ 有排隊 → 啟動下一個
  └─ 顯示獎勵
```

## 狀態系統

### 戰鬥狀態
```typescript
type BattleStatus =
  | 'initializing'  // 初始化
  | 'player_turn'   // 玩家回合
  | 'enemy_turn'    // 敵人回合
  | 'processing'    // 處理中
  | 'victory'       // 勝利
  | 'defeat'        // 失敗
  | 'fled';         // 逃跑成功
```

### 異步戰鬥狀態
```typescript
type AsyncBattleStatus =
  | 'queued'        // 排隊中
  | 'in_progress'   // 進行中
  | 'paused'        // 已暫停
  | 'completing'    // 收尾中
  | 'completed'     // 已完成
  | 'failed';       // 失敗
```

### 技能狀態
```typescript
type SkillStatus =
  | 'available'        // 可用
  | 'cooldown'         // 冷卻中
  | 'insufficient_mp'  // MP 不足
  | 'locked';          // 未解鎖
```

### 夥伴狀態
```typescript
type CompanionStatus =
  | 'standby'    // 待命中
  | 'active'     // 活躍中
  | 'exhausted'  // 疲憊
  | 'ko';        // 倒下
```

```
0.4s  敵人受傷
0.5s  HP 條減少
```


## 效能指標

### 目標指標
- **初始載入**: < 800ms
- **戰鬥開始動畫**: 2s 完成
- **技能施放**: < 100ms 響應
- **幀率**: 維持 60 FPS
- **WebSocket 延遲**: < 50ms
- **並發戰鬥數**: 最多 3 個

### 優化策略
- 使用 CSS Transform 和 Opacity 做動畫
- 虛擬滾動處理長列表
- 防抖和節流處理頻繁事件
- 圖層分離減少重繪
- 預加載關鍵資源

## 無障礙設計原則

### ARIA 支援
- 所有互動元素有 `aria-label`
- 動態內容使用 `aria-live`
- 進度條使用 `role="progressbar"`
- 模態框使用 `aria-modal="true"`

### 鍵盤導航
- 所有功能可通過鍵盤操作
- Tab 順序符合邏輯流程
- 焦點指示清晰可見
- Esc 鍵統一返回/取消

### 螢幕閱讀器
- 重要事件優先朗讀
- 狀態變化即時通知
- 完整的描述性標籤
- 避免純視覺提示

### 色盲友好
- 不只用顏色區分狀態
- 配合圖標和文字標籤
- 紋理和圖案差異
- 明確的數值顯示
