## 核心原則

1. **每個工具都有對應的魔法** — 無論是 Bash、Read、Write 還是其他工具
2. **MP 消耗與複雜度成正比** — 簡單查詢 2-4 MP、修改操作 5-8 MP、重型操作 10-15 MP、特殊操作 15-20+ MP
3. **效果多樣化** — 造成傷害、獲得 Buff、發現弱點、治療恢復
4. **即時反饋** — 所有操作都有視覺反饋與戰鬥日誌
5. **失敗有代價** — 錯誤造成反噬傷害，鼓勵謹慎操作

## 特殊機制

### 連擊系統（並行操作偵測）

當 2 個以上工具同時執行時，觸發連擊加成：

| 並行數量 | 連擊名稱 | 傷害加成 | 額外 MP |
|---------|---------|---------|--------|
| 2-3 工具 | 雙重施法 | +15% | +5 MP |
| 4-5 工具 | 多重施法 | +25% | +10 MP |
| 6+ 工具 | 究極魔法 | +40% | +15 MP |

### 施法失敗與反噬

工具執行失敗時，造成反噬傷害：

```
反噬傷害 = 基礎傷害 × 嚴重度 × 連續失敗倍率

基礎傷害 = 工具 MP 消耗 × 5
嚴重度:
  語法錯誤 = ×1.0
  邏輯錯誤 = ×0.8
  權限錯誤 = ×1.2
  網絡錯誤 = ×0.6
  系統錯誤 = ×1.5
連續失敗倍率 = 1.0 → 1.5 → 2.0 → 2.5...（每次連續失敗遞增）
```

### 施法速度分級

| 速度 | 時間範圍 | 進度條顯示 | 範例工具 |
|------|---------|-----------|---------|
| 即時 | 0-0.5s | 無進度條 | ls, pwd, git status |
| 快速 | 0.5-2s | 短進度條 | Read, git commit |
| 中等 | 2-10s | 標準進度條+百分比 | Edit, npm test |
| 慢 | 10s+ | 長進度條+剩餘時間 | npm install, docker build |

### 流式輸出（持續效果）

長時間操作（npm install, npm build, docker build）顯示持續進度，類似持續傷害效果。每個里程碑提供部分獎勵，全部完成給予完整獎勵。
