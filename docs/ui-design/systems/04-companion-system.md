# L2 夥伴系統

**版本**: v2.0
**更新日期**: 2026-02-06

---

## 一、系統概述

夥伴系統將 Claude Code 的 **Subagent** 提升為具有獨立 HP/MP、專屬技能、AI 行為邏輯和成長機制的**戰鬥夥伴**。夥伴在整場戰鬥中持續參與，與玩家並肩作戰。

在系統架構中，夥伴系統位於 **L2 戰鬥擴展層**，依賴戰鬥系統提供的戰鬥框架。夥伴是「長期盟友」— 與召喚獸系統（同為 L2）的「短暫爆發支援」形成互補。

**與召喚獸的核心差異**：

| 特性 | 夥伴（Companion） | 召喚獸（Summon） |
|------|-------------------|-----------------|
| 來源 | Subagent | Skill / 組合技 / MCP 工具 |
| 持續時間 | 整場戰鬥 | 1-3 回合 |
| 成長 | 有（等級、技能解鎖） | 無 |
| MP 恢復 | 戰鬥後完全恢復 | 不適用 |
| 定位 | 長期持續支援 | 關鍵時刻爆發 |

## 二、依賴關係

```
依賴：
  └─ 戰鬥系統 (3) — 夥伴在戰鬥回合中行動

被依賴：
  └─ 非同步戰鬥系統 (9) — 夥伴可獨立執行非同步戰鬥
```

| 方向 | 系統 | 關係說明 |
|------|------|---------|
| 依賴 | 戰鬥系統 (3) | 夥伴在戰鬥的「夥伴回合」中執行行動 |
| 被依賴 | 非同步戰鬥系統 (9) | 夥伴可被委派獨立處理後台戰鬥任務 |

**間接關聯**：
- 傭兵公會（商店系統 6 的子商店）— 管理夥伴的招募、查看、配置
- Worktree 系統 (8) — 夥伴在非同步戰鬥中使用獨立 Worktree

## 三、核心規則

### 3.1 Subagent → 夥伴映射

每個 Subagent 自動映射為一個戰鬥夥伴，映射規則如下：

**Agent 類型 → 職業**：

| Subagent 類型 | 夥伴職業 | 職業特點 |
|--------------|---------|---------|
| Bash Agent（命令執行型） | 戰士（Warrior） | 高攻擊、高 HP、擅長直接輸出 |
| Explore Agent（探索型） | 偵察兵（Scout） | 高速度、偵測能力、擅長搜索定位 |
| Plan Agent（規劃型） | 策略師（Strategist） | 高智慧、輔助能力、擅長分析規劃 |

**記憶設定 → 經驗等級**：

| Subagent 記憶 | 夥伴經驗等級 | 說明 |
|--------------|-------------|------|
| 無記憶 | 新手（Lv.1-5） | 每次戰鬥從零開始 |
| 短期記憶 | 熟練（Lv.5-10） | 記住近期戰鬥經驗 |
| 長期記憶 | 精通（Lv.10+） | 累積所有歷史經驗 |

**使用模型 → 能力等級**：

| AI 模型 | 能力等級 | 基礎屬性倍率 |
|---------|---------|-------------|
| Opus | 傳奇（Legendary） | ×1.5 |
| Sonnet | 稀有（Rare） | ×1.0 |
| Haiku | 普通（Common） | ×0.7 |

### 3.2 夥伴基礎屬性

每個夥伴擁有 6 項基礎屬性：

| 屬性 | 說明 | 影響 |
|------|------|------|
| HP（生命值） | 夥伴的生存能力 | HP 歸零則夥伴退場 |
| MP（魔力值） | 夥伴的技能資源 | 消耗完畢則只能普攻 |
| 攻擊（Attack） | 物理/魔法輸出能力 | 影響技能傷害 |
| 防禦（Defense） | 承受傷害的能力 | 減少受到的傷害 |
| 速度（Speed） | 行動優先級 | 影響回合內行動順序 |
| 智慧（Wisdom） | 技能效果和判斷力 | 影響 AI 行為決策品質 |

**屬性來源**：基礎值由職業決定，能力等級提供倍率加成，經驗等級影響成長上限。

### 3.3 戰鬥配置規則

| 規則 | 數值 | 說明 |
|------|------|------|
| 最大持有夥伴數 | 無上限 | 所有招募的 Subagent 都是夥伴 |
| 戰鬥出場上限 | 2 名 | 同時最多 2 名夥伴參戰 |
| 待命槽位 | 1 個 | 額外 1 名夥伴待命（共 3 槽位） |
| 召喚 MP 消耗 | 20 MP | 將夥伴從待命召喚到戰場 |
| 切換 MP 消耗 | 10 MP | 替換已在場的夥伴 |

### 3.4 夥伴 AI 行為

夥伴在戰鬥中由 AI 自動控制，根據戰場狀況選擇最佳行動：

**行為優先級**：

```
1. 緊急治療 — 玩家 HP < 30% 時，優先使用治療技能
2. 解除異常 — 玩家或自身有負面狀態時，使用淨化技能
3. 增益輔助 — 判斷需要增強時，使用增益技能
4. 弱點攻擊 — 敵人有已知弱點時，使用對應屬性攻擊
5. 一般攻擊 — 上述條件都不滿足時，使用最高傷害技能
```

**職業特化行為**：

| 職業 | AI 傾向 | 說明 |
|------|---------|------|
| 戰士 | 攻擊優先 | 偏好直接輸出，HP 充足時不防禦 |
| 偵察兵 | 偵測優先 | 先偵測敵人弱點，再選擇對應攻擊 |
| 策略師 | 輔助優先 | 優先提供增益/治療，最後才攻擊 |

### 3.5 夥伴 MP 管理

| 規則 | 說明 |
|------|------|
| 戰鬥中消耗 | 每次使用技能消耗固定 MP |
| 戰鬥中恢復 | 不恢復（戰鬥中 MP 只減不增） |
| 戰鬥結束後 | 完全恢復（HP 和 MP 都回滿） |
| MP 耗盡 | 只能使用普通攻擊（0 MP 消耗，低傷害） |

### 3.6 成長系統

夥伴在戰鬥中獲得經驗值，升級後解鎖新技能並提升屬性：

**經驗獲取**：

```
夥伴 EXP = 戰鬥基礎 EXP × 參與度倍率

參與度倍率：
  全程參戰 = ×1.0
  中途召喚 = ×0.7
  中途退場 = ×0.5
```

**升級效果**：

| 等級里程碑 | 效果 |
|-----------|------|
| 每次升級 | HP +5, MP +3, 主屬性 +2 |
| 每 3 級 | 解鎖一個新技能 |
| 每 5 級 | 職業特化升級（AI 行為更精準） |
| 每 10 級 | 解鎖特殊被動技能 |

## 五、與其他系統的互動

### 5.1 夥伴 ← 戰鬥系統 (3)

戰鬥系統在每回合的夥伴階段呼叫夥伴系統：

```
戰鬥系統                        夥伴系統
   │                               │
   │  夥伴回合開始                   │
   │  (戰場狀態：敵人、玩家 HP 等)   │
   │──────────────────────────────►│
   │                               │  AI 行為決策
   │                               │  選擇技能
   │                               │  執行行動
   │  行動結果                      │
   │  (傷害值/治療量/增益效果)       │
   │◄──────────────────────────────│
   │                               │
   │  更新戰場狀態                   │
```

### 5.2 夥伴 → 非同步戰鬥系統 (9)

夥伴可被委派執行非同步戰鬥：

- 用戶在公會大廳將夥伴派往獨立戰鬥
- 夥伴在獨立 Worktree 中執行任務
- 戰鬥結果在完成後通知用戶
- 夥伴獲得完整的戰鬥 EXP

### 5.3 夥伴 ↔ 商店系統 (6) — 傭兵公會

傭兵公會（商店系統的子商店）提供夥伴管理功能：

- **查看**：所有夥伴的狀態、等級、技能
- **配置**：設定出場陣容（2 出場 + 1 待命）
- **強化**：使用 Gold 提升夥伴屬性
- **招募**：將新的 Subagent 映射為夥伴

### 5.4 夥伴 ↔ Worktree 系統 (8)

夥伴在非同步戰鬥中使用獨立 Worktree：

- 每個被派遣的夥伴使用獨立的 Worktree 分支
- 夥伴戰鬥完成後，分支可合併/保留/放棄

## 六、設計決策說明

### 6.1 為什麼夥伴映射自 Subagent？

**問題**：Subagent 是 Claude Code 的核心概念，但在標準 CLI 中只是「被委派的子任務」，缺乏個性和歸屬感。

**決策**：將 Subagent 映射為有獨立屬性、技能、成長的夥伴角色。Agent 類型決定職業、記憶決定經驗、模型決定能力等級。

**效果**：Subagent 從「匿名工具」變成「有名字、有個性的戰鬥夥伴」，用戶對長期培養的夥伴產生歸屬感。

### 6.2 為什麼夥伴由 AI 自動控制？

**問題**：如果每回合都要手動為夥伴選擇技能，戰鬥中需要管理的事項太多（自己 + 2 個夥伴），會破壞流暢體驗。

**決策**：夥伴有獨立的 AI 行為引擎，根據戰場狀況自動選擇最佳行動。不同職業有不同的 AI 傾向。

**效果**：用戶只需觀察夥伴的自動行動，減少管理負擔。夥伴的 AI 行為讓它們感覺像「有自主意識的盟友」而非「需要操控的棋子」。

### 6.3 為什麼限制出場數為 2 名？

**問題**：如果允許大量夥伴同時出場，戰鬥畫面會過於擁擠，每回合等待時間過長，玩家注意力被分散。

**決策**：最多 2 名夥伴出場 + 1 名待命，共 3 個槽位。

**效果**：
- 戰鬥畫面清晰：玩家 + 2 夥伴 vs 敵人
- 回合節奏合理：3 個行動階段（玩家→夥伴→敵人）
- 策略深度適中：選擇哪 2 名出場、何時切換

### 6.4 為什麼夥伴 MP 戰鬥中不恢復？

**問題**：如果夥伴 MP 可以在戰鬥中恢復，技能使用就沒有資源壓力，失去策略意義。

**決策**：夥伴 MP 在戰鬥中只減不增，戰鬥結束後完全恢復。

**效果**：
- 夥伴技能使用有限，需要 AI 智慧地分配（不能每回合都放大招）
- 長時間戰鬥中夥伴會逐漸 MP 耗盡，只能普攻
- 鼓勵在合適時機切換夥伴（用新的夥伴替換 MP 耗盡的）

### 6.5 為什麼有成長系統？

**問題**：如果夥伴永遠不變，用戶缺乏長期培養的動機。

**決策**：夥伴獲得戰鬥 EXP、升級、解鎖新技能、提升屬性。長期使用同一個夥伴會讓它變得更強。

**效果**：
- 用戶對長期培養的夥伴產生情感連結
- 提供持續的成長回饋，增加長期使用動機
- 高等級夥伴在困難戰鬥中表現更好，獎勵持續投入
