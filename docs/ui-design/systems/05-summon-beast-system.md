# L2 召喚獸系統

**版本**: v2.0
**更新日期**: 2026-02-06

---

## 一、系統概述

召喚獸系統為玩家提供**關鍵時刻的短暫爆發型強力支援**。與夥伴系統（長期持續）不同，召喚獸是高消耗、高回報、限時存在的戰術工具。

在架構中的定位：

- **層級**：L2（戰鬥擴展層）
- **編號**：系統 5
- **核心角色**：補充戰鬥系統的戰術深度，與夥伴系統形成互補關係
- **設計哲學**：夥伴 = 長期戰友（整場陪伴），召喚獸 = 特殊支援（關鍵一擊）

### 依賴的系統

| 系統 | 依賴內容 |

無。召喚獸系統是 L2 葉節點，不被其他系統依賴。

## 三、核心規則

### 3.1 四種召喚來源

| # | 召喚類型 | 觸發方式 | 範例 |
|---|---------|---------|------|
| 1 | 技能召喚 | 使用特定 Skill 觸發 | 使用 `summon-code-dragon` → 召喚代碼之龍 |
| 2 | 組合技召喚 | 觸發組合技時有機率出現 | 完美代碼連擊 → 召喚不死鳥（50% 機率） |
| 3 | MCP 工具召喚 | 啟動特定 MCP 工具觸發 | 啟動 database-mcp → 召喚資料庫魔像 |
| 4 | 道具召喚 | 消耗特定道具觸發 | 使用召喚卷軸 → 召喚幫助精靈 |

### 3.2 四種行為類型

| # | 行為類型 | 持續時間 | 行動方式 | 適合場景 |
|---|---------|---------|---------|---------|
| 1 | 立即行動 (Immediate) | 1 回合 | 出現 → 施放 → 離去 | 緊急爆發（如 AoE 傷害） |
| 2 | 自動行動 (Automatic) | 2-3 回合 | 每回合自動選擇最佳技能 | 持續支援（如連續治療） |
| 3 | 被動效果 (Passive) | 3-5 回合（或直到解除） | 不主動攻擊，提供被動增益 | 增益加成（如 +20% 傷害） |
| 4 | 互動行動 (Interactive) | 2-3 回合 | 玩家手動選擇技能 | 戰術操控（如選擇攻擊/防禦） |

### 3.3 槽位規則

```
戰鬥單位配置：
├─ 玩家 (1)           ← 固定
├─ 夥伴槽 (最多 2)    ← 夥伴系統管理
└─ 召喚獸槽 (最多 1)  ← 召喚獸系統管理

規則：
• 同時只能有 1 個召喚獸在場
• 召喚新召喚獸會解除現有召喚獸
• 夥伴槽與召喚獸槽獨立
• 立即行動型不占用槽位（出現即離去）
```

### 3.4 資源消耗

MP 消耗依稀有度分級：

| 稀有度 | MP 消耗範圍 | 冷卻時間 | 威力等級 |
|-------|-----------|---------|---------|
| 普通 (Common) | 20-30 MP | 120 秒 | 中等 |
| 稀有 (Uncommon) | 30-50 MP | 180 秒 | 較強 |
| 精良 (Rare) | 50-70 MP | 300 秒 | 強力 |
| 史詩 (Epic) | 70-90 MP | 450 秒 | 很強 |
| 傳說 (Legendary) | 90-120 MP | 600 秒以上 | 極強 |

### 3.5 召喚獸不成長

召喚獸**沒有經驗值和升級機制**。能力固定，不會隨時間變強。這是與夥伴系統的核心差異：

- 夥伴 = 「寵物」，陪伴成長
- 召喚獸 = 「工具」，即用即棄

## 五、與其他系統的互動

### 5.1 與戰鬥系統 (3) 的互動

| 互動點 | 說明 |
|-------|------|
| 回合順序 | 自動行動 / 互動行動型召喚獸加入回合順序，在夥伴回合之後行動 |
| 傷害計算 | 召喚獸技能使用戰鬥系統的傷害公式（基礎威力 x 屬性相性 x 隨機浮動） |
| 戰鬥日誌 | 召喚獸的所有行動記錄到戰鬥日誌 |
| 戰鬥結束 | 戰鬥結束時所有召喚獸自動離場 |

### 5.2 與夥伴系統 (4) 的互動

| 互動點 | 說明 |
|-------|------|
| 槽位獨立 | 夥伴槽（最多 2）與召喚獸槽（最多 1）互不衝突 |
| 協同效果 | 特定夥伴 + 特定召喚獸同時在場觸發協同加成 |
| 行動順序 | 玩家 → 夥伴 → 召喚獸 → 敵人 |

### 5.3 與商店系統 (6) 的互動

| 互動點 | 說明 |
|-------|------|
| 寶物庫 | 召喚獸圖鑑在寶物庫中展示，追蹤收集進度 |
| 訓練場 | 可在訓練場測試召喚獸效果（不消耗資源） |

## 六、設計決策說明

### 為什麼與夥伴系統分離？

夥伴是 Subagent 的映射，代表持續存在的協作 Agent。召喚獸代表特定技能 / 工具 / 組合技的爆發效果。兩者本質不同，分開設計避免概念混淆。

### 為什麼限制同時只有 1 個召喚獸？

- 降低戰鬥畫面複雜度（玩家 + 2 夥伴 + 1 召喚獸 = 最多 4 個單位）
- 強迫玩家做出選擇，增加戰術深度
- 避免多召喚獸堆疊導致平衡崩壞

### 為什麼高 MP 消耗？

高消耗創造有意義的資源決策。如果召喚獸便宜，玩家每回合都會召喚，失去「關鍵時刻」的戰術價值。高消耗 + 強效果 = 需要判斷使用時機。

### 為什麼召喚獸不成長？

- 與夥伴系統形成差異化：夥伴成長（情感連結），召喚獸固定（工具定位）
- 簡化系統複雜度，避免兩套成長系統互相競爭
- 召喚獸的價值來自「使用時機」而非「養成投入」

### 為什麼設計四種行為類型？

提供多樣化的戰術選擇：
- 立即行動：適合緊急救場、一擊致命
- 自動行動：適合持續輸出、穩定支援
- 被動效果：適合長期增益、資源積累
- 互動行動：適合高操作玩家、精確控制
