# L0 商店系統

**版本**: v2.0
**更新日期**: 2026-02-06

---

## 一、系統概述

商店系統將 Claude Code 的功能管理（Skills、MCP、Subagent）**RPG 化為 7 個商店**，位於城鎮的商業街。使用者透過探索商店來發現和管理功能，取代傳統的文檔查閱方式。

在架構中的定位：

- **層級**：L0（基礎層）
- **編號**：系統 6
- **核心角色**：所有功能管理的統一入口，將技術概念簡化為 RPG 世界中的「逛商店」體驗
- **設計哲學**：探索式發現 > 閱讀文檔；成長解鎖 > 一次全開

## 三、核心規則

### 3.1 七大商店

所有商店位於城鎮商業街，探索模式專屬。

| # | 商店名稱 | 功能定位 | 管理對象 |
|---|---------|---------|---------|
| 1 | 技能商店 | 查看、解鎖、升級技能 | Claude Code Skills |
| 2 | 工匠鋪 | 創建自定義技能 | 自定義 Skill |
| 3 | 魔法圖書館 | 瀏覽、安裝 MCP 工具 | MCP Server Tools |
| 4 | 傭兵公會 | 管理戰鬥夥伴 | Subagents |
| 5 | 寶物庫 | 成就、圖鑑、收藏品 | 遊戲進度 |
| 6 | 訓練場 | 教學、模擬戰鬥、技能測試 | 學習與練習 |
| 7 | 錢莊 | 資源統計、成本追蹤、預算管理 | Gold / EXP / AI 成本 |

### 3.2 各商店詳細規則

#### 1. 技能商店（Skill Shop）

- 瀏覽所有可用技能（官方 + 自定義）
- 查看技能詳情：MP 消耗、冷卻時間、傷害、使用統計
- 解鎖新技能：需滿足等級要求 + 支付 Gold
- 升級已有技能：降低 MP 消耗、減少冷卻、增加傷害

**升級成本公式**：

```
升級成本 = 基礎價格 x 1.5^(當前等級 - 1)

範例：
  Lv.1 → Lv.2: 50 Gold
  Lv.2 → Lv.3: 75 Gold
  Lv.3 → Lv.4: 112 Gold
  Lv.4 → Lv.5: 168 Gold
```

#### 2. 工匠鋪（Skill Forge）

**解鎖條件**：玩家等級 Lv.8

六步驟創建引導流程：

```
步驟 1: 技能命名     → 設定名稱、圖標、描述
步驟 2: 命令設定     → 定義調用命令和實際執行命令
步驟 3: 資源設定     → 設置 MP 消耗和冷卻時間
步驟 4: 戰鬥效果     → 配置傷害、特殊效果
步驟 5: 解鎖條件     → 等級、Gold、前置技能要求（可選）
步驟 6: 預覽和測試   → 測試執行並保存
```

預設模板分類：測試類、Git 類、建構類

#### 3. 魔法圖書館（MCP Tools Library）

- MCP 工具映射為「魔法書」
- 瀏覽已安裝工具、發現新工具
- 查看工具詳情：功能、版本、使用統計
- 安裝 / 卸載 / 配置工具

分類：已安裝、熱門、最新、按類型（數據庫、網頁、AI、開發工具）

#### 4. 傭兵公會（Subagent Guild）

- 瀏覽所有可用傭兵（Bash、Explore、Plan 等）
- 查看傭兵詳情：等級、職業、專長、技能
- 管理當前隊伍（最多 2 個夥伴）
- 設置默認召喚配置和自動召喚規則

#### 5. 寶物庫（Treasury）

- 成就系統：查看、追蹤進度、領取獎勵
- 收藏品圖鑑：召喚獸、技能、裝備
- 隱藏內容解鎖
- 成就類型：戰鬥成就、技能成就、探索成就、特殊成就

#### 6. 訓練場（Training Ground）

- 新手教學課程（3 課：基礎操作、技能使用、戰鬥系統）
- 模擬戰鬥：不消耗 HP/MP、不進入冷卻、不獲得獎勵、可無限重試
- 技能測試：不消耗 MP、顯示詳細執行日誌
- Combo 練習：練習已知組合技、自由組合發現新 Combo

#### 7. 錢莊（Bank）

- 資源總覽：當前 Gold、今日收支
- AI 模型成本統計：各模型使用次數與成本、總成本
- 預算管理：每日預算設定、超支策略（切換便宜模型 / 停止 / 僅警告）
- 統計圖表：過去 7 天趨勢、收支對比
- **多模型整合入口**：模型切換在此操作

### 3.3 存取規則

| 規則 | 說明 |
|------|------|
| 場所限制 | 只能在城鎮（探索模式）進入 |
| 戰鬥限制 | 戰鬥中無法訪問任何商店 |
| 快捷鍵 | 按 1-7 直接進入對應商店；K = 技能商店、G = 傭兵公會、M = 圖書館 |
| 等級門檻 | 部分商品和功能需達到特定等級才解鎖（如工匠鋪需 Lv.8） |

### 3.4 經濟循環

```
戰鬥勝利 / 成就獎勵
        │
        ▼
   獲得 Gold + EXP
        │
   ┌────┴────┐
   │         │
   ▼         ▼
 技能商店   工匠鋪
 (解鎖升級) (創建技能)
   │         │
   └────┬────┘
        │
        ▼
   戰鬥力提升
        │
        ▼
   挑戰更強敵人
        │
        ▼
   獲得更多獎勵 → (循環)
```

## 五、與其他系統的互動

### 5.1 與地圖系統 (1) 的互動

| 互動點 | 說明 |
|-------|------|
| 空間定位 | 商業街位於城鎮中心偏東，地圖上顯示商店圖標 |
| 導航 | 從地圖可直接點擊城鎮進入商業街 |
| 通知標記 | 有新功能解鎖時，地圖上的城鎮圖標顯示通知 |

### 5.2 與場景系統 (2) 的互動

| 互動點 | 說明 |
|-------|------|
| 模式限制 | 探索模式可進入商店；戰鬥模式禁止進入 |
| 菜單整合 | 探索模式主菜單中的「城鎮」選項進入商業街 |
| 模擬戰鬥 | 訓練場的模擬戰鬥使用獨立的戰鬥實例，與真實戰鬥隔離 |

### 5.3 與戰鬥系統 (3) 的互動

| 互動點 | 說明 |
|-------|------|
| 技能數據 | 戰鬥中使用的技能數據來自技能商店（等級、MP、冷卻） |
| 升級生效 | 技能升級後立即影響戰鬥表現 |
| 使用統計 | 戰鬥中的技能使用次數回傳商店統計 |
| Gold 獲得 | 戰鬥勝利獲得 Gold 用於商店消費 |

### 5.4 與多模型整合系統 (10) 的互動

| 互動點 | 說明 |
|-------|------|
| 成本統計 | 錢莊追蹤各 AI 模型的使用次數和成本 |
| 模型切換 | 在錢莊中切換 AI 模型（Claude / Gemini 等） |
| 預算聯動 | 超出預算時自動觸發模型切換策略 |

## 六、設計決策說明

### 為什麼把所有管理功能放進商店？

傳統 CLI 工具的功能管理分散在文檔、配置文件、命令列中。商店系統將所有管理入口統一到 RPG 世界的「商業街」，使用者用「逛商店」的方式探索和管理功能，降低學習曲線。

### 為什麼是 7 個商店而不是更少？

每個商店對應一個明確的管理職責，避免單一商店過於臃腫。7 個商店覆蓋：技能管理、技能創建、工具管理、夥伴管理、進度追蹤、學習練習、資源統計。職責清晰，使用者能快速定位需求。

### 為什麼保留快捷鍵？

商店的 RPG 包裝增加了沉浸感，但不應犧牲效率。快捷鍵（1-7 / K / G / M）讓熟練使用者跳過導航直達目標，兼顧新手友好和進階效率。

### 為什麼訓練場的模擬戰鬥不消耗資源？

訓練場的設計目的是降低嘗試成本。如果模擬戰鬥也消耗資源，使用者會避免測試和學習，違背設計初衷。零成本讓使用者放心探索。

### 為什麼錢莊整合多模型功能？

AI 模型選擇本質上是成本決策。將模型切換放在「錢莊」而非獨立設定頁面，強化了成本意識，讓使用者在資源統計的上下文中做出模型選擇。
