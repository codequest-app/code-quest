# L2 互動事件系統

**版本**: v2.0
**更新日期**: 2026-02-06

---

## 一、系統概述

互動事件系統處理戰鬥過程中**需要用戶互動的事件**（Plan Mode、AskUserQuestion、錯誤、權限請求），將它們用 RPG 元素包裝，使中斷感轉化為戰鬥中的自然事件。

在架構中的定位：

- **層級**：L2（戰鬥擴展層）
- **編號**：系統 7
- **核心角色**：確保 Claude Code 的互動機制在 RPG 戰鬥中正常運作，同時維持沉浸感
- **設計哲學**：功能包裝而非功能替換 — RPG 化是視覺層，原始功能完整保留

## 三、核心規則

### 3.1 四種事件類型

| # | 事件類型 | Claude Code 原始行為 | RPG 包裝 | 優先級 |
|---|---------|-------------------|---------|-------|
| 1 | Plan Mode | EnterPlanMode 被調用，等待用戶批准 | 戰術規劃階段 | 3（中） |
| 2 | AskUserQuestion | 需要用戶決策，顯示選項 | 魔法師詢問 | 4（低） |
| 3 | 錯誤/警告 | 執行失敗，顯示錯誤訊息 | 敵人特殊攻擊 / 反噬傷害 | 2（高） |
| 4 | 權限請求 | 需要額外權限，請求授權 | 解鎖新能力確認 | 1（最高） |

**事件優先級**：權限請求 > 錯誤/警告 > Plan Mode > AskUserQuestion

當多個事件同時發生時，按優先級依序處理。

### 3.2 事件詳細規則

#### 事件 1：Plan Mode → 戰術規劃階段

```
觸發：Claude Code 調用 EnterPlanMode
        │
        ▼
戰鬥暫停，時間凍結
        │
        ▼
玩家傳送到「靜止之間」
（城鎮北端的獨立空間）
        │
        ▼
AI 展示計劃內容
        │
    ┌───┼───┐
    ▼   ▼   ▼
  批准  拒絕  修改
    │   │   │
    │   │   └──→ 重新規劃
    │   └──────→ 重新規劃
    ▼
返回戰鬥，獲得戰術 Buff（+10% 傷害，持續 3 回合）
```

- **靜止之間**：時間暫停的規劃室，概念類似「精神時光屋」
- 進入台詞：「時間在此靜止，唯有你的思緒在流動。」
- 離開台詞：「帶著你的計劃回到世界吧。」
- 批准計劃後獲得 Buff 獎勵

#### 事件 2：AskUserQuestion → 魔法師詢問

```
觸發：Claude Code 調用 AskUserQuestion
        │
        ▼
戰鬥暫停，彈出詢問面板
        │
        ▼
以 NPC 魔法師形式提問
（「這個敵人有兩個弱點，
  你想攻擊哪一個？」）
        │
        ▼
玩家選擇選項或輸入自定義答案
        │
        ▼
戰鬥繼續
```

- 呈現為魔法師在戰鬥中向玩家請求指引
- 戰鬥時間暫停直到回答
- 回答後繼續戰鬥流程

#### 事件 3：錯誤/警告 → 敵人特殊攻擊 / 反噬傷害

```
觸發：工具執行失敗、語法錯誤、權限錯誤、網絡錯誤
        │
        ▼
顯示為「技能反噬」或「敵人反擊」
        │
        ▼
反噬傷害 = 錯誤嚴重度 x 5
（連續錯誤傷害 x1.5 倍遞增）
        │
    ┌───┼───┐
    ▼   ▼   ▼
  重試  防禦  取消
  (MP) (HP)  (放棄)
```

- 錯誤嚴重度映射：語法錯誤 = 3、權限錯誤 = 5、網絡錯誤 = 2
- 重試消耗 MP，防禦減少 HP 損失，取消放棄操作
- 連續錯誤會增加傷害倍率，鼓勵謹慎操作

#### 事件 4：權限請求 → 解鎖新能力確認

```
觸發：執行需要額外權限的操作
     （如修改系統文件、Bash 危險命令）
        │
        ▼
彈出「力量借用」確認面板
        │
        ▼
顯示請求內容 + 風險等級
        │
    ┌───┴───┐
    ▼       ▼
  授予     拒絕
(解鎖能力) (操作受限)
```

- 風險等級用圖標表示（低/中/高）
- 授予 = 解鎖新能力，拒絕 = 操作受限
- 最高優先級，其他事件必須等待權限處理完成

### 3.4 並行操作檢測

```
偵測到 2+ 個工具同時執行
        │
        ▼
判定為「連擊魔法」(Combo Magic)
        │
        ▼
施加獎勵倍率：
  2 個並行 → EXP x1.2
  3 個並行 → EXP x1.5
  5+ 個並行 → EXP x2.0
```

### 3.5 通用規則

- **所有互動事件暫停戰鬥時間**：事件處理期間不消耗回合
- **原始功能完整保留**：RPG 包裝不改變 Claude Code 的核心行為
- **用戶選擇正確傳遞**：Plan Mode 的批准/拒絕、權限的授予/拒絕，都精確傳遞給底層

## 五、與其他系統的互動

### 5.1 與戰鬥系統 (3) 的互動

| 互動點 | 說明 |
|-------|------|
| 事件觸發 | 互動事件在戰鬥回合中觸發，暫停戰鬥流程 |
| 時間凍結 | 事件處理期間不消耗回合，不推進戰鬥進度 |
| 傷害整合 | 錯誤事件的反噬傷害使用戰鬥系統的 HP 機制 |
| 戰鬥日誌 | 所有互動事件記錄到戰鬥日誌（「觸發戰術規劃」「受到反噬傷害 15 點」） |
| 工具映射 | 每個 Claude Code 工具自動轉化為 RPG 魔法，記入戰鬥流程 |

### 5.2 與場景系統 (2) 的互動

| 互動點 | 說明 |
|-------|------|
| 靜止之間 | Plan Mode 觸發時，玩家傳送到靜止之間（場景系統管理的場所） |
| 場景切換 | 從戰鬥場景 → 靜止之間 → 返回戰鬥場景 |
| 模式切換 | 事件不改變場景模式（始終在戰鬥模式下處理） |

### 5.3 與地圖系統 (1) 的間接互動

|-------|------|

## 六、設計決策說明

### 為什麼用 RPG 包裝而不是直接顯示原始事件？

原始事件（Plan Mode 彈窗、錯誤訊息）會中斷 RPG 沉浸感。將它們包裝為戰鬥中的自然事件（戰術規劃、敵人攻擊），讓中斷變得合理，玩家不會感覺「出戲」。

### 為什麼 Plan Mode 需要獨立的空間（靜止之間）？

Plan Mode 涉及深度思考和計劃制定，需要一個安靜的環境。靜止之間提供「時間凍結」的概念，讓玩家理解：在這裡可以不受干擾地制定戰術，外面的戰鬥不會推進。

### 為什麼錯誤映射為反噬傷害而非敵人攻擊？

錯誤是「我方施法失敗」的結果，映射為「反噬」比「敵人攻擊」更符合因果關係。這也暗示玩家：錯誤是自己的行為導致的，需要改正。

### 為什麼權限請求優先級最高？

權限請求涉及安全決策（如授權修改系統文件）。如果被其他事件延後處理，可能導致操作阻塞或安全問題。最高優先級確保權限決策第一時間被處理。

### 為什麼所有事件都暫停戰鬥時間？

如果事件處理期間戰鬥繼續推進，玩家會被迫在時間壓力下做出決策。暫停時間確保玩家有充足時間思考（特別是 Plan Mode 和權限請求），不會因為匆忙而做出錯誤選擇。
