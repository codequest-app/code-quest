## 系統概述

### 核心概念

地圖系統為 Code Quest 提供具體的「空間感」，取代抽象的「模式」概念。

**問題**：
- 當前設計只有抽象的「探索模式」和「戰鬥模式」
- 商店系統設計了 7 個商店，但沒說明它們在哪裡
- 用戶可以在任何地方觸發任何操作，缺乏空間感和沉浸感

**解決方案**：
- 地圖 = 具體的場所（城鎮、野外、副本）
- 每個場所有明確的功能和限制
- 遭遇戰只在特定區域發生（傳統 RPG 規則）
- 用戶通過地圖導航在不同場所間移動

## 依賴關係

### 系統依賴

```
地圖系統 (L0 - 基礎層)
├─ 無直接依賴
└─ 被依賴於：
   ├─ 場景系統
   ├─ 商店系統
   ├─ Worktree 系統
   └─ 戰鬥系統
```

**說明**：
- 地圖系統是 L0 基礎系統，不依賴其他系統
- 提供空間結構和場所定義給其他系統使用

## 核心規則

### 規則 1：三大區域定義

#### 🏰 城鎮區域 (Town Zone)

**核心概念**：
- **安全區**：不會觸發戰鬥
- **服務中心**：提供各種開發輔助功能
- **社交空間**：與 AI 對話、管理資源

**子場所列表**：

```
城鎮
├─ 🏪 商業街 (Shopping District)
│  ├─ 🔮 技能商店
│  ├─ ⚒️ 工匠鋪
│  ├─ 📚 魔法圖書館
│  ├─ 🧙 傭兵公會
│  ├─ 🏆 寶物庫
│  ├─ 🎯 訓練場
│  └─ 💰 錢莊
│
├─ 🍺 酒館 (Tavern)
│  └─ AI 對話、日常討論、技術諮詢
│
├─ ⏸️ 靜止之間 (Stasis Chamber)
│  └─ Plan Mode 專屬空間、深度思考
│
├─ 🏛️ 公會大廳 (Guild Hall)
│  └─ Worktree 管理中心、平行時間線
│
└─ 🏠 玩家住所 (Player Home)
   └─ 休息、存檔、設定
```

**限制**：
- ❌ 戰鬥中無法進入城鎮
- ✅ 探索模式隨時可進入
- ✅ 可使用快捷鍵直接進入特定場所

---

#### 🌲 野外區域 (Wilderness Zone)

**核心概念**：
- **危險區域**：會隨機遭遇戰鬥
- **任務執行地**：代碼任務在此觸發戰鬥
- **資源豐富**：完成戰鬥獲得經驗值和金幣

**子區域**：

```
野外
├─ 🌳 森林 (Forest)
│  • 入門級任務（複雜度 5-8）
│  • 遭遇率：30%
│
├─ ⛰️ 山脈 (Mountains)
│  • 中級任務（複雜度 8-12）
│  • 遭遇率：50%
│
├─ 🏜️ 荒野 (Wasteland)
│  • 高級任務（複雜度 12-15）
│  • 遭遇率：70%
│
└─ 🌋 火山 (Volcano)
   • 專家級任務（複雜度 15+）
   • 遭遇率：90%
```

**遭遇戰觸發條件**：

```
觸發判定:
• 檢查當前區域是否為野外
• 檢查 Prompt 類型是否為任務型

複雜度判定:
• 分析 Prompt 複雜度

複雜度 ≥ 8:
• 創建戰鬥實例
• 傳入 Prompt、複雜度、區域資訊

複雜度 < 8:
• 在主 CLI 直接處理
• 不觸發戰鬥
```

**限制**：
- ❌ 戰鬥中無法進入城鎮（需先完成或逃跑）
- ✅ 可隨時返回城鎮（非戰鬥時）
- ✅ 可在野外使用道具和技能

---

#### 🏔️ 副本區域 (Dungeon Zone)

**核心概念**：
- **特定任務專屬**：特殊的複雜任務或劇情任務
- **完成前無法退出**：需要完成戰鬥才能離開
- **Boss 戰**：高難度敵人和特殊機制

**副本類型**：

```
副本列表
├─ 🐛 Bug 洞窟 (Bug Cave)
│  • 類型：Debug 任務專屬
│  • Boss：Bug 之王
│  • 特殊機制：需要追蹤錯誤日誌
│
├─ 🏗️ 架構迷宮 (Architecture Maze)
│  • 類型：架構重構任務專屬
│  • Boss：遺留代碼守護者
│  • 特殊機制：需要制定完整計劃
│
├─ 🧪 測試試煉場 (Testing Arena)
│  • 類型：測試覆蓋任務專屬
│  • Boss：測試覆蓋監督者
│  • 特殊機制：需要達到 100% 覆蓋率
│
└─ 🔒 安全地牢 (Security Dungeon)
   • 類型：安全漏洞修復任務
   • Boss：漏洞注入魔王
   • 特殊機制：需要通過安全檢查
```

**限制**：
- ❌ 進入後無法返回城鎮
- ❌ 無法在副本中創建新 Worktree（已綁定）
- ✅ 可以使用道具和召喚夥伴
- ✅ 可以在副本中進入靜止之間（Boss 戰前）

---

### 規則 2：場所限制規則

**操作限制表**：

| 操作 | 城鎮 | 野外 | 副本 | 戰鬥中 |
|-----|------|------|------|--------|
| 進入商店 | ✅ | ❌ | ❌ | ❌ |
| AI 對話（酒館） | ✅ | ✅ | ✅ | ⏸️ |
| 創建 Worktree（公會） | ✅ | ❌ | ❌ | ❌ |
| 進入靜止之間 | ✅ | ✅ | ✅ | ⏸️ |
| 觸發戰鬥 | ❌ | ✅ | ✅ | - |
| 使用道具 | ✅ | ✅ | ✅ | ✅ |
| 切換場所 | ✅ | ✅ | ❌ | ❌ |
| 休息恢復 | ✅ | ❌ | ❌ | ❌ |

說明：
- ✅ 允許
- ❌ 禁止
- ⏸️ 戰鬥暫停後可用

---

### 規則 3：場所轉換規則

```
轉換圖：

城鎮 ←→ 野外（雙向，隨時可切換，非戰鬥時）
城鎮 → 副本（單向，完成後自動返回城鎮）
野外 → 副本（單向，特定觸發）

戰鬥中：
❌ 無法切換場所
✅ 可以進入靜止之間（時間暫停）
✅ 可以在酒館對話（時間暫停）
```

---

### 規則 4：遭遇戰機制

**觸發邏輯流程圖**：

```
用戶輸入 Prompt
    ↓
檢查當前區域
    ↓
┌─────────────────────┐
│ 城鎮？              │
│ Yes → 不觸發戰鬥    │
│ No → 繼續           │
└─────────────────────┘
    ↓
┌─────────────────────┐
│ 副本？              │
│ Yes → 強制戰鬥      │
│ No → 繼續           │
└─────────────────────┘
    ↓
分析 Prompt 類型
    ↓
┌─────────────────────┐
│ 對話型？            │
│ Yes → 不觸發戰鬥    │
│ No → 繼續           │
└─────────────────────┘
    ↓
分析複雜度
    ↓
┌─────────────────────┐
│ 複雜度 < 8？        │
│ Yes → 主 CLI 執行   │
│ No → 觸發戰鬥       │
└─────────────────────┘
```

**遭遇率設定**：

```
各區域遭遇率:
• 森林: 30% 機率
• 山脈: 50% 機率
• 荒野: 70% 機率
• 火山: 90% 機率
```

---

### 規則 5：資源恢復規則

**區域資源規則**：

| 區域 | HP 恢復 | MP 恢復 | 說明 |
|------|---------|---------|------|
| 城鎮 | 2/秒 | 1/秒 | 快速恢復，安全區 |
| 野外 | 2/秒 | 1/秒 | 非戰鬥時恢復 |
| 副本 | 0/秒 | 0/秒 | 無自然恢復 |
| 戰鬥中 | 0/秒 | 0.1/秒 | 戰鬥狀態 |

## 系統整合

### 與 Shop System 整合

```
整合點：
├─ 商店位於城鎮的商業街
├─ 進入商店需要先導航到商業街區域
├─ 快捷鍵可直接進入（跳過導航動畫）
└─ 商店營業時間：探索模式（城鎮）

技術實現：
- MapManager.getCurrentLocation() === 'shopping_district'
- ShopManager.canAccessShop(shopId)
```

---

### 與 Scene System 整合

```
映射關係：
├─ 探索模式 → 城鎮區域 + 野外（非戰鬥）
├─ 戰鬥模式 → 野外（戰鬥中）+ 副本
└─ 場景切換觸發地圖轉換

Scene System 被 Map System 取代：
- 舊：抽象的「探索模式」
- 新：具體的「城鎮區域」
- 舊：抽象的「戰鬥模式」
- 新：具體的「野外/副本」
```

---

### 與 Battle System 整合

```
戰鬥觸發規則：
├─ 城鎮：❌ 禁止戰鬥
├─ 野外：✅ 根據複雜度觸發
├─ 副本：✅ 強制戰鬥
└─ 戰鬥中：鎖定當前區域

遭遇戰流程：
1. MapManager.getCurrentZone() // 'wilderness'
2. MapManager.checkEncounter(prompt)
3. BattleManager.startBattle(enemyData)
4. MapManager.lockLocation() // 戰鬥期間鎖定
```

---

### 與 Worktree System 整合

```
Worktree 管理中心 = 公會大廳：
├─ 只能在城鎮的公會大廳操作
├─ 野外遭遇戰自動創建 Worktree
├─ 戰鬥結束後在公會大廳處理合併
└─ 顯示所有活躍戰鬥和 Worktrees

限制規則：
- 戰鬥中：無法操作 Worktree
- 副本中：無法創建新 Worktree
```

---

### 與 Interactive Events 整合

```
事件場所映射：
├─ Plan Mode → 靜止之間（可從任何地方進入）
├─ AskUserQuestion → 戰鬥暫停（任何場所）
├─ 錯誤/警告 → 戰鬥反噬（戰鬥場所）
└─ 權限請求 → 彈窗（任何場所）

特殊規則：
- 靜止之間：時間暫停，外界不流動
- 暫停事件：保持當前場所狀態
```

## 設計決策

### 決策 1：為什麼需要地圖系統？

**問題**：
- 原設計只有抽象的「探索模式」和「戰鬥模式」
- 用戶缺乏空間感和沉浸感
- 功能分布不明確

**解決方案**：
- 提供具體的場所（城鎮、野外、副本）
- 每個場所有明確的功能和限制
- 符合傳統 RPG 玩家直覺

**優勢**：
- ✅ 空間感：用戶知道自己在哪裡
- ✅ 沉浸感：更像真實的 RPG 世界
- ✅ 直覺性：熟悉 RPG 的玩家一看就懂
- ✅ 擴展性：可輕鬆添加新場所

---

### 決策 2：城鎮為什麼是安全區？

**傳統 RPG 規則**：
- 城鎮 = 安全區（休息、補給、社交）
- 野外 = 危險區（冒險、戰鬥）
- 副本 = 挑戰區（特殊任務）

**Code Quest 應用**：
- 城鎮：用戶管理、購物、對話
- 野外：執行代碼任務、觸發戰鬥
- 副本：複雜任務、Boss 戰

**好處**：
- ✅ 分離關注點
- ✅ 降低認知負擔
- ✅ 符合用戶預期

---

### 決策 3：為什麼野外有子區域？

**目的**：
- 提供難度梯度
- 讓用戶選擇挑戰等級
- 增加探索樂趣

**設計**：
- 森林：新手友好（複雜度 5-8，遭遇率 30%）
- 山脈：中級挑戰（複雜度 8-12，遭遇率 50%）
- 荒野：高級挑戰（複雜度 12-15，遭遇率 70%）
- 火山：專家級（複雜度 15+，遭遇率 90%）

**好處**：
- ✅ 難度選擇：用戶可控制挑戰難度
- ✅ 視覺差異：不同區域有不同視覺風格
- ✅ 成長感：從森林到火山的進階感

---

### 決策 4：靜止之間的設計靈感

**概念來源**：
- 《龍珠》的精神時光屋
- 時間靜止的獨立空間
- 可以無限思考和規劃

**Code Quest 映射**：
- Plan Mode = 深度思考時間
- 外界時間暫停 = 不影響戰鬥進度
- 灰色調 = 時間凍結的視覺表現

**實現**：
- 按 P 鍵或主動進入靜止之間建築
- 時間減速動畫 + 灰階濾鏡
- Plan Mode 在此展開
- 完成後時間恢復流動

**好處**：
- ✅ 視覺化 Plan Mode
- ✅ 提供安靜思考的空間
- ✅ 增加 RPG 氛圍

---

### 決策 5：副本系統的必要性

**為什麼需要副本**？

傳統方式（無副本）：
- 所有任務都在野外執行
- 缺乏特殊性
- 難以實現劇情戰鬥

副本系統（有副本）：
- 特定任務專屬區域
- 完成前無法退出（強制完成）
- 可以有多階段 Boss 戰
- 劇情感更強

**應用場景**：
- Bug 洞窟：專門修復嚴重 Bug
- 架構迷宮：大型重構任務
- 測試試煉場：完整測試覆蓋
- 安全地牢：安全漏洞修復

**好處**：
- ✅ 特殊任務有儀式感
- ✅ 強制完成增加挑戰性
- ✅ Boss 戰更有成就感

---

### 決策 6：快捷鍵設計

**為什麼需要快捷鍵**？

問題：
- 每次都要走路很慢
- 頻繁訪問的場所應該快速進入

解決方案：
- 數字鍵 1-7：直達 7 個商店
- H 鍵：直達酒館
- G 鍵：直達公會大廳
- P 鍵：進入靜止之間
- B 鍵：返回城鎮

**實現**：
- 快捷鍵 → 傳送動畫（閃光 0.3 秒）
- 步行進入 → 淡入動畫（0.5 秒）

**好處**：
- ✅ 提高效率
- ✅ 降低重複操作
- ✅ 保留步行選項（沉浸感）

## 總結

地圖系統為 Code Quest 提供：

**空間結構**：
- 三大區域（城鎮、野外、副本）
- 14+ 個具體場所
- 明確的功能分布

**遊戲規則**：
- 城鎮 = 安全區
- 野外 = 戰鬥區
- 副本 = 挑戰區

**用戶體驗**：
- ✅ 符合 RPG 直覺
- ✅ 提供空間感和沉浸感
- ✅ 功能分布清晰
- ✅ 擴展性強

地圖系統是整個 RPG-CLI 的空間基礎，所有其他系統都建立在此之上。
