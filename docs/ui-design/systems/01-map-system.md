# L0 地圖系統

**版本**: v2.0
**更新日期**: 2026-02-06

---

## 一、系統概述

地圖系統是 CodeLand 世界的空間基礎，定義了冒險者所能前往的所有場所及其空間關係。整個世界由三大區域組成：**城鎮**（安全區）、**野外**（危險區）、**副本**（Boss 區）。

在系統架構中，地圖系統位於 **L0 基礎層**，是唯一沒有任何依賴的系統。它為所有需要「場所」概念的系統提供空間結構，是整個 Code Quest 的根基。

---

## 二、依賴關係

```
依賴：無（根基系統）

被依賴：
  ├─ 場景系統 (2)    — 需要知道當前區域來決定場景規則
  ├─ 商店系統 (6)    — 商店只存在於城鎮區域
  └─ Worktree 系統 (8) — Worktree 操作限定在城鎮區域
```

| 方向 | 系統 | 關係說明 |
|------|------|---------|
| 被依賴 | 場景系統 (2) | 場景系統根據玩家所在區域決定可觸發的場景類型 |
| 被依賴 | 商店系統 (6) | 7 個商店的物理位置位於城鎮商業街 |
| 被依賴 | Worktree 系統 (8) | 公會大廳（城鎮內）是 Worktree 管理的唯一場所 |

## 三、核心規則

### 3.1 城鎮區域（Town Zone）— 安全區

城鎮是冒險者的根據地，**不會觸發任何戰鬥**。包含 5 個場所：

| 場所 | 功能 | 限制 |
|------|------|------|
| 商業街 | 7 個商店（技能商店、工匠鋪、魔法圖書館、傭兵公會、寶物庫、訓練場、錢莊） | 戰鬥中無法進入 |
| 酒館 | AI 對話、日常討論、技術諮詢 | 副本中無法進入 |
| 靜止之間 | Plan Mode 專屬空間，深度規劃 | 任何場景均可進入 |
| 公會大廳 | Worktree 管理中心 | 戰鬥中無法操作 |
| 玩家住所 | 休息（完全恢復 HP/MP）、存檔、設定 | 無特殊限制 |

**商業街包含 7 個商店**：

```
商業街
├─ 技能商店     — 查看、解鎖、升級技能
├─ 工匠鋪       — 創建自定義技能（6 步驟引導）
├─ 魔法圖書館   — 瀏覽和安裝 MCP 工具
├─ 傭兵公會     — 管理戰鬥夥伴（Subagents）
├─ 寶物庫       — 成就、召喚獸圖鑑、收集品
├─ 訓練場       — 教學、模擬戰鬥（不消耗資源）
└─ 錢莊         — 資源統計、AI 模型成本追蹤
```

### 3.2 野外區域（Wilderness Zone）— 危險區

野外是代碼任務的執行地，**會觸發遭遇戰**。包含 4 個子區域，難度遞增：

| 子區域 | 遭遇率 | 複雜度範圍 | 建議等級 | 定位 |
|--------|--------|-----------|---------|------|
| 森林（Forest） | 30% | 5-8 | Lv.5-10 | 入門級任務區 |
| 山脈（Mountains） | 50% | 8-12 | Lv.8-15 | 中級任務區 |
| 荒原（Wasteland） | 70% | 12-15 | Lv.12-20 | 高級任務區 |
| 火山（Volcano） | 90% | 15+ | Lv.18+ | 專家級任務區 |

**戰鬥觸發條件**：必須同時滿足以下條件：
1. Prompt 類型為任務型（包含動作關鍵字）
2. 複雜度分析結果 ≥ 8
3. 玩家位於野外區域

### 3.3 副本區域（Dungeon Zone）— Boss 區

副本是特定複雜任務的專屬區域。**進入後無法中途退出**，必須完成戰鬥才能離開。

| 副本 | 類型 | Boss | 建議等級 |
|------|------|------|---------|
| 蟲洞（Bug Cave） | Debug 任務 | Bug 之王 | Lv.8-12 |
| 架構迷宮（Architecture Maze） | 架構重構 | 遺留代碼守護者 | Lv.12-18 |
| 測試試煉場（Testing Arena） | 測試覆蓋 | 測試覆蓋監督者 | Lv.10-15 |
| 安全地城（Security Dungeon） | 安全漏洞修復 | 漏洞注入魔王 | Lv.15-20 |

### 3.4 移動規則

```
城鎮 ←──→ 野外     雙向自由移動（非戰鬥時）
城鎮  ──→ 副本     單向進入（特定 Prompt 或主動選擇）
野外  ──→ 副本     單向進入（特定 Prompt 觸發）
副本  ──→ 城鎮     戰鬥完成後自動返回（不可手動）

戰鬥中移動限制：
  ✗ 無法切換區域
  ✓ 可進入靜止之間（時間暫停）
```

### 3.5 場所操作限制總表

| 操作 | 城鎮 | 野外 | 副本 | 戰鬥中 |
|------|------|------|------|--------|
| 進入商店 | 可 | 不可 | 不可 | 不可 |
| 創建 Worktree | 可 | 不可 | 不可 | 不可 |
| 觸發戰鬥 | 不可 | 可 | 強制 | 不可 |
| 休息恢復 | 可 | 不可 | 不可 | 不可 |
| AI 對話 | 可 | 可 | 可 | 暫停後可 |
| 進入靜止之間 | 可 | 可 | 可 | 暫停後可 |
| 切換場所 | 可 | 可 | 不可 | 不可 |

## 五、與其他系統的互動

### 5.1 地圖 → 場景系統 (2)

場景系統依據玩家所在區域決定場景規則：

```
玩家位置（地圖系統提供）
    │
    ├─ 城鎮 → 場景系統：強制探索模式，不允許戰鬥觸發
    ├─ 野外 → 場景系統：允許遭遇判定，根據 Prompt 決定是否戰鬥
    └─ 副本 → 場景系統：強制戰鬥模式，直到副本完成
```

### 5.2 地圖 → 商店系統 (6)

商店只存在於城鎮的商業街，地圖系統提供商店的存在場所：

- 玩家不在城鎮時，商店系統入口不可用
- 商店系統需要查詢地圖系統確認玩家位置

### 5.3 地圖 → Worktree 系統 (8)

Worktree 管理功能只在城鎮的公會大廳提供：

- 創建 / 切換 / 合併 Worktree 均須在公會大廳操作
- 戰鬥中的 Worktree 為自動建立，不受場所限制
- 查看 Worktree 列表在戰鬥中也可（唯讀）

## 六、設計決策說明

### 6.1 為什麼用具體「空間」而非抽象「模式」？

**問題**：早期設計中商店系統有 7 個商店，卻沒有物理位置承載。抽象的「模式切換」讓用戶難以理解自己在哪裡、能做什麼。

**決策**：建立完整的空間結構（城鎮 / 野外 / 副本），每個功能都有對應的物理場所。商業街承載 7 個商店、公會大廳承載 Worktree 管理、酒館承載日常對話。

**效果**：用戶在空間中移動，直覺地理解可用操作。「去商業街買技能」比「切換到技能管理模式」更符合 RPG 體驗。

### 6.2 為什麼遭遇戰只在特定區域觸發？

**問題**：如果任何地方都能觸發戰鬥，用戶在管理技能或規劃時會被意外打斷。

**決策**：遵循傳統 RPG 規則 — 城鎮絕對安全、野外有遭遇、副本強制戰鬥。

**效果**：用戶可預期何時安全、何時危險。想安靜管理就留在城鎮，想挑戰任務就去野外。

### 6.3 為什麼副本不能中途退出？

**問題**：副本對應高複雜度的完整任務（如全面重構、安全修復），中途放棄會導致不完整的代碼狀態。

**決策**：副本進入後必須完成才能離開，但允許在 Boss 戰前進入靜止之間規劃。

**效果**：確保複雜任務的完整性，同時提供戰術規劃的喘息空間。
