## 文件目的

本文件定義 Code Quest 的整體系統架構，說明 11 個核心系統的依賴關係、實作優先級和跨系統整合策略。為 UI 開發者提供清晰的系統結構視圖。

## 系統總覽

Code Quest 由 11 個核心系統組成，分為 5 個依賴層級（L0-L4），從基礎功能到呈現層逐層構建。

## 依賴層級詳解

### L0: 基礎層（Foundation Layer）

**包含系統**: 地圖系統(1)、場景系統(2)、商店系統(6)、Worktree 系統(8)

**特性**:
- 無依賴或僅依賴同層/下層系統
- 提供核心功能基礎
- 最先實作

**地圖系統 - 唯一無依賴系統**:
```
功能:
• 3 大區域: 城鎮、野外、副本
• 14+ 場所定義
• 遭遇戰觸發規則

UI 重點:
• 2.5D 俯視地圖視圖
• 場所進入/退出動畫
• 場所內部界面設計
```

**場景系統 - 雙場景架構**:
```
功能:
• 探索模式（非戰鬥）
• 戰鬥模式（RPG 戰鬥）
• 智能場景切換
• 資源恢復規則

UI 重點:
• 模式切換動畫
• 探索 UI vs 戰鬥 UI
• 主菜單系統
```

**商店系統 - 7 個商店**:
```
功能:
• 技能商店、工匠鋪、魔法圖書館
• 傭兵公會、寶物庫、訓練場、錢莊
• Skill Forge 六步驟創建流程

UI 重點:
• 商業街總覽界面
• 7 個獨立商店 UI
• 購買/升級流程
```

**Worktree 系統 - 平行世界**:
```
功能:
• Git Worktree → RPG 平行世界
• 5 種世界類型
• 創建/切換/合併操作

UI 重點:
• 時空管理器介面
• 世界列表視圖
• 時間線圖/看板視圖
```

### L1: 核心戰鬥層（Core Battle Layer）

**包含系統**: 戰鬥系統(3)

**特性**:
- 依賴場景系統
- 所有戰鬥相關系統的基礎
- 最關鍵的樞紐系統

**戰鬥系統 - RPG 戰鬥核心**:
```
功能:
• 敵人生成（複雜度分析）
• 回合制戰鬥流程
• 傷害計算與相性系統
• 獎勵系統

UI 重點:
• 戰鬥主畫面
• 敵人展示區
• 戰鬥日誌
• 行動選單
• 傷害數字動畫
• 勝利/失敗畫面
```

**敵人類型與 UI 對應**:
```
code-task     💻  藍色調
bug-hunt      🐛  紅色調
architecture  🏰  紫色調
documentation 📜  黃色調
testing       🧪  綠色調
optimization  ⚡  橙色調
general       ❓  灰色調
```

## 核心依賴關係說明

### 1. 場景系統的核心地位

場景系統雖然在 L0，但被多個系統依賴：

**依賴場景系統的系統**:
```
戰鬥系統(3)    → 判斷是否在戰鬥模式
商店系統(6)    → 只能在探索模式訪問
互動事件系統(7) → Plan Mode 需要場景暫停
Worktree 系統(8)→ 只能在探索模式操作
```

**為什麼場景系統如此重要**:
- 定義全局狀態（探索 vs 戰鬥）
- 控制資源恢復規則
- 限制可用操作範圍
- 管理場景切換動畫

---

### 2. 戰鬥系統的樞紐地位

戰鬥系統被 5 個系統依賴，是最關鍵的樞紐：

**依賴戰鬥系統的系統**:
```
夥伴系統(4)      → 在戰鬥中協同
召喚獸系統(5)    → 戰鬥中召喚
互動事件系統(7)  → 戰鬥中事件處理
非同步戰鬥系統(9)→ 並發戰鬥管理
多模型整合系統(10)→ 戰鬥中 AI 處理
```

**戰鬥系統提供的核心接口**:
```typescript
interface BattleSystem {
  // 戰鬥生命週期
  startBattle(prompt: string): Battle;
  endBattle(battleId: string): BattleResult;

  // 戰鬥行動
  useSkill(skillId: string): void;
  summonCompanion(companionId: string): void;
  useSummon(summonId: string): void;

  // 戰鬥狀態
  getCurrentBattle(): Battle | null;
  getBattleLog(): LogEntry[];
  getEnemyInfo(): Enemy;
}
```

---

### 3. 地圖系統的基礎地位

地圖系統是唯一無依賴的系統，是整個架構的基石：

**地圖系統提供**:
```
空間結構      → 城鎮、野外、副本
場所定義      → 酒館、商業街、公會大廳等
遭遇戰規則    → 在野外觸發戰鬥
場所限制      → 戰鬥中無法進入商店
```

**被地圖系統影響的系統**:
```
場景系統(2)   → 探索模式在城鎮，戰鬥模式在野外
商店系統(6)   → 商店位於城鎮商業街
Worktree 系統(8)→ 公會大廳是 Worktree 管理中心
```

---

### 4. 特殊依賴關係

**非同步戰鬥的雙重依賴**:
```
戰鬥系統(3)  → 執行戰鬥邏輯
Worktree 系統(8)→ 自動創建隔離環境

工作流程:
1. 偵測到複雜任務
2. 自動創建 Worktree (依賴 8)
3. 在 Worktree 中啟動戰鬥 (依賴 3)
4. 戰鬥結束後合併 Worktree (依賴 8)
```

**多模型整合的跨層依賴**:
```
戰鬥系統(3)  → 執行 AI 任務
商店系統(6)  → 錢莊顯示成本

工作流程:
1. 戰鬥中使用 AI (依賴 3)
2. 追蹤成本
3. 在錢莊查看成本 (依賴 6)
4. 調整模型選擇
```

## 跨系統整合策略

### 1. 地圖-場景-戰鬥整合鏈

**整合流程**:
```
[地圖系統]
  ↓ 定義場所和區域
[場景系統]
  ↓ 判斷當前模式（探索/戰鬥）
[戰鬥系統]
  ↓ 在野外觸發戰鬥

使用場景:
1. 用戶在城鎮（地圖系統）
2. 當前為探索模式（場景系統）
3. 用戶前往野外（地圖系統）
4. 輸入任務型 Prompt
5. 場景系統判斷→切換到戰鬥模式
6. 戰鬥系統啟動→生成敵人
```

**UI 整合點**:
```
地圖 UI 顯示
  ├─ 當前場所標示
  ├─ 模式指示（探索/戰鬥）
  └─ 可用操作提示

場景切換動畫
  ├─ 城鎮 → 野外: 漸變 0.8s
  ├─ 探索 → 戰鬥: 震動 + 閃紅 1.0s
  └─ 戰鬥 → 探索: 勝利動畫 2s

戰鬥 UI 覆蓋
  ├─ 保留地圖背景（模糊）
  ├─ 戰鬥主畫面置頂
  └─ 戰鬥結束後移除
```

## 系統間通訊協議

### 事件總線（Event Bus）

**跨系統事件**:
```typescript
// 場景切換事件
EventBus.emit('scene:change', {
  from: 'exploration',
  to: 'battle',
  data: { enemy, complexity }
});

// 戰鬥開始事件
EventBus.emit('battle:start', {
  battleId: string,
  enemy: Enemy,
  location: Location
});

// 技能使用事件
EventBus.emit('skill:use', {
  skillId: string,
  target: 'enemy' | 'self',
  cost: { mp: number }
});

// 資源變化事件
EventBus.emit('resource:change', {
  type: 'hp' | 'mp' | 'exp' | 'gold',
  value: number,
  change: number
});
```

### 共享狀態（Global State）

**核心狀態數據**:
```typescript
interface GlobalState {
  // 場景狀態
  currentScene: 'exploration' | 'battle';
  currentZone: 'town' | 'wilderness' | 'dungeon';
  currentLocation: string;

  // 戰鬥狀態
  activeBattle: Battle | null;
  backgroundBattles: Battle[];

  // 玩家狀態
  player: {
    level: number;
    hp: number;
    maxHp: number;
    mp: number;
    maxMp: number;
    exp: number;
    gold: number;
  };

  // 系統狀態
  worktrees: Worktree[];
  companions: Companion[];
  summons: Summon[];
  skills: Skill[];
}
```
