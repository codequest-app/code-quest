## 關鍵依賴說明

### 地圖系統的基礎地位

地圖系統是**唯一無依賴**的系統，定義了 CodeLand 的空間結構（城鎮 / 野外 / 副本）。所有需要「場所」概念的系統都直接或間接依賴它。

### 場景系統的核心地位

場景系統區分「探索模式」與「戰鬥模式」，被多個系統依賴：
- **戰鬥系統** → 判斷是否進入戰鬥
- **商店系統** → 只能在探索模式訪問
- **Worktree 系統** → 只能在探索模式操作
- **互動事件系統** → Plan Mode 需要進入「靜止之間」

### 戰鬥系統的樞紐地位

戰鬥系統被 **5 個系統**依賴，是架構樞紐。所有戰鬥擴展（夥伴、召喚獸、互動事件）和高級功能（非同步戰鬥、多模型）都以它為基礎。

### 特殊關係

- **夥伴 vs 召喚獸**：並列關係，都依賴戰鬥系統但互相獨立。夥伴 = 持續支援（整場戰鬥），召喚獸 = 爆發支援（1-3 回合）
- **非同步戰鬥的雙重依賴**：同時依賴戰鬥系統（邏輯）和 Worktree 系統（隔離環境）
- **多模型整合的跨層依賴**：依賴戰鬥系統（執行任務）和商店系統（錢莊顯示成本）

## 建議實作順序

### 階段 1：基礎層（L0）

```
1. 地圖系統 (1)     ← 無依賴，最先實作
2. 場景系統 (2)     ← 依賴地圖
3. 商店系統 (6)     ← 依賴地圖 + 場景
4. Worktree 系統 (8) ← 依賴地圖 + 場景
```

### 階段 2：核心戰鬥（L1）

```
5. 戰鬥系統 (3)     ← 依賴場景
```

### 階段 3：戰鬥擴展（L2）

```
6. 夥伴系統 (4)     ← 依賴戰鬥（可與 7、8 並行）
7. 召喚獸系統 (5)   ← 依賴戰鬥（可與 6、8 並行）
8. 互動事件系統 (7) ← 依賴戰鬥 + 場景
```

### 階段 4：高級功能（L3）

```
9.  多模型整合 (10)  ← 依賴戰鬥 + 商店
10. 非同步戰鬥 (9)   ← 依賴戰鬥 + Worktree
```

### 階段 5：呈現層（L4）

```
11. UI 互動系統 (11) ← 依賴所有系統，最後實作
```

## 系統一句話速覽

| # | 系統 | 一句話說明 |
|---|------|-----------|
| 1 | 地圖系統 | 定義 CodeLand 的空間結構：城鎮（安全）、野外（危險）、副本（Boss） |
| 2 | 場景系統 | 區分探索模式與戰鬥模式，管理場景切換與資源規則 |
| 3 | 戰鬥系統 | 將 AI 對話過程視覺化為回合制 RPG 戰鬥 |
| 4 | 夥伴系統 | 將 Subagent 映射為全程參戰、可成長的戰鬥夥伴 |
| 5 | 召喚獸系統 | 提供關鍵時刻的短暫爆發型強力支援 |
| 6 | 商店系統 | 將 Skills/MCP/Subagent 管理 RPG 化為 7 個商店 |
| 7 | 互動事件系統 | 處理 Plan Mode、錯誤、權限請求等需要用戶互動的事件 |
| 8 | Worktree 系統 | 將 Git Worktree 管理 RPG 化為「平行世界」 |
| 9 | 非同步戰鬥系統 | 複雜任務後台執行，自動建立獨立 Worktree，用戶不阻塞 |
| 10 | 多模型整合系統 | 支持 Claude/Gemini 多模型，智能路由與成本追蹤 |
| 11 | UI 互動系統 | 定義所有互動細節、動畫效果、無障礙設計的完整 UI 規範 |
