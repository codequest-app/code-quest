## 設計理念

### 核心概念

```
【外觀】100% 傳統 DQ 風格
  - 全屏單一畫面
  - 菜單式界面
  - 傳統邊框和配色

【功能】現代多任務管理
  - 支援多 AI 並行戰鬥
  - 快速切換查看
  - 實時狀態監控

【平衡點】用 DQ 的方式實現現代功能
```

### 設計目標

1. **保持 DQ 風格純度** ⭐⭐⭐⭐⭐
   - 單一焦點、全屏沉浸
   - 菜單驅動操作
   - 傳統 JRPG 視覺美學

2. **支援多戰鬥並行** ⭐⭐⭐⭐
   - 多個 AI Agent 同時執行任務
   - 背景戰鬥自動進行
   - 隨時查看任意戰鬥狀況

3. **便利的管理方式** ⭐⭐⭐⭐
   - 快速切換菜單（DQ 風格彈窗）
   - 實時狀態監控
   - 緊急事件通知

---

## 核心架構

### 三方案對比

| 方案 | DQ 純度 | 切換便利性 | 視覺風格 | 最終選擇 |
|------|---------|-----------|---------|---------|
| 方案 2：派遣系統 | ⭐⭐⭐⭐⭐ | ⭐⭐ | 傳統 JRPG | ✅ 基礎 |
| 方案 3：多窗口 PIP | ⭐ | ⭐⭐⭐⭐ | RTS 風格 | ❌ |
| 方案 4：Tab 系統 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Web 應用 | ❌ |
| **混合方案** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | **DQ + 便利** | ✅ **採用** |

### 混合方案架構

```
【核心】方案 2 的派遣系統
  - 全屏主畫面（地圖/對話/戰鬥）
  - 背景任務不可見（只顯示小圖標）
  - 通過菜單系統管理

【增強】DQ 風格的快速切換菜單
  - 點擊右下角圖標彈出菜單
  - 菜單內顯示所有戰鬥卡片
  - 選擇後全屏切換到該戰鬥

【結果】既保持 DQ 感，又方便切換
```

## 技術實現

### React 組件結構

```tsx
// 主應用結構
<GameContainer>
  {/* Layer 1: 主畫面 */}
  <MainScreen mode={currentMode}>
    {currentMode === 'map' && <MapExploration />}
    {currentMode === 'dialogue' && <DialogueMode />}
    {currentMode === 'battle' && <BattleScreen battleId={activeBattle} />}
  </MainScreen>

  {/* Layer 2: 狀態欄（固定在底部）*/}
  <StatusBar>
    <PlayerStats hp={hp} mp={mp} gold={gold} level={level} />
    <BattleIndicator
      count={activeBattles.length}
      hasUrgent={hasUrgentBattles}
      onClick={() => setShowQuickSwitch(true)}
    />
  </StatusBar>

  {/* Layer 3: 快速切換菜單（彈窗）*/}
  {showQuickSwitch && (
    <QuickSwitchMenu
      battles={activeBattles}
      onSelect={switchToBattle}
      onClose={() => setShowQuickSwitch(false)}
    />
  )}

  {/* Layer 4: 通知系統 */}
  {notifications.map(notif => (
    <NotificationDialog
      key={notif.id}
      type={notif.type}
      message={notif.message}
      onAction={notif.action}
    />
  ))}
</GameContainer>
```

### 狀態管理

```typescript
interface GameState {
  // 當前焦點
  currentMode: 'map' | 'dialogue' | 'battle';
  activeBattle: string | null;

  // 地圖狀態
  mapState: MapState;

  // 對話狀態
  dialogueState: DialogueState;

  // 所有戰鬥
  battles: Map<string, BattleState>;

  // UI 狀態
  showQuickSwitch: boolean;
  notifications: Notification[];
}

interface BattleState {
  id: string;
  aiAgent: 'opus' | 'sonnet' | 'haiku';
  enemy: Enemy;
  progress: number; // 0-100
  status: 'active' | 'paused' | 'completed' | 'failed';
  needsUserInput: boolean;
  isInWorktree: boolean;
  worktreePath?: string;
  startTime: number;
  estimatedDuration: number;
}
```

```typescript
  // 全局
  'Tab': () => toggleQuickSwitch(),
  'M': () => openMainMenu(),
  'Escape': () => closeMenuOrGoBack(),
  'B': () => goBack(),
  'C': () => switchToDialogue(),

  // 在快速切換菜單中
  'ArrowUp': () => selectPrevBattle(),
  'ArrowDown': () => selectNextBattle(),
  'Enter': () => confirmSelection(),
  'A': () => confirmSelection(),
  'X': () => abortSelectedBattle(),

  // 在戰鬥畫面中
  'Space': () => togglePause(),
};

function useKeyboardShortcuts() {
  useEffect(() => {
    const handler = (e: KeyboardEvent) => {
      const action = SHORTCUTS[e.key];
      if (action) {
        e.preventDefault();
        action();
      }
    };

    window.addEventListener('keydown', handler);
    return () => window.removeEventListener('keydown', handler);
  }, []);
}
```

## 關鍵設計決策記錄

### 為什麼選擇混合方案？

**問題**：需要在「DQ 風格純度」和「多戰鬥管理便利性」間平衡

**考慮的方案**：
1. 純派遣系統 - DQ 純度最高，但切換不便
2. PIP 多窗口 - 切換方便，但完全不像 DQ
3. Tab 系統 - 折衷，但 Tab 欄不是遊戲元素
4. 混合方案 - DQ 風格的快速切換

**最終選擇**：混合方案

**理由**：
- ✅ 保持 100% DQ 視覺風格（全屏、菜單式）
- ✅ 提供便利的切換方式（快速菜單）
- ✅ 背景任務不干擾主畫面
- ✅ 符合玩家的 JRPG 操作習慣

---

### 為什麼用彈窗而非 Tab？

**問題**：如何讓用戶快速切換戰鬥？

**Tab 方案的問題**：
- ❌ Tab 欄是 Web 應用元素，不是遊戲元素
- ❌ 視覺上會提醒玩家「這是網頁」
- ❌ 破壞 DQ 的沉浸感

**彈窗方案的優勢**：
- ✅ DQ 經典的對話框設計
- ✅ 符合傳統 JRPG 的菜單操作
- ✅ 保持沉浸感

---

### 為什麼背景戰鬥不可見？

**問題**：是否要實時顯示所有戰鬥？

**傳統 DQ 的設計哲學**：
- 單一焦點 - 同一時間只做一件事
- 不會有多個戰鬥同時可見
- 保持畫面乾淨、專注

**我們的選擇**：
- ✅ 背景戰鬥只顯示小圖標
- ✅ 通過菜單查看詳情
- ✅ 保持 DQ 的單一焦點設計
