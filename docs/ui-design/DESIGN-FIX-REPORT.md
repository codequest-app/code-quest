**修改文件**:


### ✅ 問題 #9: 術語統一

**修正方案**: 明確定義術語層級

**修改文件**:
- ✅ references/glossary.md

**新增內容**:
```
術語定義：
1. 區域 (Zone)：地圖空間
   - 城鎮區域、野外區域、副本區域

2. 場所 (Location)：區域內的具體地點
   - 酒館、商業街、森林、山脈

3. 模式 (Mode)：系統狀態
   - 探索模式、戰鬥模式

使用原則：
- 談地圖用「區域/場所」
- 談狀態用「模式」
- 避免使用「場景」（容易混淆）
```

### ✅ 問題 #14: 失敗處理機制

**修正方案**: 添加完整失敗設計

**修改文件**:
- ✅ systems/03-battle-system.md

**新增內容**:
```
失敗處理機制：

【失敗條件】
- 玩家 HP = 0

【失敗後果】
1. 戰鬥立即結束
2. 不獲得經驗值和金幣
3. 損失 10% 當前金幣（死亡懲罰）
4. 異步戰鬥的 Worktree 保留（可稍後重試或放棄）
5. 玩家 HP 恢復到 50%
6. 玩家 MP 恢復到 30%

【復活機制】
- 自動復活（無需道具）
- 傳送回城鎮住所
- 可重新挑戰相同任務（不重置任務進度）

【Worktree 處理】
- 主目錄戰鬥失敗：無 Worktree 影響
- 異步戰鬥失敗：Worktree 保留，用戶選擇：
  - 重試：繼續使用該 Worktree
  - 放棄：刪除 Worktree
  - 保留：Worktree 留在列表中稍後處理
```
