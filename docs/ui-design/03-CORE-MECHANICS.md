## 資源系統

### HP/MP 系統

#### 資源定義

```
玩家資源:
├─ HP (生命值)
│  ├─ 當前值 / 最大值
│  ├─ 初始: 100 / 100
│  └─ 每級增加: +20
│
└─ MP (魔力值)
   ├─ 當前值 / 最大值
   ├─ 初始: 100 / 100
   └─ 每級增加: +15
```

#### 資源恢復規則

| 模式 | HP 恢復 | MP 恢復 | 說明 |
|-----|---------|---------|------|
| 探索模式 | 2 HP/秒 | 1 MP/秒 | 快速恢復 |
| 戰鬥模式 | 0 HP/秒 | 0.1 MP/秒 | 極慢恢復 |
| 休息（住所） | 即時全滿 | 即時全滿 | 立即恢復 |

#### HP/MP 條 UI 設計

```
HP 條（綠色 → 黃色 → 紅色）:
┌────────────────────────────┐
│ ❤️ HP: 85/100              │
│ ████████░░░░░░░░░░  85%   │
└────────────────────────────┘

顏色變化:
• HP > 70%: #00ff00 (綠色)
• HP 50-70%: #ffff00 (黃色)
• HP < 50%: #ff6600 (橙色)
• HP < 20%: #ff0000 (紅色，閃爍)

MP 條（青色）:
┌────────────────────────────┐
│ ⚡ MP: 65/100              │
│ ████████████░░░░░░  65%   │
└────────────────────────────┘

顏色:
• 全程: #00ccff (青色)
• MP < 20%: #ff9900 (橙色警告)
```

#### 資源消耗視覺反饋

**MP 消耗動畫**:
```
使用技能時:
1. MP 條快速減少（0.3s）
2. 數字跳動 -30 MP（紅色）
3. 粒子效果從 MP 條飛向技能圖標
4. 技能圖標閃光
```

**HP 受損動畫**:
```
受到傷害時:
1. 畫面震動（輕微）
2. HP 條減少（0.5s）
3. 傷害數字彈出（紅色，從玩家位置）
4. 玩家頭像閃紅（0.2s）
5. HP < 30% 時播放警告音
```

### 經驗值與等級系統

#### 經驗值公式

```
升級所需經驗值:
基礎值 × (成長倍率 ^ (等級 - 1))

範例:
Lv.1 → Lv.2: 100 EXP
Lv.2 → Lv.3: 150 EXP
Lv.3 → Lv.4: 225 EXP
Lv.4 → Lv.5: 337 EXP
Lv.5 → Lv.6: 506 EXP
Lv.10 → Lv.11: 3,834 EXP
```

## 戰鬥機制

### 敵人生成系統

#### 複雜度計算

```
複雜度評分因素:

1. 長度因素 (0-3分):
   • 超過 200 字: +3分
   • 超過 100 字: +2分
   • 其他: +1分

2. 關鍵字複雜度:
   重量級關鍵字 (+2分):
   • architecture, 架構
   • refactor, 重構
   • optimize, 優化
   • design pattern, 設計模式

   中量級關鍵字 (+1分):
   • implement, 實作
   • create, 創建
   • review, 審查
   • test, 測試

3. 多步驟任務 (+2分):
   包含 "and", "then", "after", "before", "以及", "然後"

4. 技術棧複雜度 (+1分/個):
   檢測: React, Vue, Node, Python, Java, TypeScript

5. 最終等級計算:
   等級 = 最小值(15, 最大值(1, 向下取整(分數 / 2) + 1))
```

#### 敵人等級與難度

| 等級 | 分數範圍 | 難度 | 特性 |
|-----|---------|------|------|
| 1-3 | 0-6 | Simple | 無反擊 |
| 4-7 | 7-14 | Medium | 一般任務 |
| 8-12 | 15-24 | Hard | 有反擊 |
| 13-15 | 25+ | Boss | 多階段 |

#### 敵人 HP 計算

```
最大HP = 等級 × 基礎值 × HP倍率

範例:
Lv.5 code-task:   500 HP (5 × 100 × 1.0)
Lv.8 bug-hunt:    1200 HP (8 × 100 × 1.5)
Lv.12 architecture: 2400 HP (12 × 100 × 2.0)
Lv.15 optimization: 2700 HP (15 × 100 × 1.8)
```

### 傷害計算系統

#### 基礎傷害公式

```
基礎傷害 = 固定值 + (技能MP消耗 × 倍數) + (玩家等級 × 倍數)

範例:
• Lv.10 玩家使用 30 MP 技能:
  基礎傷害 = 100 + (30 × 3) + (10 × 10) = 290

• Lv.15 玩家使用 50 MP 技能:
  基礎傷害 = 100 + (50 × 3) + (15 × 10) = 400
```

#### 相性倍率系統

```
相性表:

code-generator:
• code-task: ×1.2 (有效)
• documentation: ×1.1 (有效)
• bug-hunt: ×0.8 (效果不佳)
• testing: ×0.9 (效果不佳)

code-reviewer:
• bug-hunt: ×1.5 (非常有效)
• code-task: ×1.2 (有效)
• optimization: ×1.1 (有效)
• architecture: ×0.9 (效果不佳)

debug-helper:
• bug-hunt: ×1.8 (極度有效)
• optimization: ×1.2 (有效)

test-generator:
• bug-hunt: ×1.3 (有效)
• code-task: ×1.1 (有效)
• testing: ×1.5 (非常有效)

doc-writer:
• documentation: ×1.5 (非常有效)
• architecture: ×1.2 (有效)
• code-task: ×0.9 (效果不佳)
```

#### 弱點/抗性倍率

```
弱點倍率判定:
• 技能命中弱點 → 1.5倍
• 其他情況 → 1.0倍

抗性倍率判定:
• 敵人有抗性 → 0.5倍
• 其他情況 → 1.0倍
```

#### 最終傷害計算

```
最終傷害 = 向下取整(
  基礎傷害 ×
  相性倍率 ×
  弱點倍率 ×
  抗性倍率
)

範例 1: 普通攻擊
基礎傷害 = 290
相性倍率 = 1.0 (無相性)
弱點倍率 = 1.0 (無弱點)
抗性倍率 = 1.0 (無抗性)
→ 最終傷害 = 290

範例 2: 弱點攻擊
基礎傷害 = 290
相性倍率 = 1.5 (debug-helper vs bug-hunt)
弱點倍率 = 1.5 (擊中弱點)
抗性倍率 = 1.0
→ 最終傷害 = 向下取整(290 × 1.5 × 1.5) = 652

範例 3: 抗性攻擊
基礎傷害 = 290
相性倍率 = 0.8 (code-generator vs bug-hunt)
弱點倍率 = 1.0
抗性倍率 = 0.5 (敵人抗性)
→ 最終傷害 = 向下取整(290 × 0.8 × 0.5) = 116
```

#### 傷害顯示標籤

| 倍率 | 標籤 | 顯示顏色 | 字體大小 |
|-----|------|---------|----------|
| ≥2.0 | 極度有效！ | #ff0000 | 2.5em |
| ≥1.5 | 非常有效！ | #ff6600 | 2.2em |
| ≥1.3 | 很有效！ | #ff9900 | 2.0em |
| ≥1.1 | 有效 | #ffcc00 | 1.8em |
| 1.0 | - | #ffffff | 1.5em |
| <1.0 | 效果不佳 | #999999 | 1.2em |

### 回合制系統

#### 回合順序

```
行動順序計算:
基於速度值排序: 速度高者先行動

單位速度:
• 玩家: 基礎 50 + 裝備加成
• 夥伴: 基礎 30-60 (依職業)
• 召喚獸: 基礎 40-70
• 敵人: level × 3 + 隨機(0-10)

速度相同時優先級:
玩家 > 夥伴 > 召喚獸 > 敵人
```

#### 回合流程

```
┌─────────────────────────┐
│ 回合 N 開始              │
├─────────────────────────┤
│                         │
│ 1. 計算行動順序          │
│    [玩家, 夥伴1, 敵人, 夥伴2]
│                         │
│ 2. 玩家回合              │
│    • 選擇行動             │
│    • 執行技能/道具        │
│    • 更新狀態             │
│                         │
│ 3. 夥伴1 回合（自動）     │
│    • AI 選擇技能         │
│    • 執行行動             │
│    • 更新狀態             │
│                         │
│ 4. 敵人回合              │
│    • AI 選擇目標         │
│    • 執行攻擊             │
│    • 反擊機制（如有）     │
│                         │
│ 5. 夥伴2 回合（自動）     │
│    • AI 選擇技能         │
│    • 執行行動             │
│    • 更新狀態             │
│                         │
│ 6. 回合結束檢查           │
│    • 檢查戰鬥結束條件     │
│    • 更新技能冷卻         │
│    • 應用持續效果         │
│    • Buff/Debuff倒數     │
│                         │
│ 7. 進入下一回合          │
│    回合數 +1             │
└─────────────────────────┘
```

#### 戰鬥結束條件

```
勝利條件:
• 敵人 HP = 0
• 任務完成

失敗條件（可選）:
• 玩家 HP = 0
• MP 耗盡且無法行動
• 自願逃跑（消耗 30% MP）

獎勵計算:
baseExp = level × 30
baseGold = level × 15

finalExp = floor(baseExp × typeMultiplier × difficultyMultiplier × bonusMultiplier)
finalGold = floor(baseGold × typeMultiplier × difficultyMultiplier × bonusMultiplier)
```

```

統計:
• 出戰次數

```

### 夥伴召喚規則

```
召喚限制:
• 最多同時 2 個夥伴
• 召喚消耗: 20-30 MP
• 召喚冷卻: 200-300 秒
• 同一夥伴同時只能存在一個實例

召喚流程:
1. 選擇夥伴
2. 檢查 MP 是否足夠
3. 檢查夥伴是否在冷卻中
4. 播放召喚動畫（2s）
5. 夥伴加入戰場
6. 參與回合順序
```

### 夥伴 AI 行為

```
AI 決策優先級:

1. 玩家生命危險 (HP < 30%):
   → 優先使用支援技能 (護盾、治療)

2. 敵人類型相性:
   → 選擇對敵人有效的技能 (相性 ≥ 1.2)

3. 預設攻擊:
   → 使用可用的攻擊技能
   → 如無技能可用，執行普通攻擊

技能選擇邏輯:
遍歷夥伴所有技能:
  檢查技能是否在冷卻中 → 跳過
  檢查MP是否足夠 → 跳過

  計算該技能的預期傷害
  比較並記錄最高傷害的技能

選擇最佳技能執行
```

### 夥伴成長系統

```
經驗值獲取:
• 戰鬥勝利: 固定值
• 使用技能: 小量值
• 造成傷害: 傷害值 / 除數
• 治療玩家: 治療量 / 除數
• 命中弱點: 額外值

升級需求:
基礎值 × (成長率 ^ (等級 - 1))

範例:
Lv.1 → Lv.2: 100 EXP
Lv.2 → Lv.3: 130 EXP
Lv.3 → Lv.4: 169 EXP

升級獎勵:

• 攻擊力: +固定值
• 防禦力: +固定值
• 速度: +固定值
• 智慧: +固定值

特殊等級解鎖:
```

## 召喚獸系統

### 召喚獸分類

```
1. 技能召喚獸（Skill Summons）
   • 通過特定 Skill 召喚
   • 高 MP 消耗（60-100 MP）
   • 強力即時效果
   • 立即行動型

2. 組合技召喚獸（Combo Summons）
   • 觸發組合技時自動召喚
   • 不需手動召喚
   • 支援型
   • 持續 2-3 回合

3. MCP 工具召喚獸（MCP Tool Summons）
   • 綁定 MCP Server 工具
   • 實用型功能
   • 互動型行為
   • 持續 3 回合

4. 道具召喚獸（Item Summons）
   • 使用特殊道具召喚
   • 被動型
   • 長期增益
   • 持續 5 回合
```

### 召喚獸行為類型

```
1. Immediate（立即行動）
   • 召喚 → 技能施放 → 立即離去
   • 單次高爆發傷害
   • 適合緊急情況

2. Automatic（自動行動）
   • 持續 2-3 回合
   • 每回合自動選擇最佳技能
   • 參與回合順序

3. Passive（被動效果）
   • 持續 3-5 回合
   • 不執行主動技能
   • 提供持續性增益

4. Interactive（互動行動）
   • 持續 2-3 回合
   • 玩家手動選擇技能
   • 戰術性使用
```

### 召喚獸成本設計

|--------|---------|----------|----------|
| Common | 20-30 | 120s | 中等 |
| Uncommon | 30-50 | 180s | 較強 |
| Rare | 50-70 | 300s | 強力 |
| Epic | 70-90 | 450s | 很強 |
| Legendary | 90-120 | 600s+ | 極強 |

### 召喚獸槽位系統

## 組合技系統

### 組合技觸發條件

```
組合技需要:
1. 特定技能組合
2. 特定順序（有些需要）
3. 時間窗口（通常 30-60 秒）
4. 足夠的 MP

觸發流程:
使用技能 A
  ↓
在時間窗口內使用技能 B
  ↓
檢測組合技條件
  ↓
顯示組合技提示
  ↓
自動觸發組合技
  ↓
造成額外傷害/效果
```

### 組合技傷害計算

```
組合傷害 = 向下取整(
  (基礎值 + (技能數量 × 倍數) + (玩家等級 × 倍數)) ×
  經驗倍率
)

範例:
2 技能組合，玩家 Lv.10，EXP倍率 1.2:
組合傷害 = 向下取整((300 + 100 + 200) × 1.2) = 720
```

### 組合技視覺效果

```
組合技動畫:
1. 提示出現「組合技可用！」（0.3s）
2. 技能圖標連線特效（0.5s）
3. 畫面閃光（0.2s）
4. 組合技名稱大字彈出（1.0s）
5. 特殊攻擊動畫（2.0s）
6. 傷害數字（特大、金色）（1.0s）
7. 獎勵提示（經驗值加成等）（0.8s）

音效:
• combo_available.wav
• combo_trigger.wav
• combo_impact.wav
```
