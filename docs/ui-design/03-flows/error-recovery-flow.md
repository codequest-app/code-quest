# 錯誤恢復流程

**日期**: 2026-02-05
**版本**: v1.0

---

## 流程概述

本文檔說明系統錯誤檢測、分類、顯示、用戶操作選項、自動重試機制、狀態恢復策略和降級處理的完整流程。

### 目的
- 提供完善的錯誤處理機制
- 最小化用戶損失
- 提供清晰的錯誤資訊
- 支援自動和手動恢復

### 觸發條件
- 網路連接失敗
- API 請求錯誤
- 資料驗證失敗
- 系統異常
- 用戶操作衝突

### 預期結果
- 優雅降級
- 資料不丟失
- 用戶可繼續使用
- 錯誤可追溯

## 錯誤分類和嚴重程度

### 分級系統

| 等級 | 圖示 | 說明 | 用戶影響 | 恢復策略 |
|-----|------|------|---------|---------|
| 致命 | ⛔ | 無法繼續使用 | 完全阻塞 | 重啟應用 |
| 錯誤 | ❌ | 功能不可用 | 部分阻塞 | 重試/回退 |
| 警告 | ⚠ | 可能影響體驗 | 輕微影響 | 提示用戶 |
| 資訊 | ℹ | 僅供參考 | 無影響 | 顯示訊息 |

### 錯誤類型

```
網路錯誤 (Network)
├─ 連接中斷
├─ 請求超時
├─ DNS 解析失敗
└─ 服務器無響應

資料錯誤 (Data)
├─ 資料驗證失敗
├─ 資料格式錯誤
├─ 資料損壞
└─ 資料衝突

業務錯誤 (Business)
├─ 資源不足
├─ 權限不足
├─ 操作無效
└─ 狀態衝突

系統錯誤 (System)
├─ 記憶體不足
├─ 儲存空間不足
├─ 瀏覽器不支援
└─ 未預期異常
```

### 錯誤分類邏輯

```javascript
function classifyError(error) {
  // 致命錯誤
  if (error.code === 'SYSTEM_CRASH' ||
      error.message.includes('out of memory')) {
    return { level: 'fatal', canRetry: false };
  }

  // 網路錯誤（可重試）
  if (error.code === 'NETWORK_ERROR' ||
      error.code === 'ETIMEDOUT') {
    return { level: 'error', canRetry: true };
  }

  // 業務錯誤（不可重試）
  if (error.code === 'INSUFFICIENT_GOLD' ||
      error.code === 'INVALID_OPERATION') {
    return { level: 'business', canRetry: false };
  }

  // 警告
  if (error.code === 'DEPRECATED_API') {
    return { level: 'warning', canRetry: false };
  }

  // 預設為錯誤
  return { level: 'error', canRetry: true };
}
```

## 階段 4：自動重試機制

```
[步驟 1] 判定是否可重試
           │
           ▼
<可重試?>
  ╱      ╲
 否       是
 │        │
 │        ▼
 │   [步驟 2] 計算重試延遲
 │        │
 │        │ 指數退避算法
 │        │ Delay = baseDelay × 2^retryCount
 │        │
 │        ▼
 │   [步驟 3] 等待延遲
 │        │
 │        ├─→ 第 1 次：1 秒
 │        ├─→ 第 2 次：2 秒
 │        └─→ 第 3 次：4 秒
 │        │
 │        ▼
 │   [步驟 4] 執行重試
 │        │
 │        ▼
 │   <重試成功?>
 │     ╱      ╲
 │    是       否
 │    │        │
 │    ▼        ▼
 │   成功   <還有重試次數?>
 │   恢復    ╱      ╲
 │    │    是       否
 │    │    │        │
 │    │    ↻        ▼
 │    │  重試   顯示錯誤
 │    │         對話框
 │    │            │
 └────┴────────────┘
           │
           ▼
      繼續流程
```

### 重試策略

```javascript
async function retryWithBackoff(fn, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxRetries - 1) {
        // 最後一次重試失敗
        throw error;
      }

      // 計算延遲（指數退避）
      const delay = Math.min(1000 * Math.pow(2, i), 10000);

      // 顯示重試提示
      showRetryNotification(i + 1, maxRetries, delay);

      // 等待後重試
      await sleep(delay);
    }
  }
}
```

```
[自動檢查點]

     │
     ├─→ 角色狀態
     ├─→ 當前位置
     ├─→ 進行中的任務
     └─→ 臨時數據
     │
     ▼

     │
     ▼
<發生錯誤時>
     │
     ▼

     │
     ▼
恢復到檢查點狀態
```

### 恢復策略

```javascript
function recoverFromError(error) {
  // 1. 讀取最後檢查點
  const checkpoint = loadCheckpoint();

  if (!checkpoint) {
    // 無檢查點，返回初始狀態
    return resetToInitialState();
  }

  // 2. 驗證檢查點
  if (validateCheckpoint(checkpoint)) {
    // 3. 恢復狀態
    restoreState(checkpoint);

    // 4. 顯示提示
    showNotification({
      type: 'info',
      message: `已恢復到 ${formatTime(checkpoint.timestamp)} 的狀態`
    });

    return true;
  } else {
    // 檢查點無效
    return rollbackToPreviousCheckpoint();
  }
}
```

## 常見錯誤處理

### 1. 網路連接中斷

```
偵測：navigator.onLine = false
處理：
├─ 顯示離線提示
├─ 啟用離線模式
├─ 定期檢測重連
└─ 重連後自動同步

離線模式：
├─ 僅本地操作
├─ 禁用網路功能
├─ 隊列待同步操作
└─ 顯示離線標誌
```

### 2. API 請求失敗

```
偵測：HTTP 狀態碼 4xx/5xx
處理：
├─ 400：資料驗證錯誤 → 提示用戶修改
├─ 401：未授權 → 重新登入
├─ 403：權限不足 → 提示升級
├─ 404：資源不存在 → 返回上一頁
├─ 500：服務器錯誤 → 自動重試
└─ 503：服務維護 → 顯示維護公告

重試策略：
├─ 4xx：不重試（客戶端錯誤）
└─ 5xx：自動重試 3 次
```

### 3. 資料驗證失敗

```
偵測：Schema 驗證錯誤
處理：
├─ 高亮錯誤欄位
├─ 顯示具體錯誤訊息
├─ 提供修正建議
└─ 防止送出

範例：
「技能名稱」不能為空
「MP 消耗」必須在 10-100 之間
「冷卻時間」必須是整數
```

### 4. 資源不足

```
偵測：業務邏輯檢查
處理：
├─ 金幣不足 → 提示賺錢方式
├─ MP 不足 → 建議休息或使用道具
├─ 槽位已滿 → 提示整理背包
└─ 等級不足 → 顯示升級進度

用戶友好提示：
"金幣不足 (需要 500，你有 300)"
"建議：前往戰鬥賺取金幣"
[去戰鬥] [取消]
```

### 5. 狀態衝突

```
偵測：同時修改相同資源
處理：
├─ 樂觀鎖衝突 → 重新載入最新資料
├─ Worktree 合併衝突 → 智能合併
├─ 戰鬥中切換 → 警告並確認
└─ 並行操作衝突 → 隊列處理

衝突解決：
├─ 以服務器為準
├─ 以最新修改為準
├─ 手動選擇
└─ 智能合併
```

```
├─ 可能的原因
├─ 建議的解決方法
└─ 可執行的操作按鈕

範例：
「戰鬥啟動失敗」
原因：網路連接不穩定
建議：檢查網路連接並重試
[重試] [取消] [報告問題]
```

### 視覺回饋

```

├─ 致命：紅色 + 震動
├─ 錯誤：橙色 + 提示音
├─ 警告：黃色
└─ 資訊：藍色

動畫效果：
├─ 對話框淡入
├─ 錯誤圖標脈動
├─ 進度條顯示重試

```
