## 流程概述

本文檔描述從應用啟動到各種使用場景的完整用戶旅程。涵蓋首次使用者的引導流程、老用戶的快速存取路徑，以及常見的遊戲場景（戰鬥、技能創建、Worktree 管理）和錯誤恢復流程。

### 目的
- 提供完整的用戶體驗地圖
- 識別關鍵決策點和流程分支
- 優化新舊用戶的不同體驗路徑
- 確保所有場景都有明確的導航路徑

### 入口點
- 應用程式啟動
- 深度連結（從外部應用）
- 推送通知

### 預期結果
- 用戶能順利完成目標任務
- 提供清晰的導航和回饋
- 減少用戶困惑和錯誤操作

 │   │

### 快速存取功能

**快捷鍵系統**:
- `1-7`: 直達商業街 7 個商店
- `H`: 直達酒館
- `G`: 直達公會大廳

- `I`: 開啟背包
- `C`: 查看角色
- `S`: 技能列表

**最近使用**:
- 顯示最近 3 次訪問的場所
- 快速返回上次戰鬥地點
- 恢復未完成的任務

**收藏功能**:
- 標記常用技能（快捷施放）
- 收藏常去場所（一鍵傳送）
- 保存常用對話模板

### 狀態持久化

**自動保存**:
- 每 30 秒自動保存到本地
- 關鍵操作後立即保存（戰鬥結束、購買物品等）
- 退出應用前完整保存

**雲端同步**:
- 連接時每 5 分鐘同步
- 手動觸發同步
- 衝突解決策略（時間戳優先）

### 狀態恢復

**應用啟動**:
1. 讀取本地存檔
2. 嘗試雲端同步
3. 比對版本號
4. 恢復遊戲狀態
5. 繼續未完成的任務

**錯誤恢復**:
- 狀態損壞時回滾到上一個檢查點
- 保留最近 5 個存檔版本
- 提供手動恢復選項

## 用戶體驗考量

### 載入指示器

**全屏載入**（應用啟動）:
- Logo 動畫
- 進度條顯示百分比
- 載入提示文字輪播
- 預計時間：2-3 秒

**局部載入**（場景切換）:
- 淡入淡出動畫
- 小型 spinner
- 不阻塞 UI
- 預計時間：0.3-0.8 秒

**背景載入**（資源預載）:
- 靜默載入
- 不顯示指示器
- 完成後觸發事件

### 進度回饋

**戰鬥進度**:
- 回合數顯示（第 X 回合）
- HP 條即時更新
- 技能冷卻倒數
- 傷害數字彈出

**任務進度**:
- 進度百分比
- 步驟完成勾選
- 預計剩餘時間
- 可中斷提示

**非同步進度**:
- 右側面板實時更新
- WebSocket 推送通知
- 完成時音效提示
- 結果摘要顯示

### 確認對話框

**高風險操作**（必須確認）:
- 刪除 Worktree
- 花費大量金幣
- 放棄任務進度
- 重置設定

**中風險操作**（可選確認）:
- 購買物品
- 離開戰鬥（逃跑）
- 切換 Worktree
- 合併分支

**低風險操作**（無需確認）:
- 場景切換
- 查看資訊
- 施放技能
- 使用道具（戰鬥中）

### 成功/失敗通知

**成功通知**（綠色，3秒消失）:
- ✓ 技能創建成功
- ✓ 購買完成
- ✓ 戰鬥勝利
- ✓ 升級達成

**失敗通知**（紅色，5秒消失）:
- ✗ 資源不足
- ✗ 戰鬥失敗
- ✗ 操作無效
- ✗ 連接錯誤

**警告通知**（黃色，4秒消失）:
- ⚠ HP 低於 20%
- ⚠ MP 不足
- ⚠ 即將斷線
- ⚠ 存檔衝突

### 鍵盤快捷鍵

**全局快捷鍵**:
- `Esc` - 返回上一層 / 取消操作
- `Enter` - 確認 / 送出
- `Tab` - 切換焦點
- `Space` - 快速確認

**探索模式**:
- `1-7` - 快速傳送到商店

**戰鬥模式**:
- `1-9` - 快速施放技能 1-9

- `Space` - 普通攻擊
- `Q/E` - 切換技能頁

**提示顯示**:
- 懸停在按鈕上顯示快捷鍵
- 首次使用時高亮提示
- 設定中可查看完整快捷鍵表

## 實作注意事項

### 技術需求

1. **狀態管理**
   - 使用 Redux 或 Zustand 管理全局狀態
   - 實作 undo/redo 機制
   - 狀態序列化和反序列化

2. **路由管理**
   - 使用 React Router 或類似框架
   - 支援深度連結
   - 歷史記錄管理

3. **資料持久化**
   - IndexedDB 本地存儲
   - 壓縮存檔減少空間
   - 定期清理舊存檔

4. **網路通信**
   - WebSocket 實時通信
   - 斷線重連機制
   - 請求隊列和重試

### 性能考量

**首次啟動**:
- 目標：< 3 秒進入遊戲
- 懶加載非關鍵資源
- 預載關鍵圖像和音效
- 壓縮和最小化資源

**場景切換**:
- 目標：< 0.5 秒完成轉場
- 使用 CSS transform（GPU 加速）
- 避免 layout thrashing
- 卸載不可見元素

**記憶體管理**:
- 限制歷史記錄數量
- 及時釋放大型物件
- 圖片懶加載和卸載
- 監控記憶體使用

### 測試場景

**功能測試**:
1. 新用戶完整引導流程
2. 老用戶快速啟動
3. 所有場景切換路徑
4. 資源不足情況
5. 網路中斷和恢復

**壓力測試**:
1. 連續 100 次戰鬥
2. 創建 50 個 Worktree
3. 背包 1000 件物品
4. 10 個非同步戰鬥同時進行

**邊界測試**:
1. 等級 MAX 後繼續獲得經驗
2. 金幣溢出（超過 2^31）
3. 極長的技能名稱
4. 網路極慢（2G）情況

### 已知限制

1. **同時戰鬥上限**: 最多 3 個非同步戰鬥
2. **Worktree 數量**: 建議不超過 10 個
3. **背包容量**: 1000 件物品
4. **技能數量**: 100 個自定義技能
5. **存檔大小**: 約 5-10 MB
