# 非同步戰鬥流程

**日期**: 2026-02-05
**版本**: v1.0

---

## 流程概述

本文檔說明非同步戰鬥系統（Plan Mode），從啟動規劃模式、設定多戰鬥隊列、Worktree 分配、並行執行（最多 3 場）、即時更新（WebSocket）到結果聚合和合併決策。

### 目的
- 支援多任務並行處理
- 提升遊戲效率
- 減少等待時間
- 整合 Worktree 系統

### 入口點
- 快捷鍵 P 進入 Plan Mode
- 系統自動偵測複雜任務
- 主動規劃多個戰鬥

### 預期結果
- 同時執行多個戰鬥
- 即時查看進度
- 自動合併結果

### 相關流程
- 【參考：battle-flow.md】- 戰鬥系統
- 【參考：worktree-flow.md】- Worktree 管理
- 【參考：screen-transitions.md】- 時間凍結特效

### 時間凍結特效（參考 screen-transitions.md）

```
時間軸：
0.0s  所有動畫速度 × 0.5
0.5s  顏色飽和度降至 0 (灰階)
1.0s  靜止粒子出現（白色微粒）
1.5s  時鐘指針停止
1.5s  靜止之間界面淡入
2.0s  Plan Mode 看板滑入
2.5s  轉場完成

音效：time_stop.wav + stasis_ambient.mp3
```

```
事件類型：
├─ battle_start：戰鬥開始
├─ battle_update：進度更新
├─ battle_complete：戰鬥完成
├─ battle_failed：戰鬥失敗
├─ level_up：升級
└─ rare_drop：稀有掉落

├─ Toast 提示 (右上角)
├─ 音效提示
├─ 戰鬥卡片閃爍

```

### 合併衝突解決

```javascript
function resolveConflicts(battles) {
  const resolved = {
    hp: Math.max(...battles.map(b => b.player.hp)),
    mp: battles.reduce((acc, b) => acc - b.mpLost, initialMp),
    gold: battles.reduce((acc, b) => acc + b.goldGained, 0),
    exp: battles.reduce((acc, b) => acc + b.expGained, 0),
    items: battles.flatMap(b => b.itemsGained)
  };

  return resolved;
}
```

## 性能和限制

### 並行限制

```
最多同時執行：3 個戰鬥
最多排隊數量：10 個任務
Worktree 上限：7 個（3 並行 + 4 備用）
```

### 資源隔離

```
每個戰鬥：
├─ 獨立 Worktree
├─ 獨立戰鬥狀態
├─ 獨立 HP/MP 計算
└─ 獨立獎勵累積

合併時：
├─ 智能衝突解決
├─ 數值累加/取優
└─ 自動清理臨時分支
```

### WebSocket 更新頻率

```
戰鬥進度：每 2 秒更新
重要事件：即時推送
完成通知：即時推送

帶寬優化：
├─ Delta 更新（僅傳送變化）
├─ 壓縮數據
└─ 批次推送
```
