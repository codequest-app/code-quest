## 📚 文檔索引

本目錄包含所有主要流程的詳細文檔，描述從用戶輸入到系統響應的完整流程。

### 核心流程

1. **[user-journey.md](./user-journey.md)** - 完整用戶旅程圖
   - 從應用啟動到各種場景
   - 首次用戶引導流程
   - 老用戶快速存取
   - 常見使用路徑
   - 邊緣情況處理

2. **[battle-flow.md](./battle-flow.md)** - 戰鬥流程詳解
   - 戰鬥觸發方式
   - 回合制戰鬥循環
   - 技能執行機制
   - 夥伴和召喚獸整合
   - 勝利/失敗處理

3. **[skill-creation-flow.md](./skill-creation-flow.md)** - 技能創建流程
   - 6 步驟創建流程
   - Prompt 模板調整
   - 工具權限配置
   - 測試和迭代
   - 保存和管理

### 管理流程

4. **[worktree-flow.md](./worktree-flow.md)** - Worktree 工作流程
   - 公會大廳導航
   - 3 種視圖模式
   - 創建和切換分支
   - 合併和清理
   - 非同步戰鬥整合

5. **[companion-flow.md](./companion-flow.md)** - 夥伴召喚和管理流程
   - 傭兵公會訪問
   - 瀏覽和召喚夥伴
   - 陣型配置
   - 戰鬥中自動行動
   - 升級和培養

### 交易流程

6. **[shop-flow.md](./shop-flow.md)** - 商店購物流程
   - 7 種商店介紹
   - 商品瀏覽和篩選
   - 購買確認流程
   - 庫存更新
   - 特殊機制（特價、會員）

### 進階流程

7. **[async-battle-flow.md](./async-battle-flow.md)** - 非同步戰鬥流程
   - Plan Mode 啟動
   - 多戰鬥隊列設定
   - 並行執行（最多 3 場）
   - WebSocket 即時更新
   - 結果聚合和合併

8. **[error-recovery-flow.md](./error-recovery-flow.md)** - 錯誤恢復流程
   - 錯誤分類系統
   - 自動重試機制
   - 狀態恢復策略
   - 降級處理
   - 錯誤報告

### 轉場動畫

9. **[screen-transitions.md](./screen-transitions.md)** - 畫面轉場流程圖
   - 所有場景轉場
   - 動畫時序規格
   - 特殊轉場效果
   - 音效配置
   - 性能優化

## 📖 如何閱讀流程圖

### ASCII 流程圖符號說明

```
符號指南：

方框（步驟）：
┌─────────┐
│  步驟   │  - 流程中的一個步驟
└────┬────┘

箭頭（方向）：
     │       - 垂直流向
     ▼       - 向下
     →       - 向右
     ←       - 向左
     ↑       - 向上
     ↻       - 循環/重複

決策（條件判斷）：
<條件判斷?>  - 菱形決策點
  ╱    ╲
 是     否

連接線：
  │      - 直線連接
  ├─     - 分支
  └─     - 匯合

特殊符號：
  ✓      - 成功路徑
  ✗      - 失敗路徑
  ⚠      - 警告
  ℹ      - 資訊
  [按鈕]  - 用戶操作
  [步驟 N] - 步驟編號

強調：
═══════  - 重要流程
───────  - 分隔線
╔═════╗  - 強調區塊
║     ║
╚═════╝
```

### 閱讀順序建議

1. **先看概述**
   - 閱讀「流程概述」章節
   - 了解目的和預期結果
   - 確認相關流程

2. **理解主流程**
   - 查看完整流程圖
   - 理解整體結構
   - 識別關鍵決策點

3. **深入各階段**
   - 按順序閱讀各階段
   - 查看詳細步驟說明
   - 理解資料流和狀態管理

4. **查看範例**
   - 閱讀使用案例
   - 對照 ASCII 界面示意
   - 理解實際應用

5. **參考相關文檔**
   - 查看畫面參考連結
   - 查閱組件文檔
   - 理解完整脈絡

## 📝 文檔規範

### 每個流程文檔包含

1. **流程概述** - 目的、入口點、預期結果
2. **完整流程圖** - ASCII 藝術流程圖
3. **階段說明** - 詳細步驟描述
4. **決策點** - 條件判斷和分支
5. **狀態管理** - 資料流和狀態轉換
6. **畫面參考** - 相關畫面連結
7. **資料結構** - 關鍵資料格式
8. **用戶體驗** - UX 考量
9. **實作注意事項** - 技術要點
10. **範例和使用案例** - 實際場景

### 命名規範

- 檔名：小寫字母 + 連字符
  - ✓ `battle-flow.md`
  - ✗ `BattleFlow.md`

- 標題：中文 + 清晰描述
  - ✓ `戰鬥流程詳解`
  - ✗ `Battle`

- 步驟編號：[步驟 N]
  - ✓ `[步驟 1] 檢查條件`
  - ✗ `Step 1: Check`

### 引用規範

**內部引用**（同一文檔）:
```markdown
參考上方「階段 3」
```

**跨文檔引用**（其他流程）:
```markdown
【參考：battle-flow.md】- 戰鬥系統
```

**畫面引用**（02-screens）:
```markdown
【參考：02-screens/battle/battle-main.md】
```

**組件引用**（04-components）:
```markdown
【參考：04-components/button.md】
```

## 🎨 流程圖最佳實踐

### 設計原則

1. **自上而下**
   - 流程由上往下流動
   - 主流程居中
   - 分支向兩側

2. **清晰對齊**
   - 使用等寬字體
   - 對齊框線和箭頭
   - 保持統一間距

3. **適當分組**
   - 相關步驟靠近
   - 使用分隔線
   - 階段性區塊

4. **標註完整**
   - 步驟有編號
   - 條件有說明
   - 路徑有標籤

### 複雜度控制

**簡單流程**（< 10 步）:
- 單一流程圖
- 線性流程
- 少量分支

**中等流程**（10-20 步）:
- 分階段流程圖
- 適度分支
- 子流程引用

**複雜流程**（> 20 步）:
- 總覽 + 詳細流程圖
- 多層次結構
- 大量子流程引用
